## 3. 关键数据流

### 3.1 鉴权（本地模式）
```
Oa() (L3543) → 永远返回本地 token（不连远程）
Ba() (L3637) → 判断是否已登录
Jh() (L30712) → 给 fetch 注入 authHeaders（token 来自 AUTH_TOKEN_KEY=localStorage 'auth_token'，config L33）
```

### 3.2 双服务 base URL 桥接（红线 §3.2 三层隔离）
```
localTool base = LOCAL_ENGINE.base (config L15) = localEngineBase() (config L42)
               = USE_LOCAL_ENGINE ? 'http://127.0.0.1:18080' : REMOTE_BASE('http://127.0.0.1:9004')
网关 base     = ENDPOINTS[0].url (config L26) = 'http://127.0.0.1:9004'
ar(e,t) (L863) 按开关解析具体 API 基址
```
> 风险点（ARCHITECTURE X3.3）：`USE_LOCAL_ENGINE=false` 时文件请求会走 9004（无上传路由）→ 失败。当前 `USE_LOCAL_ENGINE=true` 固定，不触发。

### 3.3 存储后端
```
Nr (L1418) = localStorageBackend → chrome.storage/localStorage
Pr (L1446) = localforageBackend   → localforage（画布状态持久化）
jr (L1345) = cachedGet            → 带 TTL 缓存的 GET，减少重复请求
Cr (L1294) = canvasStateKey       → 生成 'canvas-state-v1-{projectId}'
```

## 5. 存疑 Bug / 排雷

- **无新增 Bug**：模块1 主要是配置解析与存储封装，本次未发现越界风险。
- **大小写陷阱**：`Oa`/`oa`、`Ba`/`ba`、`Cr`/`cr` 均为不同物，引用必须带行号（红线 §3.1 命名红线）。
- **base URL 硬编码**：`LOCAL_ENGINE` host 写死 `127.0.0.1`（config L13），与 TASKS P0「host 硬编码 18080」同源，属已知残留。
