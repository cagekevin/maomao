### 与「拖入 URL 不落盘」割裂面归因（TASKS P1）

- 资源面板 URL 拖入（TASKS 称 B 回调 L29176）只存 URL 到状态，不调 ii()/Sv()，刷新丢失。
- 而 rescan 只扫**磁盘** `uploadDir`，URL 型资源本就不在磁盘，故 rescan 无法补回 URL 型资源 → 二者割裂。
- 真身锚点：AI05 已纠偏——上传/拖入的真实局部回调是 `z`(L29159) 等组件内 useCallback，非模块级 `R`/`B`；TASKS 行14/15 的符号名 `R`/`B` 需回填（见 03）。

## 交叉流小结

- 两条流在 `Ev()`(L42883) 与 `index.ts L196` 接缝处汇合，证明**事件总线是 rescan 的唯一前端触发源**，localTool 不主动推。
- 事件总线实际只有 3 个（`mutiwindow-task-completed` / `mutiwindow-update-task-meta` / `resourceAdded`[跨进程]），ARCHITECTURE X2 表的 `canvas-state-change`/`mutiwindow-sync-local` 不存在（见 03）。
- 配置桥：localTool base 走 `LOCAL_ENGINE.base`(config L15)；网关 base 走 `ENDPOINTS[0].url`(config L26)。两服务 base 在 USE_LOCAL_ENGINE=true 时分别固定 18080/9004，不混淆。
