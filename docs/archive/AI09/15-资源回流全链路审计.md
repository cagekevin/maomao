## 三、完整回流链：`we` 是 glue（02/05/08 未读体）

**组件级 `we`(App.js L43015)**：
```
we = Y.useCallback(async () => {
  if (n.status.port) try {
    await Ev(),                              // ① 触发 localTool rescan（POST /rescan）
    D((await xv({ page:1, pageSize:300, sortBy:'timestamp', sortDir:'DESC' })).items);  // ② 拉全量列表写面板 state D
  } catch(e) { console.error('Failed to sync transit resources from local:', e); }
}, [n.status.port]);
```
- `n.status.port` = localTool 连接端口（USE_LOCAL_ENGINE 时 18080）。
- **① `await Ev()`** = module-level rescan（POST /rescan），令 localTool 重扫磁盘/DB 入库。
- **② `D(await xv(...).items)`** = 立即 `xv`(GET /resources) 拉最新列表并写入面板 state `D`。
- 即 **`Ev` 负责"让后端重扫"，`xv` 负责"把最新列表拉回前端"**，二者由 `we` 串成"先 rescan 再 refetch"的原子回流。

**节流封装 `rescanThrottledSync`(L43028)**：
```
rescanThrottledSync = Y.useCallback(async () => {
  let e = Date.now();
  if (e - rescanLastRunRef.current < 3e3) return;   // 3 秒内去重
  rescanLastRunRef.current = e, await we();
}, [we]);
```
- `Di.current = we`(L44350) —— 模块3 生成完成回调 `Di.current?.()` 即调 `we`。
- 监听 `mutiwindow-task-completed`(L31428) 时 `i==='completed' && n && (Ev().then(()=>Di.current && Di.current()))`（L31426）—— 先 `Ev` 再 `we`(=refetch)，与 `we` 内部序一致（双保险）。

## 四、缝合图（修正 02/05/08 的简写）

```
AI 生成完成
  → dispatch mutiwindow-task-completed (L38481)
  → 监听 L31428 handler:
      W(nodes.map(...写回节点 data...))        // 节点结果回填（见 11 质询2）

```
