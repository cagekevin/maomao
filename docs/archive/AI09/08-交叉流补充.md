## 流三：配置加载与双服务 base URL 桥接（T2.5.3）

### 链路

```
① 配置层加载（config.js，行号与 ARCHITECTURE L1.5 完全一致）
   LOCAL_ENGINE = {host:'127.0.0.1', port:18080, get base()} (config.js L12–L18)
   USE_LOCAL_ENGINE = true (config.js L36)
   localEngineBase() = USE_LOCAL_ENGINE ? LOCAL_ENGINE.base : REMOTE_BASE (config.js L42–L44)
   ENDPOINTS = [{label:'API网关', url:'http://127.0.0.1:9004'}] (config.js L25–L27)

② 前端引用两处 base（实测同源）
   Hr = localEngineBase()        → App.js L1732（上传/下载等走 localTool）
   vv = LOCAL_ENGINE.base         → App.js L42808（资源 save/delete/rescan 走 localTool）
   U_ = LOCAL_ENGINE.base         → App.js L41570（另一处引用）
   R / gatewayBase = ENDPOINTS[0].url → 网关 9004（Jn 内 L32796 / L33005）

③ 决定请求去向
   USE_LOCAL_ENGINE=true（当前）:
     文件/KV/资源 → 127.0.0.1:18080 (localTool)
     AI 生成      → 127.0.0.1:9004 (gateway)
   USE_LOCAL_ENGINE=false（切换时）:
     文件请求本应走 REMOTE_BASE=9004，但 9004 无 /api/files/* 路由
     → 文件上传/读取失败（ARCHITECTURE X3.3 已记，属已知切换坑）
```

### 接缝校验

| 跳 | 锚点 | 介质 | 状态 |
|----|------|------|------|
| ①→② | config L42 → App.js L1732 | 模块导入 | ✅ |
| ② 同源 | Hr L1732 / vv L42808 / U_ L41570 均 = LOCAL_ENGINE.base | 常量 | ✅ |
| ②→网关 | ENDPOINTS L26 → Jn L32796 | 字符串拼接 | ✅ |

### 排雷结论
- **P0 残留（host 硬编码）**：`LOCAL_ENGINE.host` 写死 `127.0.0.1`（config L13），非可配置；中文路径 Latin1 乱码待修（见 TASKS）。
- **base URL 一致性**：当前 `USE_LOCAL_ENGINE=true` 下 `Hr`/`vv`/`U_` 三者同源（均 `LOCAL_ENGINE.base`），TASKS 旧文「ResourceAdded base URL 不一致」仅在 `false` 时触发（ARCHITECTURE X3.3 已澄清），**当前运行不触发**。
- **切换坑**：`false` 时文件请求落 9004 无路由 → 失败。非当前运行问题。
