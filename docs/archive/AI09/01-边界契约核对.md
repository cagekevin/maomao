## 二、window CustomEvent 契约（仅前端进程内）

> ⚠️ **重大纠偏**：ARCHITECTURE §X2「跨层事件总线」表列出 5 个事件，但本次 grep `src/` 实测，**只有 3 个真实存在**：
> - `mutiwindow-task-completed` ✅
> - `resourceAdded` ✅（仅作为 `sendMessage` action 接收，源在 background.ts）
> - `mutiwindow-update-task-meta` ✅
>
> **`canvas-state-change` 与 `mutiwindow-sync-local` 在 `src/` 整个目录 grep 为 0 匹配，属 ARCHITECTURE 错锚（见 03 纠错清单）**。

| 事件 | 触发（dispatchEvent）行号 | 监听（addEventListener）行号 | 备注 |
|------|------|------|------|
| `mutiwindow-task-completed` | L38481 · L43640 · L43676 · L43697 · L44406 | L31428（注册+注销成对）| 统一同步/任务完成 → 触发 rescan |
| `resourceAdded` | **不在前端 dispatch**（由 `background.ts` L102 `chrome.runtime.sendMessage({action:'resourceAdded'})` 发出） | 前端以 `e.action === 'resourceAdded'` 接收，App.js **L43527** | 跨进程消息，非 window CustomEvent |
| `mutiwindow-update-task-meta` | L41032 | L43764 | 任务元信息更新 |

> 注意区分：`resourceAdded` 是 Chrome 扩展 `runtime.sendMessage` 的 action 字段（跨进程），不是 `window.dispatchEvent` 的 CustomEvent；其余两个是 `window` CustomEvent（前端进程内）。

## 五、边界契约小结（给 02 交叉流用）

- 前端生成 → 网关：`Jn`(L32490) → `POST /v1/images|videos/generations`(网关 L591/L641) → 轮询 `GET /v1/tasks/{id}`(网关 L873，前端 pollUrl L33005)。
- 落盘：结果下载 → `ii()`(L1888) → `POST /api/files/upload`(index.ts L135)。
- 刷新：dispatch `mutiwindow-task-completed`(L38481 等) → 监听 L31428 → 触发 `Ev()`(L42883) rescan → `POST /api/resources/rescan`(index.ts L196)。
- 配置桥：localTool base = `localEngineBase()`(config L42) = `LOCAL_ENGINE.base`(L15)；网关 base = `ENDPOINTS[0].url`(config L26)。
