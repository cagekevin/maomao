## 3. 关键数据流

### 3.1 拖放进入画布
```
onDrop (ReactFlow 装配 `onDrop={Lr}` @ App.js:L37138；`Lr` 定义 @ App.js:L36293)
  → application/x-yimao-template → 批量导入节点
  → application/x-mutiwindow-task → 创建节点（调 Z App.js:L35536）
  → dataTransfer.files → ii({subfolder:'canvas/drop'}) (App.js:L1888) → POST /api/files/upload (index.ts L135)
```

### 3.2 连线与状态
```
onConnect (xn App.js:L31883) → 建立边
onConnectEnd (Fr App.js:L36245) → 连线结束处理
onNodeClick (kn App.js:L31980) / onNodeContextMenu (wn App.js:L31947) / onPaneContextMenu (Cn App.js:L31933) → 交互
```

### 3.3 持久化
```
ReactFlow onChange (onNodesChange=fe @ App.js L37123) → 受控 state 写入 (W/G)
  → 变更检测后 r.nodes = e (App.js L31722)
  → Q.saveCanvasStateWithVersion(en.current, r, t) (App.js L31728；StorageManager 方法 L1642/L1650)
  → Cr(canvasStateKey) (App.js:L1294) 生成键 → localforage 'canvas-state-v1-{projectId}'
```
> 注意：ARCHITECTURE X2 称「canvas-state-change → localforage 保存」，但 `canvas-state-change` CustomEvent 在 `src/` 全目录 grep **0 匹配**（见 03 纠错）。实际持久化经 **ReactFlow 受控 state 变更 → `Q.saveCanvasStateWithVersion`(StorageManager, App.js L31728)**，而非事件总线，亦非直接 `Pr` 写。门4 对抗审计已实测坐实（L31722→L31728 调用链）。
