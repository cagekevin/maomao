## D. 模块行为补丁（源码层，需用户授权后实施）

### C10 · `ar`(L868-871) 死分支（配置-1 坐实）
- 现象：仅当 `$n` 非 localhost/127 时把 `localhost:3000` 导向 `$n`；本地模式 `$n`=127.0.0.1:9004 → 分支不触发。
- 建议：若需 dev 代理生效，改为「始终将 `localhost:3000` 视为本地引擎代理目标」；或显式文档标注其为本地模式死代码（非阻断）。
- 风险：低；仅影响 dev 代理路径。

### C11 · `Kn`(vendor) 字面值解包（轮询-3 遗留）
- 动作：用解包工具（rolldown 反混淆 / source-map）提取 `vendor-Cr1JWW-B.js` 中 `Kn` 导出字面值，确认是否 = `DEFAULT_GATEWAY_URL`(9004)。
- 现状：源码层面已锁语义，字面值待解包钉死（非阻断）。

### C12 · TASKS P1 行号与语义修正（13 文档坐实）
- `R`(L29149) 实为 File→dataURL 读取器，非「上传回调」；真上传动作是 `z`(L29159)。
- `B`(L29188) 实为 drop 回调；「不落盘」指图片输入区内存状态 `O`，非资源面板（资源面板 `resourceAdded`@L43527 已落盘）。
- `Sv`(L42838)/`wv`(L42857)/`Ev`(L42883)/`we`(L43015) 行号与 T0.1/TASKS 原标不符，需更正。
- 动作：TASKS P1-2/P1-3 重写行号与机制描述；T0.1 `we` 标 L43015、`Ev` 标 L42883。

### C13 · 09 文档 E-4 方向修正（14 文档坐实）
- `Zn`(canvas-add-resource-request) 监听端 @L35664 的处理函数是「按 type spawn 画布节点」（放置流），非资源入库。
- `Zn`(L168 事件名) 与 `resourceAdded`(L43527 入库流) 方向相反：前者资源面板→画布，后者 background→前端入库。
- 动作：09 文档 E-4 更正监听点 L35664 + 显式区分两事件方向。

### C14 · `pending_confirmation` 卡死根因（15 文档坐实，升级为真实 Bug）
- 现象：TASKS 标「已知限制（无害）」；实际 `markNeedsConfirm`(@L32980) 写 `await_confirm` + `confirmTaskId` 等字段后，**全 App.js 无任何消费端读取这些字段续轮询/发确认请求**。
- 结论：`AUTO_CONFIRM=false` 时生成任务永久卡 `await_confirm`，前端无确认入口 → 真实功能缺口（半截实现），非「无害」。
- 建议修复（需用户授权）：① 在节点 UI 渲染 `await_confirm` 时提供「确认/取消」按钮，点击后读取 `confirmTaskId`/`confirmGatewayBase`/`confirmToken` 调 `zc(\`${base}/v1/tasks/${id}/confirm\`, {method:'POST', ...})` 并续轮询；或 ② 设 `AUTO_CONFIRM=true` 时由网关自动确认（前端无需改动）。
- 补充：`zc`(@L19001) 代理桥接机制补全进 T2.5.3（localTool 兼作 9004 网关 `/api/proxy` 跳板）；`H`(useLocalTool, `Uc`@L19052) 状态结构 `{isConnected,port,version,message}` 坐实。
