## 一、链路全景（资源面板 → 画布节点）

```
资源面板点「添加到画布」
   └─ L(e) = dispatchEvent(new CustomEvent(Zn, { detail: resource }))   [派发端 L212-216]
        │  (window CustomEvent，跨组件事件总线)
        ▼
画布组件 useEffect 监听 Zn                                          [监听端 L35631-35664]
   └─ 处理函数 e(detail):
        ├─ 校验 detail.url 存在且非 folder                              (L35634)
        ├─ 计算画布中心坐标 r = we({x,y})                               (L35636)
        └─ 按 detail.type 调 Z(type, r, data) spawn 节点               (L35649-35662)
              ├─ text/*       → Z('textNode', r, {text, label})
              ├─ video/*      → Z('videoExtractNode', r, {videoUrl})
              ├─ audio/*      → Z('audioPlayerNode', r, {audioUrl})
              └─ 其他(图片)   → Z('imageNode', r, {imageUrl, label})
```

## 二、核心混淆字典（本流专属）

| 混淆名 | 真身 | 行号 | 角色 |
|--------|------|------|------|
| `Zn` | `canvas-add-resource-request`（事件名常量） | L168 定义 / L213 派发 / L35664 监听 | 跨组件推送事件 |

## 三、关键数据流与边界契约

### 接缝清单（带 file:line）
- **CustomEvent 名**：`canvas-add-resource-request`（L168）
- **派发**：`window.dispatchEvent(new CustomEvent(Zn, {detail}))`（L213）
- **监听**：`window.addEventListener(Zn, e)`（L35664，清理在 L35664 同一句 `() => removeEventListener`）
- **消费**：`Z(type, pos, data)` spawn（L35536 / 调用 L35649-35662）
- **依赖 config**：无（纯前端事件，不经过 localTool/网关）

### 行为要点
1. **单向推送**：资源面板 → 画布。画布监听端**只消费、不回写**——不调 `Sv` 入库、不调 `Zr` 下载。
2. **直接引用 url**：节点 `imageUrl`/`videoUrl`/`text` 直接取自 `detail.url`。该 url 若来自资源面板已入库资源，通常已是 `migrated/` 内部地址（经 `resourceAdded`@L43527 的 `Zr` 下载落盘）；若是图片输入区 `O` 内存的 dataURL/直链，则节点直接持有原 url。
3. **与 `resourceAdded` 方向相反**：`resourceAdded`（L43527）是 background→前端「入库」流；`Zn`（L35664）是资源面板→画布「放置」流。二者都叫"资源相关事件"，但方向/目的不同，不可混淆。

## 四、对前序文档的修正（回填 09 E-4）

`09` 文档 E-4 原标：`Zn`(canvasAddResourceEvent) 监听点在 L43357。
- **错误**：L43357 的 `Zn = (e,t)=>{...}` 是 `adjustApiConfigs` 配置加载函数（同名遮蔽），与事件无关。
- **正确**（门4 已修正 + 本专项坐实）：
  - 事件名常量 `Zn` = `canvas-add-resource-request`（L168）
  - 派发端 `L` @L212-216（资源面板）
  - **监听端 @L35664**，处理函数 `e`（L35632-35663）按 type 调 `Z` spawn 画布节点
  - **方向**：资源面板 → 画布节点（放置流），非资源入库流

建议 `09` 文档 E-4 更正为上述描述，并显式区分 `Zn`(画布放置) 与 `resourceAdded`(入库) 两个方向。
