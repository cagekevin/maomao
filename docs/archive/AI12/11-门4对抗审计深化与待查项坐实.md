### E-4 · `Zn` 监听点 —— 已证 + **原标注错误修正**
- **原 09 文档误标 L43357 为监听点**：错误。L43357 `Zn = (e, t) => {...}` 是 `adjustApiConfigs` 配置加载函数，与事件 `canvas-add-resource-request` **无关**，属同名遮蔽陷阱。
- **真实监听点 = L35664**：`window.addEventListener(Zn, e)`，处理函数 `e` 定义于 L35663 之前，依赖 `[we]`（Tt 状态）。
- 修正：09 §三 E-4 监听点应更正为 L35664；并显式标注 L43357 的 `Zn` 为同名遮蔽函数（避免后续审计再误判）。

### 配置-1 · `$n`(apiBaseUrl) 真实赋值 —— 已证
- `var $n = Vn()` @**L844**（模块顶层）。
- `Vn` 来源：`App.js` L4 `import { ..., r as Vn, ... } from "./entry.js"` → entry.js `r as Vn` = entry.js 的 `y` = `getActiveApiEndpoint()`。
- entry.js 链路：`y()` = `f() || d()`（L87-89）；`f()` 读 `sessionStorage['active_api_endpoint']`（L24-29）；`d()` = `u[0]?.url || c(DEFAULT_ENDPOINT)`（L20-21，`DEFAULT_ENDPOINT = 'http://127.0.0.1:9004'` config.js L30）。
- **结论**：`$n`(apiBaseUrl) = 运行时动态解析的「激活网关 base」：默认 `http://127.0.0.1:9004`，可被 `sessionStorage` 或 localTool `/api/kv` 接入点引导覆盖（`b()`/`_` L63-95，`x`=setEndpoint）。
- 死分支确认：`ar`(L863-871) 仅当 `$n`(官方base) **非** localhost/127 时，把 `localhost:3000` 请求导向 `$n`。本地模式 `$n` 默认 `127.0.0.1:9004` → 该分支**不触发**（dev 代理逻辑本地模式形同死代码）。门4 质询成立，标注为真实死分支（非阻断）。

### 轮询-3 · `R`(网关base) 来源 —— 已证（关键架构发现）
- **原假设错误**：前序文档假设 `R` 是「生图组件内局部变量，由 props/context 注入」。实际：
- `R` = `App.js` L3 import `Kn as R`（模块顶层导入别名，来自 vendor 打包产物 `vendor-Cr1JWW-B.js`）。
- App.js 内**无** `let R =`（模块级）赋值；唯一 `let R =` 在 L7212 是另一组件（画布尺寸 `useMemo`），属局部遮蔽，与 Jn 的 `R` 无关。
- Jn 用法佐证其为 **URL 字符串**：
  - L33005：`let pollUrl = \`${R}/v1/tasks/${taskId}\``（网关轮询）
  - L33061：`markNeedsConfirm(z, cid, 'image', R, h)`（确认弹窗携带网关 base）
- **结论**：生图轮询 base `R` 与 `$n`(apiBaseUrl=Vn()) **来源相互独立**——`R` 是 vendor 顶层常量（网关语义，默认与 DEFAULT_GATEWAY_URL 同族 9004），`$n` 是运行时动态解析。两者默认同指 9004，但可被不同机制覆盖，不应混为一谈。
- **与 localTool 18080 严格隔离**：`R` 用于网关 `/v1/tasks/`，`H.uploadFile`/`ii` 落盘走 localTool 18080（`Hr`/`vv` 写死常量）。门4 原质询「混用会 404」已否——`R` 与 18080 不在同一路径，无混用。
- 遗留：`Kn` 的字面量值封在 vendor 1.66MB 打包产物内（ripgrep/node 流式读取均超时）。源码层面已锁定语义为「网关 base URL 常量」，字面值需解包 vendor 才能 100% 钉死，标注为**可接受的非阻断遗留**。
