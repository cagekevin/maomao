## 三、关键数据流

### 3.1 配置加载与降级
```
config.js（静态导出）：
  USE_LOCAL_ENGINE=true → localEngineBase()=http://127.0.0.1:18080
  DEFAULT_ENDPOINT=http://127.0.0.1:9004（网关）
  REMOTE_BASE=http://127.0.0.1:9004（USE_LOCAL_ENGINE=false 时用）
前端 bootstrap：
  ar($n) 解析 API base（localhost/127 规则）→ 选 9004 或 $n
  tr(e) 规范化带 /api 后缀
```

### 3.2 存储三层后端（StorageManager 编排）
```
Q (StorageManager, L1476)
  ├─ wr (localToolBackend, L1297): window.localTool.getKV/saveKV（USE_LOCAL_ENGINE=true 主路径）
  ├─ Nr (localStorageBackend, L1418): localStorage（兜底）
  └─ Pr (localforageBackend, L1446): Sr.default（画布状态持久化）
jr (cachedGet, L1345): 资源键走 Tr/Er 缓存(600s TTL)，否则 wr.get
```

### 3.3 画布状态恢复
```
Cr(e) (L1294) = `${Z.CANVAS_STATE_PREFIX}${e}` → 'canvas-state-v1-{projectId}'
  → Pr.get(Cr(projectId)) 从 localforage 恢复
  → ReactFlow onChange → canvas-state-change 事件 → Pr.set 保存
```
