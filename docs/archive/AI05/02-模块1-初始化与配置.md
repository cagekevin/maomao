## 4. 存储后端装配与画布持久化（实证修正）

### 4.1 引擎优先级（Q.getStatus L1480）
- `localTool`(`wr.isAvailable()`) / `chromeStorage`(`Mr`) / `localStorage`(`Nr`) 三态探测。
- `getConfig`(L1491) → `jr`(cachedGet) → localTool 优先，失败回 localStorage/localforage。
- `setConfig`(L1501) → **只写 localTool**(`wr.set`) + `Ar(e)`(缓存失效)；不写 chromeStorage（去登录魔改后 chrome 态退化）。


- **直接调 `setLocalforage` → `wr.setObject` + `Pr.set`**（L1636-1637），**无 `canvas-state-change` window CustomEvent**（08 节已证伪：该事件名在代码中不存在）。
- 版本化保存 `saveCanvasStateWithVersion`(L1650) 带版本冲突检测（远程版本 > 本地则拒绝覆盖，L1659）。

## 6. 存疑 / 排雷

- **host 硬编码 18080**（TASKS P0 残留）：`Hr`/`vv`/`U_`(var L233) 三处硬编码 `127.0.0.1:18080`。`USE_LOCAL_ENGINE=false` 时文件请求走 9004 无上传路由 → 失败（ARCHITECTURE X3.3）。修复见 TASKS P2「统一 base URL」。
- **中文路径 Latin1 乱码**（TASKS P0 残留）：localTool 写盘中文目录/文件名乱码，config 层无法直接修（模块2）。
- **chromeStorage 探测存在但 setConfig 不写**：去登录魔改后 `Mr` 仅作状态探测，`setConfig` 只落 localTool+localforage（L1501-1504）。
