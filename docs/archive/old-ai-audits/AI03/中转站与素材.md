## 一、这是什么

maomao 把"资源"分成两层，二者通过本地引擎衔接：

1. **中转站（transit）**：网页图片/视频/音频/文本被捕获后，先以原始形态进入中转站暂存（持久化键 `TRANSIT_RESOURCES`），等待"本地化/迁移"落库为正式素材。它是一个**落库前的中转/暂存层**，本身不带分页（背景页只保留最近 5 条）。
2. **素材库（resources / materials）**：正式资源库，由 `src/services/resourceStore.js` 经本地引擎 REST API（`/api/resources*`）增删查改，文件本体落在本地引擎文件存储里。UI 分"素材"与"生成"两个 tab，支持收藏、删除、清空、重扫描、@引用、右键发送、剪映素材库等。

**本地引擎的角色**：`LOCAL_ENGINE`（host `127.0.0.1`，port `18080`，`config.js` L12-18）同时提供 KV 存储（`/api/kv/*`）与文件存储（`/api/files/*`）。`resourceStore.js` 全部 `/api/resources*` 请求都打 `vv`（=`LOCAL_ENGINE.base`，`constants.js` L39）。`USE_LOCAL_ENGINE=true`（`config.js` L36）时走本地引擎；本项目固定本地模式，无独立的"在线模式"分支（远程基址 `REMOTE_BASE` 仅当开关为 false 时生效，当前未启用）。

## 二、存储与键

### 2.1 中转站键 `TRANSIT_RESOURCES`
`src/config/storageKeys.js` L11：
```text
TRANSIT_RESOURCES: `transitResources`
```
真实字符串即 `transitResources`。它出现在三处存储后端：
- `chrome.storage.local`：背景页写入（`background.ts` L92-99），数组，最多 5 条（`.slice(0,5)`），避免 OOM；
- localforage：`Sr.default.getItem(Z.TRANSIT_RESOURCES)`（`App.js` L39949）与导出时被显式排除（`App.js` L41411：`n !== 'transitResources'`）；
- 本地引擎 KV：`Q.getObject(Z.TRANSIT_RESOURCES)`（`App.js` L3429 / L6774 / L39960）。

`src/utils/storage/index.js` L313、L323、L334 也把 `TRANSIT_RESOURCES` 纳入"同步到本地引擎 / 全量同步 / 是否存在本地数据"的判断集合。

数据结构（`background.ts` L55-63 的 `TransitResource` 接口，背景页构造于 L81-89）：
```text
{ id: string,            // Date.now().toString()
  url: string,           // 原始地址（网页直链 / data: / 文本本身）
  type: string,          // image|video|audio|text（info.mediaType 或 selection→'text'）
  timestamp: number,
  pageUrl: string|undefined,
  pageTitle: string|undefined,
  source: string }       // 恒为 'extension'（背景页写入时）
```

### 2.2 素材库端点（全部来自 `resourceStore.js`）
```text
src/services/resourceStore.js
xv → GET  ${vv}/api/resources?{page,pageSize,sortBy,sortDir,search,filters}   // 列表（分页/排序/搜索/筛选）
Sv → POST ${vv}/api/resources/save                            // 保存/入库
Cv → POST ${vv}/api/resources/save  (body 追加 isFavorite)    // 收藏切换（复用 save）
wv → POST ${vv}/api/resources/delete?id=<id>                  // 删除单条
Tv → POST ${vv}/api/resources/clear  body {folder, deleteFiles} // 清空（默认 deleteFiles=true 删文件）
Ev → POST ${vv}/api/resources/rescan                          // 重扫描（清理孤儿文件夹）
```
`yv`（L5-12）规范化单条：把 `isFavorite` 归一成布尔、`timestamp` 归一成数字，并删除空的 `pageUrl/pageTitle/source/folder/name`。`bv`（L13-16）把查询参数拼成 `URLSearchParams`，默认排序 `timestamp DESC`。

### 2.3 本地引擎文件/缩略图/目录端点（`localToolClient.js`、`useLocalTool.js`）
```text
GET  ${Hr}/api/status                         // 引擎可用性探测（Kr / useLocalTool）
POST ${Hr}/api/files/upload   (file|fileUrl + subfolder)  // 上传/本地化（Xr / Zr / useLocalTool.uploadFile）
GET  ${Hr}/api/files/thumbnail?url=&maxDim=&quality=      // 缩略图（ri）
GET  ${Hr}/api/files/open?subfolder=...                   // 打开本地目录（资源面板"打开目录"）
GET  ${Hr}/api/files/open-dir?filepath=...                // 打开文件所在目录（资源菜单"打开目录"）
POST ${Hr}/api/files/mkdir   {folder}                     // 新建文件夹（useLocalTool.createFolder，同步写资源表）
POST ${Hr}/api/files/move    {src,dst}                    // 移动文件到文件夹（useLocalTool.moveFile）
POST ${Hr}/api/kv/set | GET ${Hr}/api/kv/get              // KV 读写（useLocalTool.saveKV/getKV）
```
其中 `vv`（资源 REST）与 `Hr`（文件/状态）均等于 `LOCAL_ENGINE.base`（`constants.js` L13、L39）。

### 2.4 服务端资源表与磁盘语义（`localTool` 引擎，`localTool/src/routes/resources.ts`）
素材的元数据存储于本地引擎的 `resources` SQLite 表，文件实体存于上传目录（按 `tasks/`、`migrated/` 等子目录）。列与枚举（`resources.ts` L135-138 的 `SNAKE_TO_CAMEL`、L12-25 的扩展名映射）：
- 列：`id, url, type, source, folder, name, page_url, page_title, is_favorite, timestamp`（驼峰↔蛇形自动转换）。
- `type`：`image` / `video` / `audio` / `text` / `folder`（文件夹由 rescan 录入，`resources.ts` L66-86）。
- `source`：`extension`（网页捕获）/ `local-tool`（磁盘/引擎文件）/ `generated`（AI 生成）/ `pasted`（剪贴板）。
- 磁盘根目录由 `getUploadDir()` 提供，rescan 入库的 url 会被补全为绝对地址 `http://127.0.0.1:18080/files/...`（`resources.ts` L30-35），否则在 `chrome-extension://` 下会破图。

> **服务端对"删文件"的真实语义（改码前必看）**：
> - `handleResourcesDelete`（`resources.ts` L202-209）：`DELETE FROM resources WHERE id = ?` —— **只删 DB 行，不删磁盘文件**。
> - `handleResourcesClear`（`resources.ts` L211-238）：带 `folder` 时 `DELETE FROM resources WHERE folder = ?`；`deleteFiles=true` 时 `fs.rmSync(folderPath,{recursive:true})` —— **删除整文件夹目录**。不带 `folder` 则清整张表 +（deleteFiles）删所有上传子目录文件。**SQL 未对 `is_favorite` 过滤**，即按 folder 整删会一并删掉收藏项（详见 §五、§十一 的 UI 文案与事实不符提醒）。

## 三、捕获 / 进入中转站（诞生）

捕获入口是扩展**右键菜单**（仅扩展环境）：`background.ts` L7-20 注册 `id:'save-to-transit'`、`title:'发送到资源'`、`contexts:['image','video','audio','selection']`；点击走 `handleSaveToTransit`（`background.ts` L49-53、L65-115）。

- 图片/视频/音频：`info.srcUrl` 作 `url`，`info.mediaType` 作 `type`；
- 文本选择：`info.selectionText` 作 `url`，`type='text'`（`background.ts` L72-76）；
- 构造 `TransitResource`（`source:'extension'`），写入 `chrome.storage.local.transitResources`（保留最近 5 条），并 `chrome.runtime.sendMessage({action:'resourceAdded', resource})` 通知侧边栏（`background.ts` L99-108）。

App 侧监听 `resourceAdded`：`chrome.runtime.onMessage` 处理器（`App.js` L40634 起，全源码仅此一处）把资源 `folder` 置为 `migrated`、`source` 视情况定为 `extension`/`local-tool`；**外部地址（extension 来源）必须先用 `Zr` 下载到本地 `migrated/`，失败整条丢弃**（`App.js` L40386-40396 与 L40643-40653，注释"铁律"）；随后写进本地素材列表状态 `D(e=>...)` 并 `Sv` 入库（`App.js` L40403-40406 / L40660-40662）。空状态 UI 也提示"在网页图片/视频上点击右键 → 选择'发送到资源'"（`App.js` L346）。

### 3.3 其它进入素材库的方式（不止"右键→中转站"）
除网页右键经中转站捕获外，素材库还有以下直接入库通道（均走 `addTransitResource` 或 `Sv`）：
- **AI 生成结果**：生成/绘图/视频等节点完成时调用 `addTransitResource(url, type, 'generated')`（节点侧 `d.addTransitResource`，`App.js` L13208；经 context 下发 `addTransitResource: P`/`gi`，`App.js` L28871、L42442、L42567）。`source='generated'`、`folder='tasks'`（见 §四 末尾 `tasks/` 子目录规则）。图片类节点另有 `onAddImage` / `onPushImagesToImageBox` 注入（`App.js` L34137）。
- **剪贴板粘贴**：`addTransitResource(e, t, 'pasted')`（`App.js` L41305-41325，`gi` 定义）——`source='pasted'`、`folder='migrated'`；URL 已属 `/files/resources/` 则只入内存态，否则 `Sv` 入库。画布侧还有全局 `paste` 监听：图片文件 → `source:'paste'`，`http`/`data:image` 文本 → `source:'paste'`（`App.js` L26740-26761）。
- **local-tool 上传**：经 `localToolClient.js` 的 `Xr`/`Zr` 上传到 `/files/...`，再 `Sv` 入库，`source='local-tool'`（`App.js` L40383-40384）。这类资源后续迁移时因 `source==='local-tool'` 被跳过（见 §四）。

进入中转站时的数据结构（网页捕获，`background.ts` L55-63，见 §2.1）：`{id, url, type, timestamp, pageUrl, pageTitle, source:'extension'}`。

## 四、中转站 → 素材的迁移

迁移（本地化）把中转站里的外部地址转成本地引擎内部地址（`/files/migrated/...`）并入库。源码中共有 **4 条本地化落库路径**（均跳过 `local-tool`、外部地址必走 `Zr`）：

1. **启动迁移 A**（≈`App.js` L3428-3447）：`useEffect` 内 `Q.getObject(Z.TRANSIT_RESOURCES)` 取出中转资源，逐条 `if (resource.source === 'local-tool') continue;`（**跳过 local-tool 来源**），其余 `await Zr(resource.url, {subfolder:'migrated'})`，拿到 `localized.url` 后 `await Sv({...resource, url, folder:'migrated', id:String(resource.id)})`（`App.js` L3433-3439，`Zr` 在 L3435）。
2. **启动迁移 B**（≈`App.js` L6774-6784）：结构与 A 一致（`Zr` 在 L6780），同样跳过 `local-tool`。
3. **首启动"播种"（幂等）**（≈`App.js` L39947-40222）：先 `Q.getConfig(kv)` 判断是否已经播种，未播过才执行；合并 localTool KV 与 localforage 两份 `TRANSIT_RESOURCES`，过滤 `source !== 'local-tool'`，逐条 `Zr` 本地化 + `Sv` 入库（`Zr` 在 L40208），成功后在 `Q.setConfig(kv, ...)` 打标记（`App.js` L40219），日志"[App] 已播种 N 条历史资源到 SQLite"。**幂等保护**避免重复入库。
4. **实时捕获落库**（`resourceAdded` 处理器）：`chrome.runtime.onMessage` 处理器（`App.js` L40634 起，全源码仅此一处；`Zr` 本地化在 L40647），逻辑见 §三 —— 这是网页/扩展开捕获后**实时**入库的主通道，而非仅启动期。

统一规则：
- `local-tool` 来源**不重新下载**（它本就是本地路径），只在入库时改 `source`；
- `extension` 来源（网页直链 / data: / http）**先下载到 `migrated/`**，失败则丢弃；
- 本地化后地址补全为绝对路径：``localized.url.startsWith('http') ? localized.url : `${Hr}${localized.url}` ``（`App.js` L3437 / L39961 / L40399）。

> 注意：素材入库后落在 `migrated/` 子目录（"素材" tab）或 `tasks/` 子目录（"生成" tab），由当前 tab 决定（`App.js` L295、L371、L404 等：`u === 'generated' ? 'tasks/' : 'migrated/'`）。

## 五、素材库的增删查改

全部经 `resourceStore.js`，函数语义见 §2.2：

- **保存/入库** `Sv`：`POST /api/resources/save`，body 为规范化后的资源对象；
- **列表** `xv`：分页（默认 `pageSize` 由调用方给，如 `pageSize:300` 见 `App.js` L39901）、排序（默认 `timestamp DESC`）、搜索（`search`）、筛选（`filters` 序列化为 JSON）；返回 `{items,total,page,pageSize,totalPages}`；
- **收藏** `Cv`：复用 `Sv`，仅追加 `isFavorite` 字段（`resourceStore.js` L47-52）；
- **删除** `wv`：`POST /api/resources/delete?id=<id>`；**服务端仅删 DB 行、不删磁盘文件**（`localTool/src/routes/resources.ts` L202-209）。
- **清空** `Tv`：`POST /api/resources/clear`，body `{folder, deleteFiles}`；未连接本地引擎时前端提示"请先连接本地引擎"（`App.js` L307）；清空当前页签未收藏资源时带确认框"本地文件将被删除"（`App.js` L41017）。⚠️ **事实与文案不符提醒**：确认框文案称"收藏的资源保留"，但服务端 `handleResourcesClear` 按 `folder` 整删（`resources.ts` L215-220）且 SQL **未过滤 `is_favorite`**，该 folder 下的收藏项会被一并物理删除（目录级 `deleteFiles=true`）。改码前务必对齐 UI 文案与服务端语义。
- **重扫描** `Ev`：`POST /api/resources/rescan`，返回清理掉的孤儿文件夹数 `count`。资源面板每次打开（`active` 变 true）会自动 `rescan` 一次（`App.js` L169-172），手动刷新按钮同理（`App.js` L305-310，注释"清理已删孤儿"）。生成完成（`i==='completed'`）也会触发 `Ev` 统一入库（`App.js` L28965）。

## 七、拖拽与 MIME

- 文件夹卡片 `onDragOver` 仅对 `e.type==='folder'` 调 `preventDefault`（`App.js` L394-396）；`onDrop` 时从 `t.dataTransfer.getData('text/plain')` 取资源 id（`App.js` L400），再在 `transitResources` 中按 id 找资源做移动（`App.js` L401-406）。即**素材拖拽用 `text/plain` 承载资源 id**，而非专用 `application/x-*` MIME。
- 图片卡片拖拽时除 `text/plain`(资源 id) 外，还会写 `text/html` = `<img src="...">` 作为预览增强（`App.js` L553-555）；资源拖入**画布节点**时不走移动逻辑，而是派发 `canvas-add-resource-request` 自定义事件（`Zn`，`constants.js` L5；`App.js` L177），由图片框节点 / `rhWebappNode` 等经 `addTransitResource` 消费（`App.js` L33177、L42184）。
- 模板类拖拽才用专用 MIME `application/x-maomao-template`（`constants.js` L33，`ig`），与素材无关；其余 `application/x-maomao-*`（`x-maomao-layer`/`x-maomao-puzzle`/`x-mutiwindow-task`）服务于图层/拼图/多窗口任务，亦与素材拖拽无关。

## 九、与云同步的关系（重点核查）

**结论：素材库与中转站均不随 GAS 跨设备同步，仅存于本地引擎。**

证据链：
1. `syncToCloud`（`App.js` L40818-40867）推送键集合为 `['app_settings','api_configs','projects','presetPrompts','customNodeTemplates','modelSchedules','cloud_storage_config','local_templates']`（`App.js` L40822），**不含 `transitResources`，也不含任何素材资源键**。
2. 本地导出 `Ri`（`App.js` L41399-41437）遍历 localforage 时显式排除：`if (!n.startsWith('img_') && n !== 'transitResources')`（`App.js` L41411）——中转站数组明确被排除出备份 JSON。
3. 反向佐证：本地引擎同步 `syncAllToLocalTool`（`src/utils/storage/index.js` L322-330）**包含** `TRANSIT_RESOURCES` 与 `TRANSIT_GRID_COLS`，说明中转站数据会被"同步到本地引擎文件"，但**只到本地引擎这一层，不到 GAS 云端**。
4. **`local_templates` 推送时剥离二进制字段**：`syncToCloud` 推 `local_templates` 前会清洗模板——`delete clean.coverUrl`，并遍历 `graphData.nodes` 移除节点内 `imageUrl`/`thumbnailUrl`/`videoUrl` 等大体积引用字段（`src/App.js` L41063 起，`_getLocalTemplates().then(arr => arr.map(...))`）。这进一步印证"GAS 刻意只同步轻量配置 JSON，素材/中转站（文件型）不在其列"。

原因：素材/中转站是文件型二进制资源，本地引擎文件存储即其真身；GAS 仅同步轻量配置 JSON，刻意不搬运大体积媒体文件。

## 十一、改码前必查

1. **本地引擎端点一致性**：`resourceStore.js` 用 `vv`（`/api/resources*`）、`localToolClient.js`/`useLocalTool.js` 用 `Hr`（`/api/files*`、`/api/kv*`），二者都等于 `LOCAL_ENGINE.base`。改端口只动 `config.js` 的 `LOCAL_ENGINE`（host/port，`config.js` L12-18），不要在两处硬编码。
2. **`TRANSIT_RESOURCES` 键一致性**：所有读写必须走 `Z.TRANSIT_RESOURCES`（`storageKeys.js`），切勿在 `background.ts`/`App.js`/`storage/index.js` 各写死字符串；导出排除逻辑（`App.js:41411`）与本地同步包含逻辑（`storage/index.js:323`）要保持"不同步 GAS、可同步本地引擎"的对齐。
3. **迁移逻辑**：新增来源类型时，务必在 L3433 / L6777 / L39955 三处以及 `resourceAdded` 处理器（L40380）维持"跳过 `local-tool`、外部地址必走 `Zr`"的"铁律"（`App.js` L40386 注释），否则会重复下载或丢数据。
4. **rescan 孤儿清理**：`Ev` 前端触发 + 服务端 `handleResourcesRescan`（`resources.ts:37-133`）会真实扫描 upload 目录、删除"磁盘已不存在的 `local-tool` 记录"；不要在未确认本地文件已无用前关闭 rescan 或改其语义。
5. **删除/清空对文件的实际影响（重点）**：
   - `wv`/delete **只删 DB 行，不删磁盘文件**（`resources.ts:202-209`）。
   - `Tv`/clear 带 `deleteFiles=true` 会**删除整文件夹目录**（`resources.ts:215-220`），且 SQL **未过滤 `is_favorite`**——UI 确认框"收藏的资源保留"（`App.js:41017`）与实际按 folder 整删（含收藏）**不符**，改码前务必对齐文案与服务端语义，避免误删用户收藏。
6. **GAS 键集**：若未来要让素材跨设备，需在 `syncToCloud`（`App.js:40822`）加入对应键，并同步解除 `Ri` 导出排除（`App.js:41411`）——当前刻意不含，改动需同步评估云端体积与二进制字段 strip。
