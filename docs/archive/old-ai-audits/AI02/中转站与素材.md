## 一、这是什么

maomao 里"资源"分两层，二者通过本地引擎衔接：

1. **中转站（transit）**：网页图片/视频/音频/文本被捕获后，先以原始形态进入中转站暂存（持久化键 `TRANSIT_RESOURCES` = `transitResources`，存于浏览器 `chrome.storage.local`），作为落库前的暂存/中转层。背景页只保留最近 5 条，无分页。
2. **素材库（resources / materials）**：正式资源库，由 `src/services/resourceStore.js` 经本地引擎 REST API（`/api/resources*`）增删查改；元数据落**本地引擎 SQLite 表 `resources`**，文件本体落在本地引擎数据目录的 `tasks/`、`migrated/` 等子目录。UI 分"生成 / 素材"两个 tab（对应 `tasks/` 与 `migrated/`），支持收藏、删除、清空、重扫描、@引用、右键发送、剪映素材库等。

**两者关系**：中转站是素材"落库前的中转/暂存层"。实时捕获（网页右键、剪贴板、AI 生成）经中转站 `chrome.storage.local` 写入，再由 App 侧 `resourceAdded` 监听（或启动期迁移）把外部地址本地化后 `Sv` 入库；`source === 'local-tool'` 的资源已在本地引擎，跳过本地化。

**本地引擎角色**：`LOCAL_ENGINE`（host `127.0.0.1`，port `18080`，`config.js` L12-18）同时提供 KV 存储（`/api/kv/*`）、文件存储（`/api/files/*`）与 SQLite（`resources` 表）。`resourceStore.js` 全部 `/api/resources*` 请求打 `vv`（=`LOCAL_ENGINE.base`，`constants.js` L39）；`USE_LOCAL_ENGINE=true`（`config.js` L36）时固定走本地引擎，无独立"在线模式"素材通路（`REMOTE_BASE` 仅在开关 false 时生效，当前未启用）。

## 二、存储与键

### 2.1 中转站持久化键 `TRANSIT_RESOURCES`
`src/config/storageKeys.js` L11：
```text
TRANSIT_RESOURCES: `transitResources`
TRANSIT_GRID_COLS: `transit_grid_cols`   // 资源面板显示列数偏好
```
真实字符串即 `transitResources`，出现在**三个后端**：
- `chrome.storage.local`：扩展背景页写入（`background.ts` L92-99），数组，最多 5 条（`.slice(0,5)`）；
- localforage：`Sr.default.getItem(Z.TRANSIT_RESOURCES)`（`App.js` L40198）；
- 本地引擎 KV：`Q.getObject(Z.TRANSIT_RESOURCES)`（`App.js` L3429 / L6774 / L40197）。

`TRANSIT_RESOURCES` 还被纳入"同步到本地引擎"集合：`src/utils/storage/index.js` 的 `syncToLocalTool`（L305-313，对 `TRANSIT_RESOURCES` 与 `canvas-state-v1-*` 特殊处理）、`syncAllToLocalTool`（L322-323，含 `TRANSIT_RESOURCES` 与 `TRANSIT_GRID_COLS`）、`hasLocalToolData`（L332-334）。

中转站单项数据结构（`background.ts` L55-63 的 `TransitResource` 接口）：
```text
{ id: string,            // Date.now().toString()
  url: string,           // 原始地址（网页直链 / data: / 文本本身）
  type: string,          // image|video|audio|text
  timestamp: number,
  pageUrl: string|undefined,
  pageTitle: string|undefined,
  source: string }       // 背景页写入时为 'extension'
```

### 2.2 素材库（本地引擎 SQLite + 文件）
- 表 `resources` 列（snake↔camel 映射 `resources.ts` L135-140）：`id, url, type, source, folder, name, page_url, page_title, is_favorite, timestamp`。
- `type` 取值（按扩展名映射，`resources.ts` L12-25）：`image` / `video` / `audio` / `text`；rescan 时文件夹记为 `folder`。
- `source` 取值：`extension`（网页捕获）/ `local-tool`（本地引擎/磁盘文件）/ `generated`（AI 生成）/ `pasted`（剪贴板）等。

### 2.3 素材库 REST 端点（前端 `resourceStore.js` ↔ 服务端 `localTool/src/routes/resources.ts`）
| 端点 | 方法 | 语义 | 前端封装 |
|---|---|---|---|
| `/api/resources` | GET | 列表（分页/排序/搜索/筛选） | `xv` (`resourceStore.js:17-33`) |
| `/api/resources/save` | POST | upsert 单条 | `Sv` (`resourceStore.js:34-46`) |
| `/api/resources/batch-save` | POST | 批量 upsert（**服务端已实现** `resources.ts:190-200`；前端当前逐条走 `Sv`） | — |
| `/api/resources/delete?id=` | POST | 删除单条（**仅删 DB 行，不删磁盘文件**） | `wv` (`resourceStore.js:53-61`) |
| `/api/resources/clear` | POST | 清空某 folder（`{folder, deleteFiles}`；`deleteFiles=true` 删整个文件夹目录） | `Tv` (`resourceStore.js:62-78`) |
| `/api/resources/rescan` | POST | 扫描 upload 目录入库 + 孤儿清理 | `Ev` (`resourceStore.js:79-88`) |

> 端点与 `storageKeys` 一致性：前端 `vv`=本地引擎 base，所有 `/api/resources*` 都用 `vv` 拼接（`resourceStore.js:23,36,55,64,81`），与 `resources.ts` 路由一一对应，无偏移。

## 三、捕获 / 进入中转站（诞生）

### 3.1 网页右键"发送到资源"（扩展背景页）
`background.ts` L7-20 注册右键菜单 `id:'save-to-transit'`、`title:'发送到资源'`、`contexts:['image','video','audio','selection']`；点击走 `handleSaveToTransit`（`background.ts` L49-53、L65-115）：
- 图片/视频/音频：`info.srcUrl` → `url`，`info.mediaType` → `type`；
- 文本选择：`info.selectionText` → `url`，`type='text'`（`background.ts` L72-76`）；
- 构造 `TransitResource`（`source:'extension'`），写入 `chrome.storage.local.transitResources`（保留最近 5 条），并 `chrome.runtime.sendMessage({action:'resourceAdded', resource})` 通知侧边栏（`background.ts` L99-108`）。
- 捕获时**直接用原始 URL，不再经本地引擎上传**（`background.ts` L80 注释）。

### 3.2 App 实时接收并落库（`resourceAdded` 监听，主通道）
App 监听 `chrome.runtime.onMessage`（`App.js` L40635-40666）：收到 `resourceAdded` 后 `folder='migrated'`、`source = local-tool?'local-tool':'extension'`；**"铁律"注释（L40643）**：素材入库前外部地址必须转成内部地址 `/files/migrated/...`——`local-tool` 来源本就是本地路径直接入库；`extension` 来源（网页直链 / data: / http）先 `Zr` 下载到 `migrated/`，**失败则整条丢弃**（`download_failed`）；随后写入本地素材列表状态并 `Sv` 入库（`App.js` L40660-40663`，并切到"素材"tab）。

### 3.3 其它进入方式（均最终经 `Sv` / `addTransitResource` 入库）

## 四、中转站 → 素材的迁移（本地化）

迁移（本地化）把中转站里的外部地址转成本地引擎内部地址（`/files/migrated/...`）并入库。`Zr`（= `uploadRemoteUrlToLocalTool`，`localToolClient.js` L67-86）把远程 URL 经 `/api/files/upload` 写入 `subfolder=migrated` → 得到 `/files/migrated/...`。**源码中共有 4 处 `Zr` 本地化站点**，语义一致（跳过 `local-tool`，其余本地化后 `Sv({...resource, url, folder:'migrated'})`）：

1. **挂载迁移 ①** ≈ `src/App.js:3429-3447`（`if (resource.source==='local-tool') continue;` L3433）；
2. **挂载迁移 ②** ≈ `src/App.js:6774-6792`（结构同上）；
3. **首启动"播种"** ≈ `src/App.js:40196-40219`：受 `kv`（`resources_seeded_to_sqlite`）标记保护，合并 localTool KV 与 localforage 两份 `TRANSIT_RESOURCES`、过滤 `source !== 'local-tool'`，逐条 `Zr`+`Sv`，日志"已播种 N 条历史资源到 SQLite"；
4. **实时监听** ≈ `src/App.js:40635-40666`（见 §3.2，捕获即落库）。

统一规则：
- `local-tool` 来源**不重新下载**（本就是本地路径），仅改 `source`；
- `extension` 来源（网页直链 / data: / http）**先下载到 `migrated/`**，失败则丢弃；
- 本地化后地址补全为绝对路径：``localized.url.startsWith('http') ? localized.url : `${Hr}${localized.url}` ``（`App.js` L3437 / L40210 / L40656）。

**服务端文件影响（重要，改码前必查）**：

## 七、拖拽与 MIME

- **资源卡片拖拽**：网格项 `draggable`，拖拽时写入 `dataTransfer.setData('text/plain', e.id)`（图片额外带 `text/html`，`App.js` L477-481、L553-555）。即**素材拖拽统一用 `text/plain`（携带资源 id）**，**未用** `application/x-*` 专属 MIME。项目中已有的 `application/x-maomao-*`（`x-maomao-layer` / `x-maomao-puzzle` / `x-maomao-template` / `x-mutiwindow-task`）分别服务于图层、拼图、画布模板、多窗口任务，**与素材拖拽无关**（`constants.js` L33`）。
- **拖入画布节点**：资源面板拖拽时派发 `canvas-add-resource-request` 自定义事件（`Zn`，`src/config/constants.js:5`；`App.js` L177 `window.dispatchEvent(new CustomEvent(Zn,{detail:e}))`），由画布节点（如 `imageBoxNode`、`rhWebappNode` 经 `addTransitResource`，`App.js` L33177、L42442` 等）消费。
- **文件夹内移动**：文件夹卡片 `onDragOver` 仅对 `e.type==='folder'` 阻止默认（`App.js` L394-396`），`onDrop` 从 `text/plain` 取 id 在 `transitResources` 中找资源做移动（`L398-407`，仅 local-tool 可移动）。

## 九、与云同步的关系（重点核查）

**结论：素材库与中转站均不随 GAS 跨设备同步，仅存于本地引擎（SQLite `resources` 表 + 磁盘目录）；但中转站数据会被"同步到本地引擎"（多存储对齐），不到 GAS 云端。**

证据链：
1. **GAS 推送键集合不含 transit/resources**：`syncToCloud`（`App.js` L40818 起）推送键为 `['app_settings','api_configs','projects','presetPrompts','customNodeTemplates','modelSchedules','cloud_storage_config','local_templates']`（`App.js` L40822`）——**不含 `transitResources`，也不含任何素材资源键**。（注：早期个别文档把键写成含 `users`/`membership`，与当前源码不符，以本文 grep 为准。）
2. **本地导出显式排除 transit**：导出遍历 localforage 时 `if (!n.startsWith('img_') && n !== 'transitResources')`（`App.js` L41431`）——中转站数组明确被排除出备份 JSON。
3. **反向佐证（仅到本地引擎）**：`syncAllToLocalTool`（`src/utils/storage/index.js` L322-323）**包含** `Z.TRANSIT_RESOURCES` 与 `Z.TRANSIT_GRID_COLS`，`hasLocalToolData`（L332-334）也含 `TRANSIT_RESOURCES`——说明中转站数据会在多存储引擎（localTool ↔ chrome.storage ↔ localforage）间对齐，但**只到本地引擎这一层，不到 GAS 云端**。
4. **`local_templates` 推送时剥离二进制字段**：`syncToCloud` 推 `local_templates` 前会清洗模板——`delete clean.coverUrl`，并遍历 `graphData.nodes` 移除节点内 `imageUrl`/`thumbnailUrl`/`videoUrl` 等大体积引用字段（`src/App.js` L41063 起，`_getLocalTemplates().then(arr => arr.map(...))`）。这进一步印证"GAS 刻意只同步轻量配置 JSON，素材/中转站（文件型）不在其列"。

**原因**：素材/中转站是文件型资源，本地引擎文件存储即其真身；GAS 仅同步轻量配置 JSON，刻意不搬运大体积媒体文件。

## 十一、改码前必查

1. **本地引擎端点一致性**：`resourceStore.js` 用 `vv`（`/api/resources*`）、`localToolClient.js`/`useLocalTool.js` 用 `Hr`（`/api/files*`、`/api/kv*`），二者都 = `LOCAL_ENGINE.base`。改端口只动 `config.js` 的 `LOCAL_ENGINE`（host/port），不要在调用处硬编码（文件头注释强调"统一使用 vv，不改 9004"）。
2. **`TRANSIT_RESOURCES` 键一致性**：真实字符串 `"transitResources"` 在 `storageKeys.js:11`、`background.ts:99`、`App.js:3429/6774/40197`、本地引擎 KV、localforage 多处引用；改名须全量替换。导出排除（`App.js:41431`）与本地引擎同步包含（`storage/index.js:323`）要保持"不同步 GAS、可同步本地引擎"的对齐。
3. **迁移/本地化铁律**：新增来源类型时，务必在 L3433 / L6777 / L40196-40204 / `resourceAdded` 处理器（L40637-40646）维持"跳过 `local-tool`、外部地址必走 `Zr`"的"铁律"（`App.js:40643` 注释），否则重复下载或破图/丢数据。首启动播种受 `kv`（`resources_seeded_to_sqlite`）标记保护，改动前确认不会重复入库或漏播。
4. **rescan 孤儿清理**：`resources.ts:120-130` 会在 rescan 时删除磁盘已失的 `local-tool` 记录；改删除逻辑前确认不会误删用户手动放入的文件。
5. **delete 与 clear 对文件的实际影响（重点）**：`wv`/delete **仅删 DB 行不删磁盘文件**（`resources.ts:202-209`）；`Tv`/clear 带 `deleteFiles=true` 会**删除整个 folder 目录**（`resources.ts:215-219`），且 SQL **未按 `is_favorite` 过滤**。`handleClearResources` 的确认文案"收藏的资源保留"（`App.js:41026`）与实际服务端"按 folder 整删（含收藏）"**不符**——改动清空前务必对齐文案与语义。
6. **GAS 同步范围**：若未来要让素材跨设备，须在 `syncToCloud`（`App.js:40822`）显式加键，并同步解除导出排除（`App.js:41431`）；当前刻意排除，原因见 §9（体积/二进制字段）。
7. **门控条件**：任何新增资源操作需检查 `S.status.isConnected`（本地引擎）与插件环境（`chrome.runtime`），避免无连接时空操作。
8. **拖拽 MIME**：素材卡片用 `text/plain`(id) 而非 `application/x-*`；接收端若改 MIME 需同步两端；拖入画布走 `canvas-add-resource-request`（`Zn`）事件，消费方为 `imageBoxNode`/`rhWebappNode` 的 `addTransitResource`。
