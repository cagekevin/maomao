## 1. 这是什么：模型调度机制的定义，与"普通模型选择"的关系

maomao 是基于 React + Vite 的前端画布 / AI 工作流应用。每个生成类节点（绘图/文生文/视频/sd2Video/折扣视频 等）都要选一个"模型"去调用 API。项目里"模型选择"有两种形态：

1. **普通模型选择**：节点的 `selectedModel` 是一个具体模型名，来自该节点类型对应的配置列表（换行分隔的模型名字符串），如 `drawingModel` / `textModel` / `videoModel` / `sd2VideoModel` / `discountVideoModel`。
2. **模型调度（schedule）**：`selectedModel` 可为 `schedule:<id>` 形式的引用，指向一条"调度配置"。调度配置把多个模型按 steps（每步 `model` + `retries`）排成一条**失败重试 / 故障转移序列**，运行期按顺序逐个尝试，某个模型失败就自动切下一个。

一句话：**调度 = 失败重试/故障转移序列**。普通选择是"钉死一个模型"，调度是"排好一串候选模型，谁先成功用谁"。调度配置本身不调用任何 API，它只决定"尝试哪些模型及顺序"；真正的调用由运行期生成函数（`sr/cr/ur/dr/pr`）逐个发起。

适用节点类型（代码层面，分两层，详见 §9）：
- **选择器 UI 已暴露调度选择**：绘图(image)、文生文(text)、AI 空间文本(director, text)。
- **运行期已接入调度故障转移**：上述三类 + 视频(video)、sd2Video、折扣视频(discountVideo) 共 5 类生成函数。

### 2.4 steps 校验规则（核心刚性约束，`aa`/`ia` 实现）
```text
src/services/modelSchedules.js:7-27
  function ia(e) {            // 重试数归一化 1–3
    let t = Math.round(Number(e) || 1);
    return t < 1 ? 1 : t > 3 ? 3 : t;
  }
  function aa(e) {            // steps 校验：最多 5 步、总 retries ≤ 10
    if (!Array.isArray(e)) return [];
    let t = [], n = 0;
    for (let r of e) {
      if (t.length >= 5) break;                       // 最多 5 个模型
      let e = typeof r?.model == `string` ? r.model.trim() : ``;
      if (!e) continue;
      let i = ia(r?.retries);
      if (n + i > 10 && (i = 10 - n), i <= 0) break;  // 总重试超过 10 则截断
      t.push({ model: e, retries: i }), n += i;
    }
    return t;
  }
```
- 单步 `retries` 归一到 1–3（`ia`）。
- 模型步数最多 5 个；累计 `retries` 超过 10 时，剩余步被丢弃（`aa` 第 20 行 `i = 10 - n` 后再判 `i<=0` 则 break）。
- 空 model / 非字符串被 `continue` 跳过。
- UI 层也做同样的上限保护：加入第 6 个模型或累计重试达 10 时禁用“加入”按钮（`src/App.js:37744`）；每步 `retries` 的 +/− 步进按钮在 1/3 处禁用（`src/App.js:37829`、`37842`）；上限提示文案固定为（`src/App.js:37861`）。

## 6. 与模型权益（modelEntitlements）的衔接

调度**只决定"尝试哪些模型及顺序"**；最终每个模型能否真正发起调用，仍受 `src/services/modelEntitlements.js` 的权益门控。两者是"顺序编排 + 单模型可用性判定"的叠加关系：

- `ki(model)` 返回 `{ access: 'allowed' | 'denied' | 'quota_exceeded', ... }`（`modelEntitlements.js:26-48`）。
- `Qi(model)` / `ea(model)` 用于 UI 禁用与提示（`modelEntitlements.js:226-254`）；被禁用的模型在普通选择列表里就不可点。
- 在调度运行期 `ar` 的循环里，每个 `await n(模型)` 回调（如 `Jn`/`tr`/`er`）内部仍会走权益校验：若某候选模型因权益不足/额度耗尽而失败，`ir(e)` 读到错误态，循环照常切到下一个候选——即**权益失败也会触发故障转移**，但最终能否成功仍取决于队列里是否存在一个"权益允许"的模型。

因此整体语义：调度负责"排好候选顺序"，entitlements 负责"逐个判定能不能用"；二者互不影响字段，但运行期共同决定一次生成能否成功。

## 8. 本地模式

调度是**纯本地配置**（仅存于 `modelSchedules` 键，随 GAS 同步但本质是本地草稿）。本地模式（无官方账号 / `LOCAL_MODE_ALLOW_ALL`）下：

- 权益层 `ki` 在本地模式直接返回 `{ access: 'allowed' }`（`modelEntitlements.js:26-32`、`Mi` 置 `LOCAL_MODE_ALLOW_ALL_MODELS = true` at `:100-102`）。
- 因此调度序列里每个模型在本地模式都会被放行，不受官方账号权益限制；调度机制照常按序尝试、失败转移。
- 即：本地模式 = 用模型名直接调用 + entitlements 本地全放行，调度在此之上额外提供"多模型故障转移"能力，二者正交、可叠加。

## 10. 改码前必查

1. **键名三处一致**：`src/config/storageKeys.js:11`(`MODEL_SCHEDULES`)、`src/config/constants.js:15`(`ta`)、`src/services/modelSchedules.js:5`(`na` 事件名) 必须都等于 `modelSchedules` / `modelSchedules:change`，否则读不到、推送/合并错位。
2. **`schedule:` 前缀编解码对称**：`ga` 拼、`_a` 切（固定 slice(9)，前缀长度 9）。改前缀字符串必须同步改两处及 `ra` 常量（`modelSchedules.js:6`）。
3. **retries / steps 上限**：`aa` 限制最多 5 步、总 retries ≤ 10；`va` 扁平化再卡一次 ≤ 10（`modelSchedules.js:20,117`）。UI 禁用阈值（`App.js:37744,37776`）与这两个常量必须保持一致，否则会"能加但运行期被截断"或反之。
4. **`va` 上限 10 与 `aa` 总 retries ≤ 10 必须同源**：二者都基于 `retries` 累加，改其一需同步复核。
5. **GAS 合并按 id 去重、云端覆盖本地**：`ba`（`modelSchedules.js:125-130`）顺序为本地先、云端后覆盖。若需"本地优先"须改此处；拉取分支 `App.js:41129` 与推送分支 `App.js:41063` 必须都包含 `modelSchedules`，漏一个就不同步。
6. **自定义事件 + storage 事件双订阅的清理**：`ha` 返回的清理函数必须被组件 `useEffect` 返回调用（`App.js:2116/3415/23835`），面板监听 `App.js:40886-40888` 同理，否则切节点/切页会内存泄漏或串数据。
7. **`or` 只取 enabled 调度**：运行期故障转移仅在 `enabled===true` 时触发（`App.js:32357`）；禁用后节点回落到"按原始字符串当模型名直接调用"，不要误以为会报错。
8. **category 仅 `text|image|video`**：`oa` 把非法 category 落 `image`（`modelSchedules.js:31`）；UI 三选一标签来自 `v_`，新增类目（如 `discountVideo`）需同步改 `oa` 白名单与 `S_` 的 category 选项，否则调度不会出现在对应节点里。
9. **视频/sd2Video/折扣视频 选择器未接调度 UI**：当前只有 `sr/cr/ur/dr/pr` 运行期支持；若要在这些节点 UI 暴露徽标/配置入口，需补 `la().filter(category===...)` 订阅、`ga()` 构造、`mutiwindow-open-schedule-settings` 配置按钮三件套（参考绘图节点 `App.js:2115/2890/2954/2959`）。
