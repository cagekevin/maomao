# AI03 · 审计校验方案（门3 机器校验 · 防幻觉）

> 对应 TASKS 工作流「门3 机器校验」：模块文档写完后，跑脚本正则抽取 `file:line` 引用，回源码 grep 确认存在/行为字符串命中，输出 `校验报告.md`（逐条 ✅/❌）。
> **不依赖 AI 判断**，专治「基于猜测名瞎编」的幻觉。
> 本方案脚本**只放在 `docs/AI03/`**，不写 `scripts/`（遵守「文件只放 AI03」约束）。

---

## 1. 校验原理

1. **抽取**：扫描 `docs/AI03/模块*.md`，正则匹配两类引用：
   - 行号锚点：`App.js:L\d+` / `localTool/src/...:L\d+` / `apimart-gateway/...:L\d+`
   - 符号/字符串：被反引号包裹的 `函数名()`、`'字符串字面量'`、`/api/...` 路由
2. **核对**：对每个 `file:line`，读源码对应行（±3 行窗口），确认该行**确实存在且含相关符号/字符串**（如写 `Ev()=rescan`，则 L42883 附近应有 `async function Ev()` 与 `/api/resources/rescan`）。
3. **报告**：逐条输出 ✅（命中）/ ❌（未命中或行号漂移），❌ 项列出「期望符号」与「该行实际内容」，供打回重写。

---

## 2. 校验脚本（`docs/AI03/audit-check.cjs`）

用法（PowerShell / 终端，在 `maomao/` 根目录）：
```bash
node docs/AI03/audit-check.cjs
```
脚本自动：
- 遍历 `docs/AI03/模块*.md` + `映射表补全记录.md`
- 对每个 `file:Lnnnn` 引用，打开对应源码文件，取 Lnnnn±3 行
- 解析该引用附带的「期望符号」（从本行附近的反引号 token 推断，或模块文档里同一段落的函数名/字符串）
- 命中则 ✅，否则 ❌
- 输出 `docs/AI03/校验报告.md`

> 说明：完全自动推断「期望符号」较难，脚本采取保守策略——**只验证行号存在 + 该行非空 + 含该段落出现过的任一反引号 token**；对关键断言（如 `Ev=rescan`）额外要求命中 `/api/resources/rescan` 这类「行为字符串」。模型行为字符串清单见 §3。

## 3. 行为字符串白名单（强校验项）

这些字符串是「逻辑坐实」的硬证据，文档若断言某函数干某事，必须在其行号窗口内命中对应字符串：

| 断言 | 必须命中字符串 | 用于核对 |
|------|--------------|---------|
| `Ev`=rescan | `/api/resources/rescan` | 映射表补全 L42883 |
| `we`=rescan 调度 | `await Ev()` 或 `rescanThrottledSync` | L43015 |
| `Sv`=资源入库 | `/api/resources/save` | L42838 |
| `wv`=资源删除 | `/api/resources/delete` | L42857 |
| `Xr(L1802)`=上传 | `/api/files/upload` | L1802 |
| `Zr(L1827)`=按URL上传 | `/api/files/upload` + `fileUrl` | L1827 |
| `ii`=uploadFile | `uploadToLocalTool` 或 `Xr(` | L1888 |
| `Jn(L32490)`=生图 | `/v1/images/generations` 或 `task_id` | L32490 |
| `Oa`=去登录 | `auth_token` / `localStorage.getItem` | L3511 |

## 5. 防污染规则（沿用 TASKS 工作流）

- 模块文档草稿文件名带 `_DRAFT`（如 `模块2-本地数据层与Rescan_DRAFT.md`）；校验全绿后去掉 `_DRAFT` 并入终稿。
- 后续模块审计**不许把前序模块 doc 当真理**，只能当线索，最终仍回代码 + grep 核对。
- 映射表是唯一可信锚层；文档是派生，错了改文档，别改代码迁就文档（红线 §3.1）。
