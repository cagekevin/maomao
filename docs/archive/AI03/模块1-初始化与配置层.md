## 1. 运转图景

模块1 负责**画布启动的引导链** + **全局配置开关**。它是所有数据流的"地基"：
- 启动顺序：bootstrap → lazy 加载 `App.js` → 建立 `Q`(StorageManager) → 检测 localTool 连接(`Hr`) → 读 config → `syncAllToLocalTool` → 恢复画布 → 注册监听。
- 配置层 `config.js` 集中所有端点/开关，**前端只 import 它，不硬编码地址**（除残留短名 `U_`/`vv`/`ot` 外，见 §4）。
- 鉴权 `Oa()` 在本地模式下永远返回本地 token，使所有 `getCurrentToken` 拦截放行。

## 3. 关键数据流

### 3.1 配置加载与双服务桥接（边界契约）
- `config.js` 导出 `USE_LOCAL_ENGINE=true` → `localEngineBase()` 返回 `http://127.0.0.1:18080`(localTool)。
- `Hr`(L1732) = `localEngineBase()`，用于**状态检测/文件上传**；`vv` = `'http://127.0.0.1:18080'`，用于**数据请求(/api/resources)**。两者值相同（均为 18080），但来源不同：一个走 config 函数，一个硬编码字符串。
- 风险点（TASKS P0 / X3.3）：`vv`/`U_`/`ot` 是**硬编码残留短名**，改 host/port 或跨设备需同步改多处；理想是统一走 `config.js`。详见模块2 §4 与交叉流审计。

### 3.2 启动初始化链
```
main.tsx → React.lazy(import App.js)
  → Oa()(L3543) 永远返回 local token
  → Q(StorageManager) 初始化后端 wr(localToolBackend)/Mr(chromeStorage)/Nr/Pr(localforage)
  → Hr=localEngineBase()(L1732) 检测 18080 连接
  → syncAllToLocalTool（1s 延迟）
  → 恢复画布 localforage
  → 注册监听（mutiwindow-task-completed / resourceAdded）
  > 注：画布持久化走 `Q.saveCanvasState`(App.js L1642) 直接写 localforage，**无 `canvas-state-change` 事件**（源码 grep 0 命中）。
```
- 日志锚点：`[初始化完成] 当前 isLoggedIn:`(L43484) —— 排查登录/初始化看这里。

### 3.3 鉴权拦截（去登录）
- `Oa()`(L3543) 返回 `local-mode-token`，所有 `if(!Oa()) throw '请先登录'` 在本地模式恒为 false（不抛）。
- `Aa`(L3552)/`ka`(L3547) 仅用于"重置配置"按钮清/写 token。
