## 2. P1 — 特定条件触发

### P1-1 资源面板「拖入 URL 不落盘」（TASKS 文档第14/34行）
- **审计坐实**：图片节点 `B`(onDrop, App.js:L29188) 拖入 URL 仅存内存 `data.imageUrl`，不经过 `ii`(uploadFile, L1888) 落盘。呼应模块3 P1-画布-04（节点内部 onDrop 不落盘）。
- **证据**：`App.js:L29188`（节点内 `let B = Y.useCall`），模块4 已对比画布级 `Lr`(L36363) 是落盘的——**层级差异**确认。
- **修复**：拖入 URL 时调 `ii()`(L1888) 下载落盘或提示。

### P1-2 文件上传不自动入库（TASKS 文档第15/35行）
- **审计坐实**：`R`(files) 上传成功后不调 `Sv()`/`we()`，需手动 rescan。对应模块2 P1 文件上传不入库。
- **证据**：模块2 锚 `App.js:L29133`（`R` 回调上传分支）。
- **修复**：`R(files)` 成功后调 `we()`(rescan调度, L43015) / `Sv()`(入库, L42838)。

### P1-3 ★新增 画布拖放落盘与面板断裂（流C→D 缺口）
- **审计坐实**：画布 `onDrop: Lr`(App.js:L36293) 调 `ii`(L1888) 落盘 `canvas/drop/` 后，**不调 `Di.current()`**（全局仅 L31426 生成完成、L44123 清空资源两处调用）。已落盘文件资源面板看不到。
- **证据**：`App.js:L36363`(落盘) + 全局 `Di.current` 调用点仅 `App.js:L31426`/`App.js:L44123`。
- **修复**：`Lr` 落盘成功后追加 `rescanThrottledSync()`(App.js:L43028)。
- **关联**：与 P1-1 互为表里——P1-1 是「节点内不落盘」，P1-3 是「画布级落盘了但面板不显示」。

### P1-4 ★新增 外部素材入库后未刷列表（流B→D 缺口）
- **审计坐实**：`resourceAdded` 处理(App.js:L43527) 末尾 `j('materials')`(App.js:L43555) 只切 tab，未显式 `D(...)` 写面板状态，依赖后续 rescan 才可见。
- **证据**：`App.js:L43527` / `App.js:L43555`。
- **修复**：入库后调 `rescanThrottledSync()`(L43028)。

## 3. P2 — 逻辑 / 设计缺陷

### P2-1 资源删除不删磁盘（TASKS 文档第20/36行）
- **审计坐实**：`wv()`(App.js:L42857) 只 `fetch(/api/resources/delete)` 删 DB；对比 `Tv()`(App.js:L42866 清空) 支持 `deleteFiles:true` 额外删磁盘，但 `wv`(单删) 无此开关（模块2 P2 #5 根因）。
- **证据**：`App.js:L42857`(wv) vs `App.js:L42866`(Tv 带 deleteFiles)。
- **修复**：`wv()` 加 `deleteFiles` 参数可选联动。

### P2-2 Rescan 孤儿清理只清 source='local-tool'（TASKS 文档第21行）
- **审计坐实**：localTool `handleResourcesRescan` 清理逻辑仅删 `source='local-tool'` 记录，extension 来源孤儿文件不清理（模块2 已坐实 localTool `resources.ts` 孤儿清理段）。
- **证据**：localTool `resources.ts` 孤儿清理段（映射表补全记录已锚该段附近）。
- **修复**：扩展清理条件覆盖 `source='extension'`。

### P2-3 缩略图伪复制（TASKS 文档第22/37行）
- **审计坐实**：localTool `files.ts`(缩略图伪复制处) 仅 `copyFileSync` 原图，文件名 `thumb_{maxDim}x{quality}_` 有误导性（模块2 P2 #7）。
- **证据**：localTool `files.ts` 缩略图段 / `ri`(缩略图缓存, App.js:L1856)。
- **修复**：接入 sharp 做真实缩放。

### P2-4 统一 base URL（TASKS 文档第38行）
- **审计坐实**：`Hr`(App.js:L1732) 与 `vv`(App.js:L42808) 两处独立来源，跨 `USE_LOCAL_ENGINE` 切换易漂移。
- **证据**：模块1 双 base URL 风险段。
- **修复**：同一配置源。

### P2-5 ★新增 批量连线依赖 selected 无 UI 引导（模块4 P2-画布-02）
- **审计坐实**：`xn`(onConnect, App.js:L31905) 仅在 source 选中且选中>1 时批量连线；`kn`(onNodeClick, App.js:L31980) 也支持多选批量。无 UI 提示，新手易困惑。
- **证据**：`App.js:L31905` / `App.js:L31980`。
- **性质**：非 Bug，体验风险。建议补引导提示。

### P2-6 ★新增 spawnable 集合两处重复定义（模块4 P2-画布-03）
- **审计坐实**：`new Set([...])` 在 `App.js:L4141` 与 `App.js:L5446` 完全重复。
- **证据**：`App.js:L4141` / `App.js:L5446`。
- **修复**：合并为单一常量（V1 优化，非红线禁止项）。

### P2-7 ★新增 `mutiwindow-task-completed` 多 dispatch 点语义重叠（交叉流 P2-交叉-03）
- **审计坐实**：同一完成事件在 `App.js:L38481`/`L43640`/`L43676`/`L43697`/`L44406` 共 5 处 dispatch，listen 仅 `App.js:L31428` 一处。重复 dispatch 被 `rescanThrottledSync`(L43028) 3s 节流吸收，风险可控。
- **性质**：冗余非 Bug。

### P2-8 ★新增 节点类型集与 spawnable 误读（模块4 纠正项）
- **审计坐实**：旧文档误判 `textNode/audioNode/textConcatNode` 因不在 spawnable Set 而不可生成；实测 `nodeTypes`(App.js:L31121/L31128/L31137) 已注册，均可经 `Z`(L35536) 生成。spawnable 是资源收集集合（App.js:L4141/L5446），非拖放白名单。
- **性质**：**非 Bug，是文档错误**——已作废旧质疑，避免维护者误删节点能力。
