## 3. 与 CloudSyncEngine 的区别（避免混淆）

同区域 @L43940-44004 的 `ei`/`ti` 是 **GAS 云同步**（`CloudSyncEngine.push/pull`，走官方云 `Mg.get('/sync/default')`），与本地文件导入导出（`Ri`/`Bi`）**是两套机制**：
- 文件导入导出：`Ri`@L44505（local 文件）/ `Bi`@L44545（读本地文件），纯本地、不联网。
- 云同步：`ei`(push)/`ti`(pull) @L43950/L43974，走 `CloudSyncEngine` → 官方云。

> ⚠️ 注意：`ei`/`ti` 云同步在本地模式（`USE_LOCAL_ENGINE=true`）下若被触发，会向官方云 `Mg.get('/sync/default')`@L43371 发请求 —— 属潜在外联点。但 `mg`/`CloudSyncEngine` 可能以 `skipAuth`(@L43372) 或本地 token 放行，实际行为需确认；列为**外联风险待查**（不同于模块5 的纯本地桩）。

## 4. 审计新发现（模块13）

1. **导出不含画布连线**：`Ri`@L44511 仅导 10 个配置/资源 key，**漏导 `canvas-state-v1-*`**（画布节点连线存 localforage）。用户导出的备份无法恢复画布 → **数据可携带性缺口**（P 级）。建议导出时遍历 localforage 的 `canvas-state-v1-*` 一并打包。
2. **老格式兼容完备**：`importData`@L44555 对旧版扁平结构做完整 `Z.*` 映射 + `APP_SETTINGS` 字段归并，健壮。
3. **画布写回绕开 `Q`**：导入时 `Sr.default.setItem`(@L44561) 直写 localforage，属批量导入特例，非日常写入，可接受但不符"一切过 `Q`"原则——建议加注释说明。
4. **云同步外联待查**：`ei`/`ti`(GAS) 在本地模式可能外联官方云，需确认是否受 `USE_LOCAL_ENGINE` 保护。
