## 2. 机位截图捕获与落盘 @L28418-28447

```
f = Y.useCallback(async t => {
  let n = [], r = [], o = new Set(a.syncedCaptureIds || []);
  t.project.cameras.forEach(e => {            // 遍历 3D 工程所有相机
    e.captures?.forEach(t => {
      r.push(t.id);
      if (!o.has(t.id))                       // 已同步的跳过
        n.push({ url: t.dataUrl, label: e.name||'机位截图' });
    });
  });
  n.length > 0 && a.onCaptureToBox?.(e, n);  // 截图推到 imageBox
  i(e, { directorProject: t.project, syncedCaptureIds: r });
  if (t.thumbnailDataUrl) {
    i(e, { imageUrl: t.thumbnailDataUrl });
    let n = await ii(t.thumbnailDataUrl, {     // ★ 落盘 18080（红线合规）
      subfolder: 'tasks', preferThumbnail: true,
      thumbMaxDim: 480, thumbQuality: 75 });
    n.url && /^https?:\/\//i.test(n.url) && i(e, { imageUrl: n.url, thumbnailUrl: n.thumbnailUrl });
  }
});
```

> ✅ 红线 §3.2 达标：`Th` 把 3D 工程缩略图 `t.thumbnailDataUrl` 经 **`ii()`@L28438**（统一上传，模块2 坐实）落盘 `uploads/tasks/`，不用 CDN URL 直存。新截图经 `onCaptureToBox` 推到 imageBoxNode（下游节点），形成 3D→2D 产出流。

## 3. 与画布节点的衔接

- 输入：上游 imageNode / imageBoxNode 提供底图（`d`@L28403 解析）。
- 输出：`imageUrl`（缩略图，已落盘绝对路径）+ `onCaptureToBox` 把各机位截图推入 imageBoxNode → 下游生图/合成。
- `directorProject` 存节点 data，随画布持久化（localforage `canvas-state-v1-`，见模块7/交叉流）。

## 4. 审计新发现（模块14）

1. **红线合规**：3D 缩略图走 `ii()`@L28438 落盘，与全局异步契约一致，无 CDN URL 直存。
2. **增量同步**：`syncedCaptureIds`@L28422 去重，已同步机位不再重复推 box，避免重复资源（与资源面板双数据源问题呼应，此处处理得当）。
3. **3D 工程存节点 data**：`directorProject` 含 cameras/captures 全量，随画布序列化；大模型工程可能使节点 data 体积膨胀（localforage 单 key 上限隐患，非阻塞）。
4. **`t(...)` 库未显式 import 核查**：`Th` 使用的 `t(...)` 应为 react-three-fiber 的 `useLoader`/`useStore`，但具体绑定需确认是否本文件顶部 import（混淆名 `t` 局部覆盖全局，审计时需注意 `t` 在 `Th` 作用域内是 3D hook 而非全局 token 变量）。
