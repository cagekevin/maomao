## 1. `mg` 错误分类器 @L31152

```
function mg(e) {                     // 入参 e = 错误/提示文本
  if (!e) return false;
  let t = String(e).toLowerCase();
  return !!(
    /余额|算力|额度|配额|充值|欠费|insufficient|quota|balance|not enough|payment|billing|计费/.test(t)
    || /权益不够|已达到使用次数额度|model_entitlement|model_quota/.test(t)
    || /审核|违规|敏感|涉黄|涉政|安全|拦截|blocked|moderation|safety|content[_ ]?policy|policy violation|prohibited|nsfw|sensitive/.test(t)
  );
}
```

三类正则：
| 类别 | 关键词（中/英） | 语义 |
|------|----------------|------|
| 额度/计费 | 余额/算力/额度/配额/充值/欠费/insufficient/quota/balance/not enough/payment/billing/计费 | 账号额度/付费问题 |
| 模型权益 | 权益不够/已达到使用次数额度/model_entitlement/model_quota | 模型调用配额耗尽 |
| 内容审核 | 审核/违规/敏感/涉黄/涉政/安全/拦截/blocked/moderation/safety/content_policy/policy violation/prohibited/nsfw/sensitive | 内容安全拦截 |

> 调用点：@L34772 `if (mg(o)) { M('调度已中断：'+o); return; }` —— 当生图/调度返回文本命中 `mg`，判定为"需中断调度"的致命错误（额度/审核），弹出提示并中止，而非重试。

## 2. `hg` 渠道名分类器 @L31157

```
function hg(e) {                     // 入参 e = apiUrl 或渠道标识
  if (!e) return '内置';
  let t = String(e).trim();
  if (!t || /maomao|jiangwei|weishao/i.test(t)) return '内置';
  ...（后续分支判外部厂商）
}
```

> 作用：把 apiUrl/渠道标识归一为 `内置` 或外部厂商名。调用点均为**生成历史记录**的 `channelName` 字段（@L32867 / L33451 / L33789 / L34101 / L35251 / L35298），即记录"这次生图走了哪个渠道"，纯展示/审计用途，不影响控制流。

## 3. 审计新发现（模块12）

1. **`mg` 是调度中断硬判定**：命中即 `return` 中止（@L34772），不做指数退避/重试 → 额度/审核类错误**不重试**（合理，重试无意义）。但非 `mg` 类错误（如网络 5xx）是否重试取决于上游轮询 `R`@L15146，二者错误处理策略分离，需注意一致性。
2. **正则健壮但保守**：`mg` 用 `toLowerCase()` 后中英文混合匹配，关键词覆盖主流额度/审核场景；但**新增厂商的错误文案若用词不在表内会被漏判**（假阴性 → 误重试）。建议错误分类下沉到网关/状态码层面（HTTP 429/403 moderation），而非纯文本正则。
3. **`hg` 仅展示用途**：6 处调用全是历史记录 `channelName`，零控制流依赖，安全。其正则 `/maomao|jiangwei|weishao/i` 为内置渠道白名单，新增内置渠道需同步维护。
4. **与任务状态机衔接**：`mg` 判定的中断最终反映为 `globalTasks` 的 `failed`/`FAILED`（模块8），但 `mg` 本身不写任务状态，只决定"是否 return"——状态由上层 catch 统一标记。
