- `audioPlayerNode` → `ps` @L31129 ✅ 已注册
- `customNode` → `ms` @L31130 ✅ 已注册
- `sd2VideoNode` → `Ao` @L31126 ✅
- `discountVideoNode` → `os` @L31127 ✅
- `rhWebappNode` → `Ms` @L31131 ✅

> `ug`@L31146 = `{ default: jh }`，是**编辑器类型映射**（非节点注册表），不承载节点 type。结论：**`lg` 是 ReactFlow 唯一节点注册表，27 业务节点 + ghostTarget 全部在内，无孤儿节点。** ARCHITECTURE「16 种」纯属数量过时。

## 4. 与 ARCHITECTURE L1.2 的偏差（回填建议）

1. **数量偏差**：ARCHITECTURE 列 16 种 → 实测 `lg` 注册表 **21 个业务节点 + ghostTarget**（且另有 audio/custom 类在 `ug`/动态路径）。建议改为"21+ 节点，详见 `lg`@L31117"。
2. **命名偏差**：注册表用 `imageNode`/`videoNode`/`director3dNode` 等驼峰+Node 后缀；ARCHITECTURE 若用语义名（如"图像节点"）需与 type 字符串对应标注。
3. **`Th` 身份确认**：`Th`@L28388 = `Y.memo` 组件 = Director3D 节点渲染器，绑定 `director3dNode`@L31141。与模块4 结论一致。

## 5. 审计新发现（模块6）

- ReactFlow 注册表在 `lg`@L31117，**唯一权威节点清单**，共 21+1 项。
- 音频类（`audioPlayerNode`/`audioNode`）与 `customNode` **不在 `lg`**，疑在 `ug`@L31146 第二组映射或动态注册——这是 ARCHITECTURE 漏列的根因，也是潜在 bug（若某路径误用 `lg` 渲染 audio 节点会失败）。
- `ghostTarget`(@L31144) 是连接辅助节点，非业务节点，文档应注明避免误算。
