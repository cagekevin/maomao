> 模块1（`10-模块1-系统初始化与配置层.md`）覆盖了 `Oa`/`Jh`/`Ba` 等符号，但未展开"本地模式为何免登录"的机制。本篇补全认证模型 + config.js 开关，闭合红线 §3.2「local token」的来龙去脉。

## 1. 本地模式 token 模型（核心）

| 符号 | 行号 | 语义 |
|------|------|------|
| `Oa` | @L3543 | 取本地 token：`localStorage.getItem(Ca) \|\| 'local-mode-token'`。**固定写死 `local-mode-token`**，本地模式永不走真实登录。 |
| `Ba` | @L3637 | `return !!Oa()` → 已"登录"判定恒 true（只要 token 存在，而 token 恒存在）。 |
| `Jh` | @L30712 | 鉴权头构造：`{ Authorization: 'Bearer ' + Oa() }`，注入所有网关请求。 |
| `Na` | @L3561 | 另一处鉴权头（收藏/提示词接口用），同样 `Bearer ${Oa()}`。 |
| `Da` | @L3540 | `localStorage.removeItem(wa)` —— 登出清 token（本地模式实际不调用）。 |

> ✅ 结论：本地模式是**伪认证**——所有请求带 `Bearer local-mode-token`，网关/后端以该固定串放行。无真实账号、无刷新令牌（refresh token）。这与红线 §3.2「local token」一致：要求用本地 token，而非官方 CDN 账号。

---

## 2. 被桩死的官方接口（本地模式不联网）

以下函数体首行即 `return []` / `return false` / `throw` 后接注释 `// 本地模式`，确认本地模式**完全不拉官方云**：

| 函数 | 行号 | 桩行为 |
|------|------|--------|
| `Ia` | @L3579 | `return []` 不拉官方收藏夹 |
| `La` | @L3589 | `return []` 不拉收藏项 |
| `za` | @L3625 | `return false` 不删官方收藏 |

> 风险点：这些函数**保留了被注释掉的官方 fetch 代码**（如 `fetch(`${ja}/prompts/favorites`)` @L3582-3584），一旦 `Oa()` 逻辑变动误走 else 分支，会向官方云发请求。当前 `Oa()` 恒返回 token，else 不可达，安全；但属**死代码隐患**，建议后续清理或加 `USE_LOCAL_ENGINE` 硬开关保护（见 §4）。

## 3. 配置开关（config.js）

`src/_engine/config.js` 关键导出（grep 坐实）：

| 符号 | 行号 | 值/语义 |
|------|------|---------|
| `AUTH_TOKEN_KEY` | @L33 | `'auth_token'`（key 名，注意与 `Oa` 实际用的 `Ca` 可能不同名） |
| `USE_LOCAL_ENGINE` | @L36 | `true`（走本地 localTool :18080） |
| `REMOTE_BASE` | @L39 | `'http://127.0.0.1:9004'`（USE_LOCAL_ENGINE=false 时的远程基址） |
| `localEngineBase()` | @L42 | `USE_LOCAL_ENGINE ? LOCAL_ENGINE.base : REMOTE_BASE` |

> ✅ 一致性确认：`Oa`@L3543 用 `localStorage.getItem(Ca)` 取 token，而 `Ca = 'auth_token'` @L3524，与 `config.js` 的 `AUTH_TOKEN_KEY='auth_token'` @L33 **完全同值**。`config.js` 导出与 App.js 常量指向同一 localStorage key，无错配，非死导出。

## 4. 审计新发现（模块5）

1. **免登录是真·固定串**：`Oa`@L3543 恒返回 `local-mode-token`，`Ba`@L3637 恒 true。本地闭环无账号系统，符合"本地模式"定位。
2. **三处官方接口被桩死**（`Ia`/`La`/`za` @L3579-3636），但保留死代码 fetch，建议加 `USE_LOCAL_ENGINE` 硬保护或删除。
3. **config.js `AUTH_TOKEN_KEY` 与 `Ca` 同值已确认**：`Ca='auth_token'`@L3524 = `AUTH_TOKEN_KEY='auth_token'`@L33，无错配（初版误判已结案）。
4. **开关分两层**：config.js 的 `USE_LOCAL_ENGINE`(@L36) 与 App.js 内 `Oa()` 的"伪 token"是两套独立机制——前者选基址，后者决定鉴权放行。两者耦合点：`config.js` 的 `REMOTE_BASE`(@L39) 指向 `:9004`（网关），但 `USE_LOCAL_ENGINE=true` 时基址走 `LOCAL_ENGINE.base`（:18080 localTool）。当前恒 true，网关地址仅作兜底。

> 本篇为模块1 的认证/开关补强；与 `10-模块1-系统初始化与配置层.md` 合并即构成"初始化与配置"完整切面。
