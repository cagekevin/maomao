## 1. 运转图景

AI 生成链路：画布节点触发 → 前端生图回调（`Jn`@L32490）→ 同步 SSE 或异步 task 分支 → 网关(:9004) → Lovart → 轮询 `GET /v1/tasks/{id}` → 提取结果 URL → **`ii()` 落盘 18080**（红线 §3.2 强制）→ 事件总线通知资源面板刷新。

> 红线 §3.2 硬要求：结果获取后**必须**经 `uploadFile` 机制下载落盘到 18080；**严禁直接使用 CDN URL** 当最终结果。实测 `Jn` 在 L33047-33051 **已实现该落盘**（HTTP URL 也走 `ii()`），符合红线。

## 3. 关键数据流（带 file:line）

### 3.1 生图主回调 `Jn`@L32490 双分支
```
Jn(e, r, o, s, c, d)  @L32490
  ├─ 预检：参考图/提示词收集（L32532–L32587）
  ├─ 模型选择：gemini/banana/openai 分支（L32591–L32597）
  ├─ 提交：fetch 网关 /v1/images/generations（同步 SSE 或返回 task_id）
  └─ 结果解析（N = openai/auto 且非 gemini/banana 走网关归一化）：
       ├─ 同步分支：SSE 流（L32900）/ 直接 JSON（L32946, rawResp=L32948）
       └─ 异步分支（检测 task_id，L32987）：
            while(true):
              pollUrl = `${R}/v1/tasks/${taskId}`   @L33005
              pollResp = await zc(pollUrl, GET, Bearer ${h})  @L33007
              taskInfo = pollData.data || pollData
              if status==='completed'（L33028）:
                 imgUrl = result.images[0].url[0]  @L33031
                 vidUrl = result.videos[0].url[0]  @L33032
                 u = imgUrl || vidUrl
                 // ★ 落盘红线：HTTP URL 也走 ii()
                 persisted = await ii(u,{subfolder:'tasks', preferThumbnail:true})  @L33049
                 u = persisted.url  // 改为绝对本地路径
                 break
              else if status==='failed'（L33055）: throw
              else:  // pending/processing
                 cid = resolveNeedsConfirm(taskInfo)  @L33059
                 if cid: markNeedsConfirm(...)  @L33061; return  // 退出轮询等确认
                 backoff（429 退避 15s，L33067）/ 正常间隔（L33068）
                 await sleep(backoff)（L33069）
```

### 3.2 轮询参数（实测）
- 基础间隔：`(ie||3)*1e3` = 3s（L33098）
- 429 退避：`Math.min(backoff*1.5, 15e3)` = 上限 15s（L33067）
- 总 deadline：`Date.now()+9e5` = 15min（L33097）
- AbortController：`ht.current.set(n, ac)`（L33094），取消机制

### 3.3 落盘与事件总线（红线 §3.2）
- 轮询成功 → `ii(u,{subfolder:'tasks'})`（L33049）→ `Xr()` → `POST /api/files/upload` → `uploads/tasks/`
- 节点回填：`resultUrl:u`（L33118）/ `data.images.push({url:u, source:'gen'})`（L33121-33128）
- 事件总线：`G_`（mutiwindow-task-completed）→ `Ev()` rescan → 资源面板刷新（见交叉流 T2.5.1）

### 3.4 网关侧（已读 main.py）
- `POST /v1/images/generations`（L591，见 ARCHITECTURE L3.2）返回 `{"status":"submitted","task_id":"task_"+thread_id}`（L781）
- `GET /v1/tasks/{task_id}` @L873 → `_check_and_fire_task`@L783（轮询 Lovart 状态）
- `POST /v1/tasks/{task_id}/confirm` @L882（AUTO_CONFIRM=false 时人工确认）
- 任务库：内存（`_TASK_META`），重启即丢（已知限制，非 bug）
