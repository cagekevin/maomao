## 1. 网关任务提交字段映射（前端 → Lovart）

`apimart-gateway/main.py` `_submit_generation` @L687 负责把前端请求字段翻译为 Lovart API 字段。实测映射：

| 前端字段 | Lovart 字段 | 位置 | 说明 |
|----------|-------------|------|------|
| `ratio` | `aspect_ratio` | @L703 | 比例 → 宽高比，命名不一致易踩坑 |
| `seconds` | `duration` | @L704 | 视频/音频时长字段改名 |
| `input_reference` | `reference_images` | @L707 | 参考图数组改名 |
| `prompt` | `prompt` | — | 透传 |
| `model` | `model` | — | 透传 |

> ⚠️ 风险：前端 `Jn`@L32490 派发的 body 用 `ratio`/`seconds`/`input_reference`，若前端某处误用 Lovart 原名（如 `duration`）会被网关层**默默忽略**（无默认值校验），导致静默降级。建议映射表补录到 `TASKS.md` 网关契约小节。

## 2. 未实现端点（统一返回 501）

`main.py` 中以下端点**已实现路由但返回 `err(501, ...)`**，即当前不可用：

| 端点 | 位置 | 语义 |
|------|------|------|
| `/v1/music_generations` | @L655 | 音乐生成（TTS-like 音乐） |
| `/v1/audio_generations` | @L659 | 音频生成 |
| `/v1/audio/speech` (TTS) | @L663 | 文本转语音 |
| `/v1/audio/speech` (STT) | @L669 | 语音转文本 |

> 结论：前端若存在音乐/音频/语音类节点，调用这些端点会**恒失败（501）**。模块4 实测节点清单（23+）含 `audioPlayerNode`（播放本地音频），但未发现"生成音乐/语音"节点 → 当前无前端调用这些 501 端点，暂未暴露。列为**潜在隐患**：若后续加音乐/语音生成节点，需先在网关落地实现，否则静默 501。

## 3. 任务轮询契约（前端 ↔ 网关）

模块3 已审计前端轮询：`Jn`@L32490 双分支（同步 SSE + 异步 poll），poll `GET ${R}/v1/tasks/${taskId}` @L33005，终态 `completed` → `ii(u,{subfolder:'tasks'})` @L33049 落盘。

网关侧对应实现：

| 函数 | 位置 | 作用 |
|------|------|------|
| `get_task` | @L873 | 返回任务当前状态 + `result`（含 `images[0].url[0]`） |
| `confirm_task` | @L882 | 前端确认任务完成（消费结果，避免重复下发） |

> ✅ 红线 §3.2 达标：网关 `task.result.images[].url` 是**临时 CDN URL**，前端收到后**不展示 CDN URL**，而是经 `ii()` 落盘 18080 后用本地绝对路径回填节点（见模块3 @L33049/@L33118）。网关侧无需感知落盘路径。

## 4. 网关审计小结

1. **字段改名 3 处**（`ratio→aspect_ratio` / `seconds→duration` / `input_reference→reference_images` @L703-707）——前端与 Lovart 字段体系不一致的唯一翻译层，建议记入契约文档。
2. **4 个 501 端点**（音乐/音频/语音 @L655-669）——当前无前端调用，属潜在隐患。
3. **任务契约健康**：`get_task`@L873 + `confirm_task`@L882 与前端轮询 @L33005 对齐，落盘经 `ii()` 合规。
4. 网关整体**不触碰本地文件**，仅做 Lovart 代理 + 字段翻译 + 任务状态管理，符合"网关不出本地文件"的职责边界。
