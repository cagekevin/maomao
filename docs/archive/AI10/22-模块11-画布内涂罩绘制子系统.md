## 1. 蒙版绘制上下文（ImageNode 内）

该子系统是 ImageNode 上的**局部蒙版编辑器**，用于在图上涂出"要重绘/擦除"的区域，输出 `maskUrl` 供 `Jn`(生图主回调) 做局部重绘（inpaint）。

| 符号 | 行号 | 作用 |
|------|------|------|
| `y.current` | @L7309/@L7345 | **蒙版 canvas**（独立离屏 canvas，存涂抹结果） |
| `b.current` | @L7376 | **实时笔迹 canvas**（绘制中预览层） |
| `v.current` | @L7357 | 显示容器 ref（用于坐标换算 `le`@L7355） |
| `ue` | @L7374 | 实时笔迹：在 `b.current` 画线段，`lineCap=round`，`lineWidth=u`，预览色 `rgba(250,204,21,0.85)`(erase)/`rgba(96,165,250,0.85)`(draw)；并压入 `x.current` 笔画栈 |
| `V` | @L7385 | **提交笔画**：清空预览层，把 `x.current` 所有笔画 stroke 到 `y.current` 蒙版；`f==='erase'` 用 `globalCompositeOperation='destination-out'`（擦除蒙版），否则 `source-over` 白笔（绘制蒙版） |
| `ie` | @L7326 | **快照**：`getImageData` 压入 `C.current` 撤销环（>`20` 个 shift 丢最旧） |
| `ae` | @L7334 | **撤销**：`C.current.pop()` + `putImageData` 回滚 |
| `oe` | @L7342 | **导出**：`y.current.toDataURL('image/png')` → 写入节点 `data.maskUrl`（经 `z(c,{maskUrl})`） |
| `se`/`ce` | @L7349/L7352 | 确认/取消蒙版编辑（`oe()` 后 `l(null)` 关编辑态） |
| `le` | @L7355 | 屏幕坐标 → 画布坐标换算（含旋转/scale，仿射逆变换） |

> 蒙版底色：无 `maskUrl` 时填白 `#fff`(@L7318)，有则 `drawImage(maskUrl)`(@L7315) 或白底兜底。

## 2. 与 `Jn` 生图主回调的衔接

`Jn`@L32490（生图主回调，模块3 已审计）在局部重绘时会读取节点 `data.maskUrl`（蒙版）。本子系统产出的 `maskUrl`(@L7346) 即其输入。下游 `ho(e.maskUrl)`(@L7045/@L7315) 异步加载蒙版 PNG，用 `destination-in` 与底图合成掩膜区域。

> ✅ 红线关联：蒙版是 canvas dataURL（内存态），经 `z(c,{maskUrl})` 写入节点 data，后续随节点持久化走 `Q`(模块7) / 生图时随请求发出，**不单独落盘为文件**（区别于资源文件）。属 UI 态数据，符合设计。

## 3. 审计新发现（模块11）

1. **撤销环上限 20**：`C.current` 撤销栈超过 20 帧即丢最旧(@L7332)，非无限撤销——长绘制会话早期笔迹不可撤销，属已知限制（非 bug，但 UX 隐患，建议可配置）。
2. **双 canvas 分层**：实时预览(`b`) + 蒙版结果(`y`) 分离，`V`@L7385 提交时合成，避免每帧重绘全蒙版，性能合理。
3. **橡皮 = destination-out**：`f==='erase'` 用 `globalCompositeOperation='destination-out'`(@L7394) 从蒙版"抠除"区域（而非涂黑），语义正确（蒙版中透明=不重绘）。
4. **坐标换算含旋转/scale**：`le`@L7355 做完整仿射逆变换，支持旋转/缩放后节点上的精确涂抹，健壮。
