## 2. `canvas-state-change` 结案（复核交叉流 §5）

复 grep `App.js` 全文 `canvas-state-change` 字符串 **0 命中**。画布持久化不走任何 CustomEvent，而是 ReactFlow onChange → `Pr.set`(localforage, `canvas-state-v1-` 前缀 @L1283 / @L1599 / @L44299)。该事件名**不存在**，旧文档假设作废。

## 3. 跨窗口协同架构（审计结论）

1. **双层通道**：
   - **CustomEvent 总线**（15 个）：用于窗口间"请求/完成"握手（工作流运行、强制保存、设置弹窗、任务完成/重跑/元信息）。
   - **localStorage 通道**（`mutiwindow-clipboard`）：节点/图片的跨窗口复制粘贴载体（@L16376 写、@L36034 读），与 CustomEvent 配合（先写 localStorage 再 dispatch `mutiwindow-nodes`/`mutiwindow-images`）。
2. **主触发器单一**：`mutiwindow-task-completed` 是资源面板 rescan 的唯一主驱动（5 处 dispatch @L38481/L43640/L43676/L43697/L44406，监听 @L31428），事件总线设计健康。
3. **设置弹窗双事件**：`mutiwindow-open-schedule-settings` / `mutiwindow-open-builtin-settings` 在 L43774 一并注册监听、L43779 一并注销——配对管理，无泄漏。
4. **无 `canvas-state-change`**：画布状态经 `Pr`(localforage) 直接落盘，不参与事件总线（见 §2）。
