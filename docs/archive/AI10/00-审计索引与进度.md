## 4. 审计方法（四段式 + 校验门，取自 TASKS.md 五关流水线）

每个模块文档采用四段式：
1. **运转图景**：本模块职责与系统位置。
2. **核心混淆字典**：涉及混淆名→可读名（先查映射表，未收录的 grep 坐实后补入模块 doc，不回写原表）。
3. **关键数据流**：带 `file:line` 调用链。
4. **存疑 Bug**：对照 `TASKS.md` P0–P2，标出本模块相关风险与证据。

**校验门**：每模块写完，跑 `01-audit-check.cjs` 抽 `App.js:Lnnnn`/函数名 → 回源码 grep 确认存在 → 生成/更新 `校验报告-AI10.md`。
