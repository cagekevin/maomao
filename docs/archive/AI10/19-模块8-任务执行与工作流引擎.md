## 1. `globalTasks` 状态机

`globalTasks` 是全局任务列表（store 键 `Z.GLOBAL_TASKS` @L1282，持久化经 `Q.getObject` @L43130）。状态枚举（grep 实测）：

| 状态 | 设置位置 | 含义 |
|------|----------|------|
| `QUEUED` | @L15276 | 入队（映射 Lovart `QUEUED`） |
| `RUNNING` | @L15279 | 运行中（映射 Lovart `RUNNING`） |
| `completed` | @L15249 | 终态成功 |
| `failed` | @L15159/@L15169 | 终态失败（超时/HTTP 错） |
| `FAILED` | @L15155/@L15265 | 失败别名（loading=false + errorMessage） |

> ⚠️ **状态命名不一致**：同一状态机混用 `completed`/`failed`（小写，React 侧 store 值）与 `FAILED`（大写，loading 包装）。疑似两处代码风格未统一，若下游按 `=== 'failed'` 判断会漏掉 `FAILED`。建议统一为单一枚举。

## 2. 第二条轮询路径 `R` @L15146（任务清单轮询）

`R = Y.useCallback(async t => {...})` @L15146 是任务清单里单任务的轮询器：

```
D.current = t (防重入锁)              @L15147
r = async () => {
  if (Date.now() - n > 6e5) {         // 10 分钟超时
    标记 failed: '任务轮询超时'        @L15152-15161
    return
  }
  fetch(rr(p, '/task/' + t), {        // ★ 官方风格路径 /task/{id}
    headers: { Authorization: 'Bearer ' + m }  // m = 另一 token
  })                                  @L15165-15169
  i = r.status.toUpperCase()
  if (i === 'SUCCESS') {              // ★ 官方状态词 SUCCESS（非 completed）
    解析 r.results[].url → 类型判定 js()  @L15173-15185
    ...
  }
  T.current = setTimeout(r, 1500)     // 1.5s 轮询间隔  @L15285
}
```

> **关键差异（与模块3 `Jn`@L32490 路径对照）**：
> | 维度 | `Jn`@L32490（生图回调内轮询） | `R`@L15146（任务清单轮询） |
> |------|------|------|
> | 端点 | `GET ${R}/v1/tasks/${taskId}` @L33005（网关 `get_task`@L873） | `rr(p,'/task/{id}')` = `{tr(p)}/v1/gateway/ai-app/task/{id}`（**同为本地网关**，`rr`@L856 用 `tr()` 解析基址，非官方云） |
> | 鉴权 | `Jh()`→`Bearer Oa()`（local-mode-token） | `Bearer ${m}`（`m` 为组件闭包内 auth token，本地模式同样为固定串） |
> | 终态词 | 前端判 `completed` | 判 `SUCCESS`（Lovart 官方状态词） |
> | 间隔 | 见模块3 | 1500ms @L15285 |
> | 超时 | 见模块3 | 600s @L15152 |
>
> ✅ 修正（初版误判已结案）：`rr`@L856 实为 `${tr(e)}/v1/gateway/ai-app{t}`，基址由 `tr()`（模块1 URL 助手）解析，**仍是本地网关**，并非官方云。故 `R` 不会外联官方。两路轮询本质都打本地网关，差异仅在**子路径**（`/v1/tasks/` vs `/v1/gateway/ai-app/task/`）与**终态词**（`completed` vs `SUCCESS`）——疑似历史演进留下的两套网关契约并存。
>
> ⚠️ **两路轮询并存且契约不一**：生图节点内（`Jn`）与任务清单（`R`）各自轮询，子路径与终态词不一致。建议统一为单一网关契约（或确认 `R` 是否为旧版兼容分支，可随 `USE_LOCAL_ENGINE` 收敛）。属一致性隐患，非外联风险。

## 4. 审计新发现（模块8）

1. **状态枚举双风格**：`completed`/`failed` 与 `FAILED` 混用，建议统一。
2. **两路轮询并存且契约不一**：`Jn`@L32490（网关 `/v1/tasks/`，判 `completed`）vs `R`@L15146（`rr`→`/v1/gateway/ai-app/task/`，判 `SUCCESS`），均为本地网关。属历史演进两套契约并存 → **一致性隐患**，建议收敛为单一契约。
3. **轮询健壮性**：`R` 有 10 分钟超时(@L15152) + 1.5s 间隔(@L15285) + `D.current` 防重入锁(@L15147)，设计合理；但失败分支静默 `updateGlobalTasks` 标记 failed，无重试。
4. **`globalTasks` 持久化健康**：经 `Q.getObject(Z.GLOBAL_TASKS)`(@L43130) 播种到 SQLite，重开可恢复历史任务。
