# 14 - 后端双基址与数据流一致性审计

> 审计范围：消除 round 5 待查项——资源 base `vv` 与 AI 生成 base `R` 的关系；澄清"双基址"架构意图与一致性边界。
> 方法论：五道闸门。所有锚点经 grep 复核（见 `校验报告-AI11.md`）。

## 14.1 两个基址的真实身份（round 5 待查项解答）

| 变量 | 锚点 | 真值 | 端口 |
|------|------|------|------|
| `vv`（资源 base） | `App.js L42808`：`var vv = LOCAL_ENGINE.base;` | `LOCAL_ENGINE.base` = `http://${host}:${port}` | **:18080**（localTool，config L12-17） |
| `R`（AI 生成 base） | 生成组件作用域；默认值 = `DEFAULT_GATEWAY_URL`（config L59 = `http://127.0.0.1:9004`，L30） | API 网关地址（用户可在设置面板改，`ot = DEFAULT_GATEWAY_URL` @L43212） | **:9004**（apimart-gateway） |
| `$n`（HTTP 封装 base） | `App.js L844`：`var $n = Vn();` | `Vn()` = 网关请求封装，内部拼 `DEFAULT_GATEWAY_URL` | **:9004** |
| `Hr`（文件操作 base） | `App.js L1732`：`var Hr = localEngineBase();` | `localEngineBase()` = `LOCAL_ENGINE.base`（config L42-43） | **:18080** |

### 结论
**`vv` 与 `R` 故意分端口，不是 bug，是架构分工**：
- `:18080`（localTool）= KV 存储、文件上传/缩略图、资源 CRUD/rescan（`vv`/`Hr` 都用它）。
- `:9004`（apimart-gateway）= AI 生成请求转发（`R`/`$n` 用它），把第三方 API（OpenAI 兼容）协议转给 Lovart 等。

round 5 担心的"资源存 A、生成读 B 漂移"**不存在**：资源与生成结果都落 `:18080`（资源走 `vv`=:18080，生成结果 `ii` 走 localTool :18080）；仅 AI **请求**发往 `:9004` 网关，网关再外呼第三方——符合 config 顶注"AI 生成请求走用户配置的第三方 API"（config L8）。

## 14.3 `Sg` URL 拼接机制（发布路径真相）

`Sg`（`App.js L38213`）：
```js
function Sg(e, t) {
  return e.startsWith(`http`) ? e
    : `${t.useLicenseServer, $n}${e.startsWith(`/api`) ? e : `${er}${e}`}`;
}
```
- 若路径已是 http 全称（如用户文本端点），原样使用。
- 否则拼 `$n`（网关 :9004）+ `/api` + 路径。
- `er = '/api'`（L845），故 `/workflow-apps/publish` → `http://127.0.0.1:9004/api/workflow-apps/publish`。

确认发布目标 = 网关 :9004，与资源 :18080 分端口但各自正确。

## 14.4 对抗审计（Gate4）

1. **设置面板改网关地址的涟漪效应**：`ot = DEFAULT_GATEWAY_URL`(L43212) 是设置面板 state，用户改后 `st/pt/yt/Tt` 等同步（L43213/43215/43219/43223）。若用户把网关改成 :18080（误配），则 AI 请求与资源同端口——此时 `Sg` 拼出 `:18080/api/workflow-apps/publish`，但 localTool 无此端点 → 发布 404。属**用户误配风险**，非代码 bug，记 T（建议设置面板校验端口冲突）。
2. **`vv` 硬编码 localTool，不随 USE_LOCAL_ENGINE 切换**：`vv = LOCAL_ENGINE.base`(L42808) 固定 :18080，而 `R`/`$n` 经 `DEFAULT_GATEWAY_URL` 可能指向远程（`REMOTE_BASE`，config L39）。若 `USE_LOCAL_ENGINE=false`，资源操作仍打 :18080（硬编码），与"远程模式"预期不符——潜在不一致，但当前 `USE_LOCAL_ENGINE=true`（config L36）不触发。记 T（待远程模式启用时核查）。
3. **round 5 待查项关闭**：`vv` 与 `R` 关系已澄清为"分端口架构分工"，**非数据漂移 bug**。原记 T 项可降级为"架构说明"，不回写 TASKS.md 为 bug。
