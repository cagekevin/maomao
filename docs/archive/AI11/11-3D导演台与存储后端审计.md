## 11.2 导演台面板（`wh`）

`function wh({initialProject, initialPanoramaUrl, onExit})`（`App.js L28276`）。

### 独立状态 store（`$`）
导演台使用**与画布主 store 完全分离**的 Zustand store（变量 `$`），关键 selector/action：
- `$.viewMode` / `$.setViewMode`（`App.js L28281-28282`）
- `$.replaceProject`（`App.js L28283`，L28287 初始化 `a(e ?? Du())`）
- `$.addImportedAsset`（`App.js L28284`，L28298 导入全景图 `dm(t)` 结果）
- `$.project.panoramaAssetId`（`App.js L28292`）
- `$.selectedObjectId/selectedObjectIds/selectedCrowdId` + `$.deleteSelectedObject`（`App.js L28313-28314`）
- `$.copySelectedObjects/$.pasteClipboardObjects/$.undo`（`App.js L28319`）
- 快捷键 `Delete/Backspace/C/V/Z/Y` 全局监听（`App.js L28308-28321`）

### 关键结论（隔离性）
**Director3D 状态不经由 `Q.saveCanvasState` 持久化**。它只在 `onExit` 时把 `project` 序列化为 `directorProject` 字段塞进 node data（`App.js L28431`），随画布整体保存。即：导演台内部实时编辑（机位移动、对象增删）**不会中途落盘**，必须退出节点才持久化。若用户在导演台内编辑后未点退出直接刷新页面，改动丢失。属 UX 风险点，记 T。

### 全景图导入 `dm`（`App.js L28296`）
`dm(t)` 把全景 URL 转为 asset（`kind:'panorama'`），失败仅 `console.warn`（`App.js L28303`），不阻断面板打开。

### 截图 `s`/`c`（`App.js L28323-28334`）
`s()` 调 `Ed({preset:'current', source:'capture-panel'})`（`App.js L28325`）取 `dataUrl`；`c()` 组合 project + 截图供 `onExit` 使用。

## 11.4 存储后端全貌（四后端职责矩阵）

基于 `wr`/`Mr`/`Nr`/`Pr` 定义（L1297/L1364/L1418/L1446）与 `Q` 封装（L1476）汇总：

| 后端变量 | 实现 | 可用性判定 | 主要用途 |
|----------|------|-----------|----------|
| `wr` | `window.localTool` WASM | `window.localTool?.status?.isConnected`（`L1332`） | 主存储：KV、画布状态、资源文件、任务 |
| `Mr` | chrome.storage.local | `typeof chrome < 'u' && chrome.storage?.local`（`L1415`） | 扩展环境同步源，启动时被 `syncToLocalTool` 拉到 localTool（`L1593`） |
| `Nr` | window.localStorage | 永远 `true`（`L1443`） | 轻量配置/标志位 |
| `Pr` | localforage（`Sr.default`） | 由 localforage 自身决定 | 画布状态**双写**副本（`setLocalforage` L1636） |

### 同步流向（启动期）
- `Q.syncToLocalTool(e)`（`App.js L1591`）：localTool 可用且 chrome.storage 有值 → `wr.set(e,t)`（`L1594-1595`）。
- `Q.syncAllToLocalTool()`（`App.js L1608`，模块2/8 已确认）：批量从 chrome.storage/localforage 拉取缺失键补到 localTool。

### 缓存层 `jr`（`App.js L1345`）
图片类 key（`img_`/`img_thumb_`/`video_thumb_`，`Or` 定义 `L1338`）走 `jr` 带 600s TTL 缓存（`Dr=L1337`）+ 并发去重（`Er` Map，`L1351-1362`）。`Q.getConfig`/`getObject` 均经 `jr`（`L1493/L1510`）。

## 11.5 对抗审计（Gate4）

1. **导演台状态丢失风险**（高）：如 11.2 所述，未退出节点直接刷新 → 导演台内编辑全丢。建议（仅记录，不改代码）：退出前自动 `onExit` 或定时快照。
2. **`ii` 持久化的 thumbnail 与 base64 回退**（`App.js L28438-28449`）：localTool 不可用时 `ii` 失败，节点保留 base64 `thumbnailDataUrl`，node data 体积膨胀但功能不崩——降级合理。
3. **四后端数据一致性**：`setLocalforage` 双写 localTool+localforage，但 `Mr`(chrome.storage) 仅在启动同步时拉一次（`L1591`）。运行期直接写 localTool 不会回写 chrome.storage，重启后 chrome.storage 为旧值，再次 `syncToLocalTool` 会**用旧 chrome.storage 覆盖新 localTool**（`L1594` 无条件 `wr.set(e,t)`）。**这是真实的数据回退 bug**：若 localTool 数据比 chrome.storage 新，重启会被旧 chrome.storage 覆盖。**记 P 级（高危），建议用户授权后回写 TASKS.md**。
