## T2.5.2 Rescan 孤儿清理与资源入库一致性

```
[模块2] Ev()(rescan@L42883) → POST /api/resources/rescan
        → localTool resources.ts L120: 孤儿清理只清 source='local-tool' (@L122)
        → source='extension' 记录不被清理 (TASKS P2-7)
[冲突面] "拖入 URL 不落盘"(TASKS P1-2): resourceAdded 中 extension 来源经 Zr 落盘(@L43539)→Sv 入库(@L43552)
        但"资源面板 URL 拖入"(@L29176) 只存状态不下载 → 刷新丢 → 与 rescan 不清理 extension 孤儿叠加，产生僵尸记录
归因: 两处 URL 入资源逻辑不一致（一处落盘一处不落盘）。建议统一走 Zr 落盘。
```


## 交叉流审计结论

1. 四条流接缝坐标全部 grep 坐实，模块间通过 HTTP + window CustomEvent 解耦，符合三层隔离（红线 §3.2）。
2. **唯一真问题**在 T2.5.2：URL 入资源两处逻辑不一致（落盘 vs 不落盘），叠加 rescan 不清理 extension 孤儿 → 僵尸记录。建议模块2 修复 P1-2 时统一落盘路径。
3. base URL 双源硬编码（T2.5.3）为 P0 残留，前端无直接修复锚点，归 config/localTool 侧。
