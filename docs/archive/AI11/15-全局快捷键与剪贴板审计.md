## 15.2 节点复制/粘贴机制（`navigator.clipboard` 单通道）

### 复制 `jr`（`App.js L36129`）
- 取选中节点 `Qt.current.filter(selected)`（L36130），含连线 `et.current`（L36137）。
- 组装 `{type:'mutiwindow-nodes', nodes, edges, originalIds}`（L36138-36152），**剔除函数字段与临时态**（loading/progress/errorMessage/imageUrlRef 等，L36144-36146）。
- `navigator.clipboard.writeText(JSON.stringify(n))`（L36155），失败提示"复制失败，请检查浏览器权限"（L36157）。

### 粘贴 `kr`（`App.js L35898`）
- `navigator.clipboard.readText()`（L35915）→ 解析 `mutiwindow-nodes`（L35918）。
- **id 重映射**：新 id = `${type}-${Date.now()}-${random}`（L35932），`f` Map 记录旧→新映射，边 source/target 同步替换（L35938 起）。
- 位置偏移：以原节点包围盒中心为基准重定位（L35924-35929）。
- 非 `mutiwindow-nodes` 内容（如纯文本/图片文件）→ 走 `handlePaste` 的 `mutiwindow-images` 或 imageNode 分支（L36969-36998）。

### 关键缺口（一致性风险，T 级）
**节点复制无 localStorage 兜底**：`jr`(L36155) 只用 `navigator.clipboard`，权限拒绝即失败。而图片复制 `P`(L16374/L16376) 有 `localStorage('mutiwindow-clipboard')` 兜底。两套机制不对称——节点复制在 clipboard 权限受限环境（如某些 MV3 弹出页）会静默失败。建议（不改代码）给 `jr` 加同类兜底。

## 15.3 图片复制/粘贴机制（双通道 + 跨窗口）

### 复制 `P`（`App.js L16362`）
- 组装 `{type:'mutiwindow-images', images}`（L16368-16370）。
- **双通道**：先 `navigator.clipboard.writeText(t)`（L16374），失败 `catch` → `localStorage.setItem('mutiwindow-clipboard', t)`（L16376）。

### 粘贴（消费方 `App.js L36001` / 兜底 L36034）
- 主路径：监听 `mutiwindow-images` CustomEvent（L36001）→ 生成 imageNode 网格（L36006-36030）。
- 兜底：CustomEvent 未命中时读 `localStorage('mutiwindow-clipboard')`（L36034）→ 同样解析 `mutiwindow-images`（L36037）。

### 与 round 4 呼应
本机制即 round 4 所述的"多窗口图片粘贴双通道"，锚点一致（L16376/L36034/L36001）。图片复制走 `mutiwindow-images` 事件总线 + localStorage，`navigator.clipboard` 为第一通道——与节点复制（仅 clipboard）形成对照。

## 15.4 导演台独立剪贴板（隔离）

导演台快捷键 `C`/`V`/`Z`/`Y`（`App.js L28319`）调用独立 store `$` 的 `copySelectedObjects`/`pasteClipboardObjects`/`undo`（`$` 实现 L22294/L22303，克隆逻辑 `$u` L21460-21560）。**完全隔离于主画布 `jr`/`kr`**——导演台内复制的对象不会进主画布剪贴板，反之亦然。属设计隔离（round 3/11 已记 Director3D 状态隔离）。

## 15.5 对抗审计（Gate4）

1. **节点/图片剪贴板不对称（T 级）**：节点复制仅 `navigator.clipboard`（L36155），图片复制有 localStorage 兜底（L16376）。建议统一加兜底，待授权回写 TASKS.md。
2. **`mutiwindow-nodes` 与 `mutiwindow-images` 命名一致性**：两者拼写均少一个 l（`mutiwindow`，非 `multiwindow`），与 round 4 一致。文档若写 `multiwindow-nodes` 均错。
3. **跨窗口节点粘贴限制**：`mutiwindow-nodes` 走 `navigator.clipboard`（系统级，天然跨窗口/跨文档），但 `mutiwindow-images` 的 CustomEvent 仅同源同文档可见，靠 localStorage 兜底跨窗口。两套跨窗口能力不对等：节点复制天然跨窗口（系统剪贴板），图片复制需事件+localStorage 双保险。功能均可用，但机制复杂度不一。
4. **`Ctrl/Cmd+D` duplicate 时序**：`Mr`(L36162) = `await jr()` 然后 `setTimeout(kr, ...)`（L36163 起），依赖 clipboard 异步写入后再读。若 clipboard 写入慢或失败，`kr` 读到空 → duplicate 失效。低概率竞态，记 T。
