## 一、运转图景（这个模块做什么）

用户在节点（如 PromptNode/ImageNode）点击生成 → 组件内 `Jn`(useCallback) 主回调被触发 → 按模型类型选网关端点 → 提交拿 `task_id`（异步模式）→ 进入轮询循环（≥3s，退避，15min deadline）→ 拿到结果 URL → **经 `ii()` 下载落盘 18080**（不直用 CDN）→ 更新节点 `data.imageUrl/videoUrl` → 派发 `mutiwindow-task-completed` 事件 → 资源面板刷新（模块2/4 消费）。

## 三、关键数据流（端到端，已逐行坐实）

```
Jn()(局部 @L32490)
  ├─ N 分支(异步): I = `${R}/v1/images/generations` (@L32796)  [或 videos/music...]
  │     → fetch 提交 → 拿 task_id
  │     → while(true):
  │         pollUrl = `${R}/v1/tasks/${taskId}` (@L33005)
  │         pollInterval = (ie||3)*1e3  (≥3s, @L32998)
  │         deadline = Date.now()+9e5  (15min, @L32997)
  │         if status==='completed' (@L33028):
  │            u = result.images[0].url[0] / videos[0].url[0] (@L33031-33032)
  │            if u && !data: → ii(u,{subfolder:'tasks',preferThumbnail:true}) (@L33049) ★落盘18080
  │            u = persisted.url (绝对路径 @L33050)
  │            break
  │         if status==='failed' → throw (@L33056)
  │         if pending_confirmation → markNeedsConfirm + return (卡轮询, @L33058-33063)
  │         429 → backoff=min(backoff*1.5,15e3) (@L33067); else backoff=pollInterval
  │         await sleep(backoff); pollCount++ (@L33069-33070)
  └─ 同步分支(无 task_id): 直接取 b64_json/url (@L33077-33082)
→ 更新节点 data → dispatchEvent('mutiwindow-task-completed',{taskId,nodeId,resultUrl,...}) (@L38481)
```

## 五、排雷重点（对照 TASKS / 红线）

1. **轮询 7 陷阱**：间隔≥3s（@L32998）、15min deadline（@L32997）、429 退避封顶 15s（@L33067）、AbortController（@L33002）、JSON 解析失败抛错（@L33017）、进度回写（@L33024）、终态严格判定（@L33028）。均已坐实，无明显缺失。
2. **`pending_confirmation` 卡死**：`AUTO_CONFIRM=false` 时 `markNeedsConfirm` 退出轮询（@L33058），需手动 confirm 路由（网关 L882）。属已知限制（TASKS 其他限制）。
3. **CDN 直链违规（红线 §3.2）**：**已合规**——@L33047 明确 `if(u && !u.startsWith('data:'))` 才走 `ii()` 落盘，且 @L33050 用持久化绝对路径覆盖 `u`。但**风险点**：`ii()` 对 `http(s)` URL 直接返回（不落盘，见 `ii`@L1889），仅当 `subfolder` 传入时才落盘；此处传了 `subfolder:'tasks'` ✅，故落盘生效。若调用方漏传 `subfolder` 会退回直链——需全局排查（归模块2 统一同步 effect）。
4. **音频/音乐 501**：网关 `main.py` L655/L659/L663 返回 501（TASKS 已知限制），前端对此无特殊处理，会抛错→节点置 failed。

## 六、存疑 Bug（门4 留痕）

- **`R`/gatewayBase 来源**：本审计未在 L32490 附近直接 grep 到 `R=config.DEFAULT_ENDPOINT` 赋值，需确认 `R` 是否为组件 props/context 传入（来自模块1 端点选择）。建议文档维护者补充 `R` 的绑定来源行号。
- **统一同步 effect 的落盘兜底**：TASKS P0/P2 提到「统一同步 effect 遍历 globalTasks 未本地化则 uploadFile」，但 `ii` 对纯 http URL 直返——需确认 effect 是否传 `subfolder`，否则刷新即丢（红线风险）。建议模块2 交叉核对。

## 七、对抗审计质询

- Q：结果 url 是否可能绕过 `ii()` 直存？A：仅 `data:` 前缀（base64 内联）会跳过 `ii()`（@L33047），属正常（无需落盘）；http(s) 一定走 `ii()` 落盘。✅ 合规。
- Q：`mutiwindow-task-completed` 谁消费？A：L31428 监听（`we()`/rescan 刷新资源面板）；模块2 `we`@L43015 经 `rescanThrottledSync` 触发 `Ev()`+`xv()`。
