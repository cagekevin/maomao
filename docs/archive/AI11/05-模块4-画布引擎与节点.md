## 六、同名遮蔽追加证据（`we` 第三处）

- 模块2 指出 `we`@L43015=loadResources（组件局部）。
- 本模块 `Fr`@L36253 调 `we({x,y})` —— 这是**屏幕坐标转画布坐标的 helper**（参数 `{x,y}` 为坐标），与 loadResources 签名（无参 async）**完全不同**，证明 `we` 至少有 **3 处不同身份**（L4176 视口比例 / L43015 loadResources / L36253 坐标转换 helper）。
- **审计铁律**：凡引用 `we`，必须带行号+作用域，禁止只写 `we`。

## 七、排雷重点（对照 TASKS / 红线）

- **新增自定义节点冲突面**：新增节点须同时注册 `nodeTypes` 与（若可拖放）spawnable Set；否则拖放不可用。混淆逻辑（handler 用 `e.id`/`nodeId` 传递）对节点 `type` 字符串敏感，新增 type 须与后端/存储键一致。
- **`Yr`(onEdgeDoubleClick) 删边未走撤销栈**（@L36570）：直接 `G(filter)` 过滤，与节点删除可能不一致，属设计取舍（非 bug）。
- **`Th`(Director3DNode) 3D 子系统**：独立 Three.js 场景，接入点 `kp`(registerModelGenerateHandler)@L26080、`wd`(registerViewportCaptureHandler)@L22986；与 ReactFlow 画布的媒体同步走 `Ic`(useNodeMediaSources) 提取。
