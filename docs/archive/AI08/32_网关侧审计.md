# 模块5 · 网关侧（apimart-gateway）审计

> 审计对象：`apimart-gateway/main.py`（FastAPI，OpenAI 风格中继，后端 Lovart）。
> 行号快照：2026-07-21（gateway 行号稳定，但脚本仍带 ±40 软校验）。
> 关联：本模块是模块3（AI生成与网关层）的服务端对等体；模块3 在 `App.js` 侧 `GET /v1/tasks/{id}` 轮询，本模块即该接口的实现方。

---

## ① 运转图景

网关是 **OpenAI 兼容中继层**：把前端 `App.js` 发出的图片/视频生成请求，翻译成 Lovart 后端协议，并把 Lovart 的 `thread` 状态轮询翻译回 OpenAI 风格的 `task`。

核心生命周期：
1. 前端（模块3 `let Jn` @ `App.js:L32490`）派发生成 → POST `/v1/images/generations` 或 `/v1/video/generations`（`main.py` 同文件 `@app.post` 路由）。
2. 网关 `_do_submit`（`main.py:L719`）写入内存表 `_TASK_META`（`main.py:L48`），生成 `task_<thread_id>` 作为对外 `task_id`。
3. 前端轮询 `GET /v1/tasks/{task_id}`（`main.py:L873`）→ 调 `_check_and_fire_task`（`main.py:L783`）。
4. 终态时 `_check_and_fire_task` 调 `lovart_to_apimart`（`main.py:L142`）映射产物，并 `_fire_webhook`（`main.py:L369`）回调前端（若前端登记了 webhook）。
5. 音视频类请求被直接 501 拒绝（见 §④）。

关键环境变量：`AUTO_CONFIRM`（`main.py:L34`，默认 `true`）、`TASK_RESULT_TTL`（`main.py:L37`，默认 86400s）、`LOVART_TIMEOUT`、`LOVART_SECRET_KEY`。

## ③ 关键数据流（带 file:line）

### 流 A：轮询终态判定（与模块3 轮询对齐）
- `GET /v1/tasks/{task_id}` → `main.py:L873` → `_check_and_fire_task`（`main.py:L783`）。
- `status == "done"` 时网关做 **5 秒去抖**：首次见到 `done` 记 `done_first_seen`，未过 5s 仍返回 `running`（`main.py:L799-L808`），避免 Lovart 半成品被过早拉取。
- `pending_confirmation` 分支（`main.py:L823`）：若 `AUTO_CONFIRM` → 自动 `client.confirm` 后继续轮询；否则返回 `processing + error.code="pending_confirmation"`（`main.py:L854`）。**此即模块3 `App.js:L33061` 处理的 `pending_confirmation` 来源。**
- 产物映射后返回 `completed`（`main.py:L867`），含 `result` 字段；前端 `App.js:L33049` 将 `result` 中 url 经 `ii()` 落盘到 localTool。

### 流 B：确认回路（AUTO_CONFIRM=false 的安全阀）
- `confirm_task`（`main.py:L882`）→ `client.confirm(thread_id)`（`main.py:L893`）。
- 与提交期 `pending_confirmation` 处理（`main.py:L507-L536`）共用 `client.confirm`，保证高成本操作既能自动过也能手动过。

### 流 C：产物映射与过期
- `lovart_to_apimart`（`main.py:L142`）：`artifacts[].content` 即 URL，`type ∈ {image,video,audio,music}`；`expires_at = now + TASK_RESULT_TTL`（`main.py:L144`）。
- **关键边界**：映射只产出 URL，不含本地持久化。若前端 `App.js:L33051` 在 localTool 不可达时回退 CDN，则拿到的是 Lovart 临时 URL（受 `expires_at` 约束），**这正是模块3 已标记的 🔴 红线冲突**——CDN URL 不应作为最终结果，必须落 localTool。网关侧只是如实回传 URL，矛盾点在 App.js 侧。

### 流 D：音频/音乐 501（噪音实锤）
- `/v1/music/generations`（`main.py:L655`）、`/v1/audio/generations`（`main.py:L659`）、`/v1/audio/speech`（`main.py:L663`）、`/v1/audio/transcriptions`（`main.py:L667`）全部 `err(501, "...不被 Lovart 后端支持")`。
- → **证实 TASKS.md「音频/音乐生成 501」为已知噪音，非 Bug**，前端不应调用此四类接口。

## ④ 存疑 Bug / 边界契约（对照 P0–P2）

### 边界契约（本模块对外承诺）
1. `task_id` 必须形如 `task_<thread_id>`，否则 `_check_and_fire_task`（`main.py:L785`）与 `confirm_task`（`main.py:L888`）直接 `err(400)`。
2. 产物 URL 带 `expires_at`，前端须在过期前落盘（localTool）；网关不负责持久化。
3. `pending_confirmation` 是**可恢复**中间态，须由 `AUTO_CONFIRM=true` 自动过 或 前端调 `/confirm` 手动过；若两者皆无（如前端忽略该码），任务将永久 `processing`。

### 存疑项
- **P-网1（已知限制，非 Bug）**：`_TASK_META` 为内存态（`main.py:L48`），网关进程重启后所有 `task_id` 失效，`GET /v1/tasks/{id}` 将 `_TASK_META.get` 返回空 meta（`main.py:L789`），轮询无法恢复。与 localTool 的 WASM 持久化形成对比，是架构上「网关无状态恢复」缺口。
- **P-网2（边界脆弱）**：`done` 去抖窗口硬编码 5s（`main.py:L805`），且 `poll_count >= 7` 才主动探测 `pending_confirmation`（`main.py:L812`）。若前端轮询间隔 > 5s，可能错过 `done` 去抖窗口，但终态仍会返回（仅延迟），不致命。
- **P-网3（与模块3 红线的交联）**：网关 `lovart_to_apimart` 只回传 Lovart 原 URL；模块3 `App.js:L33051` catch 在 localTool 断连时回退该 URL 作最终结果 → 🔴 红线（CLAUDE.md §3：不得把 CDN URL 当最终结果）。根因在 App.js 侧，网关侧无法单方面消除。详见 `99_审计总报告.md` 红线冲突回填项。

### 与已验证结论的闭合
- 模块3 的 `pending_confirmation` 处理（`App.js:L32959/L33061`）↔ 网关 `main.py:L823/L846` 双向对齐 ✅。
- 模块3 轮询 `GET /v1/tasks/{id}`（`App.js:L33005`）↔ 网关 `main.py:L873` 接口签名对齐 ✅。
- 门4（30_）确认的 `resourceAdded` 派发源 `background.ts:L102` 与本模块 webhook（`main.py:L369`）为两条独立通知通道，互不冲突 ✅。
