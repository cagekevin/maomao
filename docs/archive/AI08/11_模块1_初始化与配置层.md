## ① 运转图景

模块1 是**系统启动握手层**：决定 token、拼出所有后端请求地址、桥接双服务（localTool `:18080` 与 AI 网关 `:9004`）。本地模式下免登录，靠 `config.js` 的 `USE_LOCAL_ENGINE` 开关切换数据面。

**权威配置源 = `config.js`**（非 App.js 内硬编码）：
- `USE_LOCAL_ENGINE = true`（config.js:L36）→ 走 localTool `:18080`
- `LOCAL_ENGINE.base = http://127.0.0.1:18080`（config.js:L12-18）
- `DEFAULT_ENDPOINT = http://127.0.0.1:9004`（网关，config.js:L30）
- `ENDPOINTS = [{label:'API网关', url:'http://127.0.0.1:9004'}]`（config.js:L25-27）
- `localEngineBase()`（config.js:L42）= `USE_LOCAL_ENGINE ? LOCAL_ENGINE.base : REMOTE_BASE`

> ⚠️ **关键割裂**：App.js 内 `Hr`/`vv`(var-mapping L117/L118) 是**硬编码字符串** `'http://127.0.0.1:18080'`，并**未引用 `config.js` 的 `LOCAL_ENGINE.base`**。这坐实 TASKS P0「host 硬编码 18080」且 `MAOMAO_DATA_DIR` 切换不生效——config 层的 base 与 App 层实际用址脱钩。

## ③ 关键数据流（带 file:line）

### 流 A：启动鉴权（本地模式免登录）
- `Oa()`(L3543) 返回 `localStorage.getItem(Ca) || 'local-mode-token'`——**永远有 token**，故所有 `Jh()`(L30712) 注入的 `Authorization` 均为固定本地 token。
- `ka`(L3547)/`Aa`(L3552) 写/删 token 同时落 `Q.setConfig/remove`（`StorageManager`，红线 §3.3 不得绕过）。

### 流 B：API URL 解析（官方/网关 base）
- `tr(e)`(L846)：规范化 base + `/api` 后缀（去空白/去尾斜杠）。
- `ar(e,t)`(L863)：`resolveApiUrl`——当 `t||!e` 返回官方 `${$n}/api`；否则做 localhost 回退：若 `e` 去 `/api` 后是 `127.0.0.1:3000` 且官方 base 非 localhost → 用官方，否则用 `e`。（本地模式此逻辑基本走官方分支）。
- `cr(e)`(L877)：`resolveAssetUrl`——`http` 直链透传，否则拼 `${$n}${er}${e}`。

### 流 C：配置桥接双服务（**P0 根因**）
- localTool 用址：`Hr`/`vv` = 硬编码 `'http://127.0.0.1:18080'`（var-mapping），贯穿 `Xr`(L1812)/`ri`(L1871)/`Sv`(L42838)/`wv`(L42857)/`Ev`(L42883)。
- 网关用址：`DEFAULT_ENDPOINT='http://127.0.0.1:9004'`(config.js:L30) / `ENDPOINTS`(L25)。
- **割裂点**：App.js 的 `Hr`/`vv` 不经 `config.js` 的 `LOCAL_ENGINE.base`，故即便改 `config.js` 的 host/port，前端仍打硬编码 18080 → TASKS P0「host 硬编码」坐实；`MAOMAO_DATA_DIR` 切目录若只改 localTool 侧、前端 base 不变，URL 前缀不一致会破图。

### 流 D：画布状态 key
- `Cr(e)`(L1294) = `${Z.CANVAS_STATE_PREFIX}${e}`，`Z.CANVAS_STATE_PREFIX='canvas-state-v1-'`(var-mapping L21)。由 `StorageManager`(`Q`) 编排存 `window.localTool`/`chrome.storage`。
