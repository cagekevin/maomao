## ① 运转图景

两条路由协同构成「磁盘文件 ↔ DB 资源」双轨：

- **`files.ts`**：纯磁盘操作。上传（`handleUpload` L14，FormData/JSON 双形态）→ 落 `~/.yimao-localtool/uploads/<subfolder>/`（默认 `canvas`，`saveFile` L104）；读取（`handleRead` L145，支持 `X-Proxy-Url` 代理读 L152/L189）；缩略图（`handleThumbnail` L224 / `tryGenerateThumbnail` L121）；目录操作（`mkdir` L261 / `move` L277 / `open` L297 / `open-dir` L317 / `list` L345）。
- **`resources.ts`**：DB 结构化存储。`rescan`（`handleResourcesRescan` L37）扫描磁盘同步进 `resources` 表；`get`/`save`/`batch-save`/`delete`/`clear` 对应前端 `Sv`/`wv`/`Tv`/`Ev`（模块2）。

前端对接（已坐实）：`Xr`(`App.js:L1802`)→`POST /api/files/upload`；`Zr`(`App.js:L1827`)→ 同路由 `fileUrl` 形态；`Sv`(`App.js:L42838`)→`POST /api/resources/save`；`wv`(`App.js:L42857`)→`POST /api/resources/delete`；`Tv`(`App.js:L42866`)→`POST /api/resources/clear`；`Ev`(`App.js:L42883`)→`POST /api/resources/rescan`。路由注册见模块6 `index.ts:L135-L197`。

## ③ 关键数据流（带 file:line）

### 流 A：前端上传落盘（模块2 `Xr`/`Zr` 服务端落地）
- `Xr`(`App.js:L1802`) → `POST /api/files/upload` → `handleUpload`（`localTool/src/routes/files.ts:L14`）。
- **FormData 形态**（`localTool/src/routes/files.ts:L25`）：`file` 字段直接 `saveFile`（L37）；`fileUrl` 字段 `fetch` 下载后 `saveFile`（L48-L57）。→ 这就是 `Zr`(`App.js:L1827` uploadFileByUrl) 的服务端归宿。
- **JSON 形态**（`localTool/src/routes/files.ts:L74`）：仅接受 `fileUrl`（L76），下载落盘（L84-L90）。
- 落盘名 `Date.now()-safeName`（`localTool/src/routes/files.ts:L112`）防重名；`sanitizeFilename`（L117）净化非法字符。
- 上传后 `tryGenerateThumbnail`（L39/L59/L92）生成伪缩略图 → `thumbnailUrl` 返回前端。

### 流 B：资源入库/删除/清空（模块2 `Sv`/`wv`/`Tv` 服务端落地）
- `Sv`(`App.js:L42838`) → `handleResourcesSave`（`localTool/src/routes/resources.ts:L181`）：缺 `id` → 400（L183）；`upsertResource`（L186）DELETE+INSERT。
- `wv`(`App.js:L42857`) → `handleResourcesDelete`（`localTool/src/routes/resources.ts:L202`）：仅 `DELETE FROM resources WHERE id=?`（L207），**无 `deleteFiles` 参数** → 磁盘孤儿。
- `Tv`(`App.js:L42866`) → `handleResourcesClear`（`localTool/src/routes/resources.ts:L211`）：`body.folder` + `deleteFiles`（L215-L220）或全清（L222-L236）；全清时遍历子目录删文件（L227-L233），但**不删子目录自身**（仅 `unlinkSync`，L230）。

### 流 C：rescan 双轨同步 + 孤儿清理（模块2 `Ev` 服务端落地）
- `Ev`(`App.js:L42883`) → `handleResourcesRescan`（`localTool/src/routes/resources.ts:L37`）：遍历 upload 子目录（L48-L53，跳过 `.thumbnails`），文件/文件夹录入 `resources`（`source='local-tool'`，L79/L110）。
- **孤儿清理**（`localTool/src/routes/resources.ts:L120-L130`）：仅查 `source='local-tool'`（L123），磁盘不存在则删（L126-L128）。→ **门4 P2 #7 坐实**：`source='extension'` 资源不在此清理范围（设计上 extension 瞬态，见 31_）。
- `toAbsoluteFileUrl`（L31）补 `http://127.0.0.1:18080`（L30），解决 chrome-extension:// 页面 `<img src="/files/...">` 破图（注释 L27-L29）。

### 流 D：缩略图伪复制（门4 P2 #8 坐实）
- `tryGenerateThumbnail`（`localTool/src/routes/files.ts:L121`）：仅图片扩展名（L123-L124）生成；实际 `fs.copyFileSync(filePath, thumbPath)`（L137）**复制原图**当缩略图，注释预留 sharp（L135）。`handleThumbnail`（L224）同逻辑（L253）。
- → **门4 P2 #8 双向坐实**：前端 `ri`(`App.js:L1856`) 仅取 URL，真实缩放缺失在 `files.ts` 这两处。

## ④ 存疑 Bug / 边界契约（对照 P0–P2）

### 边界契约
1. 上传文件名 `Date.now()-safeName`（`localTool/src/routes/files.ts:L112`）防重名，但**不同子目录可同名**（id 不全局唯一，仅 `local-<folder>-<name>` 在 rescan 内唯一，`localTool/src/routes/resources.ts:L69/L96`）。
2. `handleResourcesClear` 全清（`localTool/src/routes/resources.ts:L222`）删文件但不删子目录（`L230` 仅 `unlinkSync`），残留空目录。
3. `toAbsoluteFileUrl`（L31）硬编码 `LOCAL_TOOL_BASE=http://127.0.0.1:18080`（L30）→ 与模块1 P0（18080 硬编码）同源，改端口需同步。

### 存疑项
- **P2 #6（已坐实，建议修复）**：`handleResourcesDelete`（`localTool/src/routes/resources.ts:L202-L208`）单条删除无 `deleteFiles` → 对应前端 `wv`(`App.js:L42857`) 只删 DB。磁盘孤儿累积。修复：加可选 `deleteFiles`，调 `fs.rmSync`。（门4 已双向坐实）
- **P2 #7（已坐实，设计特性）**：`handleResourcesRescan` 孤儿清理仅 `source='local-tool'`（`localTool/src/routes/resources.ts:L123`）。extension 源不清理属有意（见 31_ 事件总线）。非 bug，文档明确即可。
- **P2 #8（已坐实，建议修复）**：`localTool/src/routes/files.ts:L137/L253` `copyFileSync` 伪缩略图。建议接 sharp 真实缩放。
- **P-本8（命令注入低风险）**：`handleOpen`/`handleOpenDir`（`localTool/src/routes/files.ts:L297/L317`）`execSync(\`${cmd} "${dirPath}"\`)`（L308/L336），`cmd` 固定 `explorer`/`open`，`dirPath` 来自 `getUploadDir()` 内部拼接（非外部 HTTP 入参）→ 无外部注入面，但 `dirPath` 若含引号理论上可断 cmd；建议用 `execFileSync(cmd,[dirPath])` 规避 shell 解析。
- **P-本9（clear 不删子目录）**：`handleResourcesClear` 全清（L222-L236）只 `unlinkSync` 文件（L230），不删空子目录 → 长期运行残留空目录，无功能危害。
- **P-本10（filename 未校验路径穿越）**：`handleUpload` 的 `filename`（L29/L81）经 `sanitizeFilename`（L117）去 `/` 等，但 `subfolder`（L28/L80）**未净化** → 若前端传 `subfolder="../x"` 可越出 uploadDir（`saveFile` L106 `path.join` 会解析到父级）。属**潜在路径穿越写入**风险，建议对 `subfolder` 也做 `sanitizeFilename` 或白名单。
