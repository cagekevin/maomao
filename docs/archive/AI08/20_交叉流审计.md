## 交叉流 1：AI 生成 → 落盘 → 资源面板刷新（主线，红线 §3.2）

追踪目标：验证"生图结果必须落盘到 18080 再通知面板"，而非直接拿 CDN 直链。

### 逐跳（file:line）

1. **完成信号**：任务完成 `i === 'completed'` → 调 `Ev().then(()=>Di.current&&Di.current())` 触发后端 rescan（`App.js:L31426`）。该监听挂在 `mutiwindow-task-completed`（`App.js:L31428` 监听、`L38481`/`L43640`/`L43676`/`L43697`/`L44406` 派发）。
2. **落盘**：生图/视频结果 URL 经 `ii(u,{subfolder:'tasks',preferThumbnail:true,...})` 落盘（`App.js:L33049`、`L33098`、`L35863`、`L36081`、`L36363` 等）。`ii`(L1888) → `Xr`(L1802) → POST `${Hr}/api/files/upload`（L1812）返回绝对路径 `{url,thumbnailUrl,path}`。
3. **入库/通知**：资源经 `Sv({...})`(L42838 → `/api/resources/save`) 写库；同时 `resourceAdded` 事件被监听（`App.js:L43527`）→ 更新面板 state + `await Sv(...)`（L43552）。
4. **面板刷新**：监听处 `D(e=>[i,...e])`、`G(e=>e+1)`（L43552 区）刷新资源列表。

### 门4 对抗质询（反向）
- Q：步骤2 的 `ii()` 若失败（localTool 未连），`L33051` 有 `catch` 兜底"使用原始 URL"——**这意味着 CDN 直链会被当作最终结果回退**，违反红线 §3.2「严禁 CDN 直链当最终结果」。⚠️ **坐实红线冲突风险**：localTool 断连时生图结果用网关 CDN URL，刷新/离线即丢。
- Q：`resourceAdded` 在 L43527 监听，但**谁 dispatch 它**？grep 未见前端 `dispatchEvent(new CustomEvent('resourceAdded'))`，仅见 `Zn`(=canvasAddResourceEvent, L213) 与 `mutiwindow-task-completed`。→ `resourceAdded` 可能由 **localTool/localTool 推送**或**另一处**派发，前端侧 grep 未捕获 → 🔜 待 locate 派发源（门4 跨审）。
