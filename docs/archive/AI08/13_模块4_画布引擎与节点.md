## ① 运转图景

模块4 是**画布表现层**：ReactFlow 承载节点图，处理拖拽落点、连线、节点点击/右键菜单、视口变换。节点组件（ImageNode/PromptNode/VideoToGifNode 等，var-mapping §节点组件）由 `nodeTypes: lg` 注册。3D 导演台通过注册式 handler（`registerModelGenerateHandler`/`registerViewportCaptureHandler`）接入，非直接内联。

## ③ 关键数据流（带 file:line）

### 流 A：拖入资源落到画布
- `onDrop: Lr`(L37138) → 定义 `Lr = Y.useCallback(e=>{...})`(L36293)：处理从资源面板/文件拖入，建节点。

### 流 B：连线建立
- `onConnect: xn`(L37125) → 定义 `xn`(L31883)：建立边 `edges` 更新（标准 ReactFlow onConnect 模式）。

### 流 C：节点交互
- 点击 `kn`(L31980) / 画布右键 `Cn`(L31933) / 节点右键 `wn`(L31947) / 连线结束 `Fr`(L36245)。
- 这些回调通过**事件总线**（`window` CustomEvent，见 `20_交叉流审计.md` 交叉流4）通知其他模块，不直接绕过 `StorageManager`(=Q)。

### 流 D：3D 导演台接入（注册式，非内联）
- `kp(e){ Op = e }`(L26080) 注册模型生成 handler；`wd(e){ Cd = e }`(L22986) 注册视口捕获 handler。
- 调用方：`Ed(e){ return Cd(e) }`(L22992) 调视口捕获；`Op` 由导演台组件调用生成。

### 流 E：节点类型注册
- `nodeTypes: lg`(L37127) / `edgeTypes: ug`(L37128)：`lg`/`ug` 为节点/连线类型 Map，含 var-mapping 列出的 `li`(ImageNode)/`Ya`(PromptNode)/`Qa`(TextNode)/`ps`(AudioNode)/`Gs`(VideoToGifNode)/`Lc`(GridSplitNode)/`Jc`(PanoramaNode) 等。
