## ① 运转图景

模块3 是 AI 任务的**派发与异步收口层**。前端把生图/视频/聊天请求发往网关 `:9004`（OpenAI 风格），拿到 `task_id` 后走轮询机制（红线 §3.2 #6），结果 URL 经 `ii()` 落盘到 localTool `:18080`，再经事件总线通知资源面板。

网关侧已知限制（红线 §3.3 #8 噪音，非 bug）：内存任务库重启即丢、未实现 API 404 报错、`pending_confirmation` 卡轮询（AUTO_CONFIRM=false 时）。

## ③ 关键数据流（带 file:line）

### 流 A：生图派发 → 网关 → task_id 检测
- 入口：生图主回调 `let Jn = Y.useCallback(async(e,r,o,...)=>...)`（L32490）。
- 同步解析分支（无 task_id）：`t.data?.[0]||t.images?.[0]||t.output?.[0]`（L33077）→ `b64_json`/`url`。
- **异步分支（有 task_id）**：`taskId = t.data?.[0]?.task_id||...`（L32988）→ 进入轮询。

### 流 B：轮询机制（红线 §3.2 #6）
- `AbortController`（L32993，`ht.current.set(n,ac)`）。
- `while(true)` 轮询 `GET ${R}/v1/tasks/${taskId}`（L33005），带 `Authorization: Bearer ${h}` 与 `localPort`（localTool 代理，L33010）。
- 自建 deadline 15min（L33097）：超时抛 `生成超时（15分钟）`。
- 429 退避：`backoff = min(backoff*1.5, 15e3)`（L33067）。
- `pending_confirmation`：L33059 `resolveNeedsConfirm` → L33061 `markNeedsConfirm(z,cid,'image',R,h)` 节点置 `await_confirm`（对应 TASKS "单任务卡轮询"噪音）。

### 流 C：结果落盘（红线 §3.2 #6，★ 红线冲突点）
- 轮询成功拿到 `u`（结果 URL）：`let persisted = await ii(u,{subfolder:'tasks',preferThumbnail:true,thumbMaxDim:480,thumbQuality:75})`（L33049）。
- `ii` → `Xr`(L1802) → POST `${Hr}/api/files/upload`(L1812) 落盘 18080，返回绝对路径。
- **⚠️ 红线冲突**：L33051 `catch(persistErr){ console.warn('[生图调试] ii() 持久化失败，使用原始 URL:', persistErr); }` —— localTool 断连时**回退使用网关 CDN 直链 `u`**，违反红线 §3.2「严禁直接使用 CDN URL 当最终结果」。刷新/离线即丢、破坏本地模式闭环。

### 流 D：完成 → 触发 rescan（接缝到模块2/交叉流1）
- 任务完成 `i==='completed'` → `Ev().then(()=>Di.current&&Di.current())` 触发后端 rescan（L31426，监听 `mutiwindow-task-completed` L31428）。
