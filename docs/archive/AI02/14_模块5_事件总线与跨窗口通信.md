## 一、模块职责

前端扩展在"多窗口 + 插件后台 + 本地端"三种形态下，靠两套机制解耦通信：

1. **页面内事件总线**：`window.CustomEvent`（同窗口跨组件广播）。
2. **扩展运行时通道**：`chrome.runtime` / `chrome.tabs` / `chrome.storage.local`（跨窗口、跨后台、跨端）。

本模块坐实两套机制的事件名常量、派发点、监听点，形成"边界契约表"。

## 四、边界契约小结（供门4 质询）

1. **页面内广播**：`Zn` / `na` / `Va` 三个常量事件 + 7 个 `mutiwindow-*` 字面量事件构成前端内部契约。
2. **跨窗口**：`mutiwindow-*` 系列靠 `localStorage`（`mutiwindow-clipboard`）+ `postMessage` 跨窗口传数据，`CustomEvent` 仅同窗口广播——**跨窗口靠 storage 不靠 CustomEvent**。
3. **跨端判定**：`chrome.runtime.id` 存在 = 插件端；否则本地端（L43490–L43491）。这是 config 开关 `USE_LOCAL_ENGINE` 在运行时的实际生效判定点（比 config.js 默认值更权威）。
4. **重复风险**：`mutiwindow-task-completed` 五派发 / 两监听，是 P2-黄 重复 rescan 的根因级证据（需运行时确认去重逻辑）。
