## 一、运转图景

模块1 是应用启动的「地基」：加载 `config.js` 配置 → 初始化 `StorageManager`(Q) 的多后端（localTool/wr、chromeStorage/Mr、localStorage/Nr、localforage/Pr）→ 解析 API 基址（`ar`/`tr`）→ 鉴权（`Oa`/`Jh`/`Ba`）→ 恢复画布状态（`Cr`）。

关键：配置开关 `USE_LOCAL_ENGINE`(config.js L36) 决定**文件/KV 走 localTool :18080**，AI 请求走网关 :9004。`config.js` 是**唯一可改配置层**（红线§3.1），本模块只描述现状不改动。

## 三、关键数据流

### 3.1 启动初始化（bootstrap）
```
src/main.tsx → lazy App.js
  → config.js 加载（USE_LOCAL_ENGINE=true）
  → StorageManager(Q) 初始化后端:
      wr(localTool) / Mr(chromeStorage) / Nr(localStorage) / Pr(localforage)
  → 检查 localTool 连接（Hr=http://127.0.0.1:18080 轮询）
  → 恢复画布: Cr(projectId) → localforage 读 canvas-state-v1-{id}
  → 同步: syncAllToLocalTool（见模块3/交叉流）
```

### 3.2 API 基址解析
```
ar(base, forceLocal)          App.js L863
  → tr($n) 规范化官方 base
  → forceLocal 或 !base → 返回 tr($n)
  → 否则按 localhost:3000 / 127 规则选 n 或 r
tr(e)                         L846  补 /api 后缀
cr(e)                         L877  非 http 补 $n+er
```

### 3.3 鉴权（本地模式）
```
Oa()  L3543 → 返回 localStorage[Ca] || 'local-mode-token'
Jh()  L30712 → {Authorization: 'Bearer '+Oa()} 或 {}
Ba()  L3637 → !!Oa() 恒 true（本地模式）
```
> 去登录改造（`PROJECT_ORIGIN.md` §4.5）：`Oa()` 永远返回本地 token，`isLoggedIn`(Ne) 恒 true。

### 3.4 画布状态键
```
Cr(projectId)  L1294 → `${Z.CANVAS_STATE_PREFIX}${projectId}`
  Z.CANVAS_STATE_PREFIX = 'canvas-state-v1-'（var-mapping L21）
  → localforage key：canvas-state-v1-{projectId}
```
