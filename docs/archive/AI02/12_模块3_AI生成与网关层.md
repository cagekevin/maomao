## 一、运转图景

模块3 是画布「生图/生视频/生音频」的核心编排：节点触发生成 → 前端调网关 `:9004`（OpenAI 风格）→ Lovart 异步任务 → 前端轮询 `GET /v1/tasks/{id}` → 结果 `ii()` 落盘到 localTool `:18080/uploads/tasks/` → 事件总线刷新资源面板。

**关键澄清**：生图主回调是**组件内局部变量** `let Jn = Y.useCallback(...)`(L32490)，它遮蔽了模块级 `Jn`(L89=LogoIcon)。任何"生图派发"引用必须带 L32490。

## 三、关键数据流

### 3.1 生图主流程（同步 SSE / 异步轮询双分支）
```
节点 onGenerate → Jn(nodeId, prompt, size, model, category, d)   App.js L32490
  → 权限校验 Xi/Ai                                   L32493-32506
  → 收集上游 images/videos/text + selectedContextResources   L32532-32548
  → 调网关:
      分支A 流式(SSE): 解析 progress/succeeded        L32910-32945（rawResp 落 L32948）
      分支B 异步(task_id):
        task_id 检测                               L32988
        while 轮询:
          GET ${R}/v1/tasks/${taskId}              L33005（≥3s, 退避, 15min deadline L33003）
          status==='completed' → 提取 images[0].url[0]  L33028-33033（注意数组[0]）
          status==='failed' → throw                L33055
          else → resolveNeedsConfirm → 可能置 await_confirm 退出轮询  L33057-33059
  → ii(u,{subfolder:'tasks'}) 落盘 18080           L33049
  → 更新节点 data.imageUrl/videoUrl
  → dispatchEvent('mutiwindow-task-completed')     L44406（统一同步 effect 内）
```

### 3.2 统一同步 effect（防死循环·本地化判定）
```
依赖 [globalTasks, isConnected, port]
遍历 globalTasks(status=completed):
  本地化判定(L44369):
    if url.startsWith('/files/') 
    || (startsWith('http://127.0.0.1') && /\/files\/tasks/.test(url))
       → 跳过（已本地化）
    else → n.uploadFile(blob, name, 'tasks') 落盘 18080   L44377
  有变更 → ev(i,t) 持久化 + dispatchEvent('mutiwindow-task-completed')  L44402-44406
```
> 死循环修复（`PROJECT_LOG` 2026-07-20 13:07 `cd3c0aa`）：补 `/files/` 相对路径检测，已本地化任务不再反复重传。

### 3.3 pending_confirmation 卡死（存疑 Bug 根因）
```
轮询返回 status 非 completed/failed（如 pending_confirmation）
  → resolveNeedsConfirm(taskInfo) 返回 taskId        L32957
  → markNeedsConfirm → 节点置 await_confirm          L32980-32984
  → break 退出轮询（不自动重试）
若 AUTO_CONFIRM=false 且无手动确认 UI 触发 makeConfirmAdapter → 任务永久卡 await_confirm
```
**确认 adapter 已写好**（L32960 `makeConfirmAdapter` → POST `/v1/tasks/{id}/confirm`），但需要**手动触发**；网关 `AUTO_CONFIRM=false` 时无人自动调。

| 路由 | 方法 | 调用方(行号) | 说明 |

- Lovart 密钥 `LOVART_ACCESS_KEY/SECRET_KEY`：网关侧，前端不可见。
