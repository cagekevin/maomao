## 二、`Xr` / `Zr` 被误当上传/下载函数

**真相（经函数体复核修正）**：
- `Xr` 是**双重定义**：
  - **模块级 `async function Xr(e,t={})`（L1802）** = **上传落盘实现**（内部 L1812 `fetch(\`${Hr}/api/files/upload\`)`，名为 openInTab 误导但实为 uploadToLocalTool）；`ii`(L1888) 在 L1898 调用它完成文件上传。✅ 这是真实上传环节。
  - **组件内 `Xr = async e=>`（L43881）** = **openInTab**（在活动标签页打开 URL），非上传。
  - 原 ARCHITECTURE/TASKS 把"上传"误归到模块级 Xr 的语义上虽不准确（模块级 Xr 名为 openInTab 实为上传），但**上传链路确实经过模块级 Xr(L1802)**；不能据此说"Xr 非上传"。准确表述：上传 = `ii`(L1888) → 内部 `Xr`(L1802) → POST /api/files/upload。
- `Zr`：组件内 `Zr = async () =>`（L43893）= **注销/重置配置（logout 改文案）**，非下载。⚠️ 但需注意：`Zr` 又是**双重定义**——另有 `async function Zr(e,t={})`（L1827）= `uploadRemoteUrlToLocalTool`（远程 URL 上传落盘，L1832 同样 POST /api/files/upload）。即模块级 `Zr`(L1827) 是**远程URL上传**，组件内 `Zr`(L43893) 是注销。原文档把 `Zr` 一律当"下载函数"是错的；模块级 Zr 实为上传（远程URL），组件内 Zr 为注销。
- 真实落盘上传入口：`ii`(L1888，本地文件) 经 `Xr`(L1802)；远程 URL 落盘走 `Zr`(L1827)。

**修正点（ARCHITECTURE.md）**：
- L211：右键"发送到资源" `Zr()→POST /api/files/upload` → `ii()(L1888)→Xr()(L1802)→POST /api/files/upload`（模块级 Xr 实为上传实现；组件内 Zr 非此路径）
- L212：AI 生成结果 `uploadFile()→Xr()/Zr()` → `uploadFile()→ii()(L1888)→Xr()(L1802)`（模块级 Xr 是上传；组件内 Xr/Zr 非上传环节）
- L215：资源面板文件上传 `R(files)→ii()→Xr()` → `R(files)→ii()(L1888)→Xr()(L1802)`（模块级 Xr 为上传实现）
- L353：资源采集 `Zr() 下载 → uploads/migrated/` → `ii()(L1888)→Xr()(L1802) 落盘 → uploads/migrated/`
- L388：rescan URL 格式 `Zr() 返回相对路径` → `ii()(L1888)/Xr()(L1802) 返回相对路径`（组件内 Zr 非上传函数）

**修正点（TASKS.md）**：
- L34：`拖入 URL 时调 Zr() 下载` → `拖入 URL 时调 ii()(L1888) 落盘`（本地文件）或 `Zr()(L1827) 远程URL上传`（远程URL）；`Zr` 模块级实为上传非下载
- L15 `R 回调`、L35 `R(files)` 保留（R 为组件内文件上传回调，L29165，语义正确，仅行号校正）
