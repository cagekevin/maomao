## 一、运转图景

ReactFlow 画布承载节点（promptNode/imageNode/videoNode/...）+ 连线。用户通过右键菜单加节点、拖放建节点、连线串联工作流、框选/多连模式批量操作。节点数据经 `Z`=StorageManager 持久化（localforage，`Cr` 构造键）。3D 导演台（`Th`=Director3DNode）是特殊节点，内嵌 Three.js 场景。

本模块是「用户直接操作的 surface」，改动风险高（自定义节点/连线规则与混淆逻辑冲突面大，见审计计划排雷点）。

## 三、关键数据流

### 流1：拖放建节点
```
onDrop(Lr, L36243):
  dataTransfer.files → ii({subfolder:'canvas/drop'}) → POST /api/files/upload (落盘)
  application/x-maomao-template → 批量导入节点
  application/x-mutiwindow-task → 创建节点
  → W(节点数组) 追加
```

### 流2：连线
```
onConnect(xn, L31883):
  同侧校验(a===o → 提示) → 反向纠正 → 选中批量连
  → G(边数组) 更新
onConnectEnd(Fr, L36245): 落空 → ghost-target/ghost-edge + connection 菜单
```

### 流3：右键菜单
```
onPaneContextMenu(Cn)/onNodeContextMenu(wn)/onSelectionContextMenu(En)
  → ve({x,y,type,nodeId?,connection?}) 设 K(contextMenu)
  → 菜单渲染(L37597)
```

### 流4：持久化
```
画布变更 → Q.saveCanvasState(L1642) 直接写 localforage（Cr 构造键），无 canvas-state-change 事件
  → Cr(projectId)(L1294) 构造键 'canvas-state-v1-{projectId}'
  → Pr(localforageBackend, L1446)
```
