# T2.9 · 资源面板上传/拖入/粘贴未落盘（P1 根因，_DRAFT · 未验证）

> 续 `README_审计索引.md` 开放问题6（P1 资源面板 `z`(L29159) 补 `Sv()`/`rescanThrottledSync()`）+ 闭合 T2.2 P1 缺陷真因。
> 目标：坐实资源面板上传路径是否落盘，对比画布上传路径，定位 P1 真因。
> 行号快照 2026-07-21，会漂移，以 grep 为准。归属 V1。未改任何源码。

---

## 结论（高置信）

**P1 根因已坐实：资源面板的「上传/拖入/粘贴」图片只经 `FileReader` 读成 base64 dataURL 塞进面板 state（`O`），全程未调用 `ii`（落盘 localTool）/`Sv`（资源库 save）/`Ev`（rescan 入库）。**
即这些图片仅活在内存面板里，**不落盘、不进资源库、刷新/重开即丢**。而画布侧的拖放/粘贴/文件节点都正确调 `ii` 落盘 —— 二者行为不一致，是 P1 缺陷的真凶（与 T2.2 已记 P1 一致）。

---

## 一、资源面板上传路径（不落盘）

组件局部（资源面板区，`App.js:L29148` 起）：
- `R`(L29149) = `useCallback(async e => { ... new FileReader(); n.readAsDataURL(e); ... })` —— **仅读成 base64 dataURL，无 `ii`**
- `z`(L29159) = `useCallback(async e => { let t = await R(e); t.length !== 0 && O(t.map(e=>({...e, source:'upload'}))) })` —— 仅 `O`（面板 setState）
- `B`(L29188) = 拖放 handler：`await R(...)` → `O(t.map(... source:'drop'))`；URL 拖入直接 `O([{url:t, source:'drop'}])` —— **无 `ii`/`Sv`**
- paste(L29168) = `window.addEventListener('paste')` → `O((await R(n)).map(... source:'paste'))` 或 URL 直 `O(...)` —— **无 `ii`/`Sv`**

`O` 是资源面板列表的 setState（非落盘函数）。全资源面板区 grep `ii(`/`Sv(`/`Ev(`：**0 命中**（见下节对比）。

---

## 二、画布上传路径（正确落盘，对比）

同样"选择文件→读→落盘"，画布侧每个入口都调 `ii`：
- `Lr` onDrop(L36363)：`await ii(e, {subfolder:'canvas/drop', preferThumbnail:l})`
- `onPaste`(L36081)：`await ii(e, {subfolder:'canvas/paste', preferThumbnail:true})`
- 文件/图片节点(L35863)：`await ii(e, {subfolder:'canvas/upload', preferThumbnail:...})`
- `ImageNode`(L4245)：`await ii(r, {subfolder:'canvas/upload', preferThumbnail:true})`
- 生图完成(L31426 `mutiwindow-task-completed`)：`Ev().then(...)` rescan 统一入库

**即：画布侧有 `ii`+`Ev` 双保险，资源面板侧全缺。**

---

## 三、资源库 save 调用点（仅在迁移/外部路径，非上传）

- `Sv`(L42838) = `fetch(`${vv}/api/resources/save`)`（资源库 save，vv=LOCAL_ENGINE.base=18080）
- grep `Sv(` 命中：L5365/L8773（旧资源迁移 `folder:'migrated'`）、L42852 `Cv`（收藏）、L5367 等 —— **均非资源面板上传入口**
- `rescanThrottledSync`(L43028) 定义在资源面板区（L43028），但 `z`/`B`/paste 均未调用它

---

## 四、死锁/丢失状态机（P1 表现）

```
用户从资源面板：上传文件 / 拖入 / 粘贴图片
  → R(FileReader) → base64 dataURL
  → O() 塞面板内存列表
  → ❌ 无 ii()（不落盘 18080）
  → ❌ 无 Sv()（不进资源库）
  → ❌ 无 Ev()/rescanThrottledSync()（不入库）
  → 刷新页面 / 重开画布 → 这些图片从面板消失（仅内存）
```

---

## 五、修复建议（仅提方案，不执行；修须单独立项 + 红线 §3.3 第10条首句确认）

1. **最小修复**：在 `z`(L29159)/`B`(L29188)/paste(L29168) 写入 `O` 前，对 dataURL/URL 调 `ii(..., {subfolder:'canvas/upload'})` 落盘，再用返回的 http URL 塞 `O`；上传成功后调 `rescanThrottledSync()` 或 `Ev()` 触发资源库入库。
2. **统一封装**：抽一个 `uploadToResourcePanel(file)` 复用画布侧的 `ii` 逻辑，避免两套路径分裂（根治"画布落盘/面板不落盘"不一致）。
3. **URL 拖入**（L29181–29184 / L29203–29207）：当前直 `O([{url:t}])`，若是远程 http URL 可不经 `ii`（直接引用），但若是本地 blob/data 仍需 `ii` 落盘。建议按 URL 类型分支。
