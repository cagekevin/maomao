## X3. 配置加载与双服务 base URL 桥接（T2.1 核心 + X3 跨层风险）

```
config.js:
  USE_LOCAL_ENGINE=true (L36) → localEngineBase()=LOCAL_ENGINE.base (L42) = http://127.0.0.1:18080
  DEFAULT_ENDPOINT=9004 (L30) = 网关
App.js:
  Hr=localEngineBase() (L1732) → 文件/资源请求基址(18080)
  vv=LOCAL_ENGINE.base (L42808) → 资源请求基址(18080)
  ot='http://127.0.0.1:9004' (L43084) → 网关 base
  ar()(L863) resolveApiUrl: localhost:3000 + 官方非本地 → 回退网关；否则用传入址
```
**接缝**：`USE_LOCAL_ENGINE`(config.js L36) 总开关 · `Hr`/`vv`(18080) vs `ot`(9004)
**跨层风险(X3.3)**：`USE_LOCAL_ENGINE=false` 时 `Hr`=REMOTE_BASE(9004) 但文件上传路由 `/api/files/upload` 只存在于 localTool(18080) → 文件请求打 9004 无路由 → 失败。P2「统一 base URL」根因。当前 USE_LOCAL_ENGINE=true 不触发。
**残留**：host 硬编码 18080（P0 残留）、中文目录/文件名 Latin1 乱码（P0 残留，sql.js 存中文）。
