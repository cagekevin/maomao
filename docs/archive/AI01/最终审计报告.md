## 五、坐实的缺陷（Bug）清单

### P0（红线禁区残留，非本次范围）
- host 硬编码 18080、中文目录/文件名 Latin1 乱码（sql.js 存中文）—— T2.5/T2.1 记录，修复需动红线禁区，须单独立项。

### P1（功能缺陷，须立项 + 红线 §3.3 第10条确认）
1. **资源面板上传/拖入/粘贴未落盘**（T2.9）：`z`(L29159)/`B`(L29188)/paste(L29168) 仅 `O(...)` 塞内存 + `R`(L29149) FileReader→base64，**全程无 `ii`/`Sv`/`Ev`**，刷新即丢。对比画布侧 `Lr`/`onPaste`/文件节点均调 `ii` 落盘。
2. **3D 机位截图未落盘**（T2.10）：`qn`(L32370) 把截图 dataURL 直进 imageBoxNode，**未调 `ii`**，大图 dataURL 累积有 localStorage 配额风险。与 P1-1 同源（"dataURL 直进节点"背离原图分离设计）。

### P2（脆弱点，当前不触发）
1. **双服务 URL 错配**（T2.8）：`USE_LOCAL_ENGINE=false` 时 `Hr`=9004，但 `/api/files/upload` 仅 18080 有路由 → 文件上传全失败；且 `useLocalTool`(L19102) 硬编码 18080 不受开关影响 → 撕裂态。当前默认 true 不触发。
2. **缩略图落盘失败静默**（T2.7）：`Th.f` 内 `ii` 抛错仅 `console.warn` 保留 base64。

### 独立死锁 Bug（已确认）
- **`pending_confirmation` 卡死**（T2.6）：前端 `requestConfirm`/`makeConfirmAdapter` 全文件 **0 调用**、`confirmTaskId` 等 **0 读取** → 确认 UI 完全缺失。网关返回 `error.code='pending_confirmation'` 时节点置 `await_confirm` 后轮询终止，无回调可推回 → 永久卡死。网关 `AUTO_CONFIRM=true`(L34) 仅减少触发概率，非根因。（PROJECT_LOG 已记"未修"）
