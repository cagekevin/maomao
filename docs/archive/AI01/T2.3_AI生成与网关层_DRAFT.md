## 一、运转图景

用户在 PromptNode/ImageNode/VideoNode 触发生成 → 前端 `Jn`(生图主回调, L32490) 提交到网关 `/v1/images|videos/generations` → 拿到 `task_id` → 进入**异步轮询分支**（L32990）轮询 `GET /v1/tasks/{id}` → 终态取 `result.images[].url[0]` → 经 `ii()`(L33049) 持久化落盘 18080 → 回写节点 → 发 `mutiwindow-task-completed` → Ev() rescan 刷新资源面板。

统一同步 effect(L44354) 另开一条「兜底落盘」路径：遍历 `globalTasks`(completed)，未本地化则 `uploadFile` 落盘，防死循环靠 `Se.current` 锁 + `/files/` 判定。

网关侧（`apimart-gateway/main.py`）：`/v1/images/generations`(L591)、`/v1/videos/generations`(L641)、`/v1/tasks/{id}`(L873)、字段映射 `_submit_generation`(L687)。**内存任务库，重启即丢（已知限制非 bug）**。

## 三、关键数据流

### 流1：生图提交→轮询→落盘（前端内）
```
Jn()(L32490) → 取上游图/文/节点参数
  → POST ${R}/v1/images/generations (R=9004, 端点见 L32731 `I=${R}/v1/images/generations`)
  → 响应 t: t.data?.[0]?.task_id (L32988 检测)
  → 有 task_id? 异步轮询分支(L32990):
      ac=new AbortController(); ht.current.set(n,ac) (L32993 重注册)
      deadline=Date.now()+9e5 (L32997 15min)
      loop:
        pollUrl=`${R}/v1/tasks/${taskId}` (L33005)
        pollResp=zc(pollUrl,GET,headers,signal) (L33007)
        taskInfo=pollData.data||pollData
        status==='completed' (L33028):
          imgUrl=result?.images?.[0]?.url?.[0] (L33031 ⚠️ .url[0] 数组)
          vidUrl=result?.videos?.[0]?.url?.[0] (L33032)
          u=imgUrl||vidUrl
          if u && !data: → ii(u,{subfolder:'tasks',preferThumbnail})(L33049) 持久化
          break
        status==='failed' (L33055): throw Error(taskInfo.error?.message) (no_artifact 透传 L33043)
        pending_confirmation: markNeedsConfirm → return (L33061)
        else: 429退避 / 继续 (L33067)
  → 无 task_id? 原同步解析分支 (L33076 起)
  → 回写节点 data.imageUrl/videoUrl
  → dispatch 'mutiwindow-task-completed' → Ev() rescan
```

### 流2：统一同步 effect（兜底落盘，防死循环）
```
依赖 [globalTasks(i), n.status.isConnected, n.status.port] (L44425)
if Se.current → 跳过 (L44354)
Se.current=true (L44358)
if i.length>0 && n.status.port:
  Promise.all(i.map(t=>{
    if t.status!=='completed' return t
    a(e,r,i):  // 本地化判定
      if e.startsWith('/files/') || (127.0.0.1 && /files/tasks/) → return e (已本地化,跳过) (L44369)
      if http/data/blob → n.uploadFile(a,c,'tasks') (L44377)
    t.resultUrl/thumbnailUrl/customResultData → a() 落盘
    有变更 → e=true
  }))
  e && 发 mutiwindow-task-completed (L44406) + ev(i,t) 持久化
finally: Se.current=false (L44422)
```
> 死循环修复点（PROJECT_LOG cd3c0aa）：本地化判定补上 `startsWith('/files/')`，已本地化任务不再重传。

## 五、存疑 Bug / 待确认（交后续 AI 对抗审计）

1. **7 陷阱覆盖核对**：FUNCTION_MAP §2.1 列 7 陷阱。实测代码已覆盖：task_id 检测(L32988)✅、AbortController 重注册(L32993)✅、15min deadline(L32997)✅、`.url[0]`(L33031)✅、ii() 持久化(L33049)✅、no_artifact 透传(L33043)✅、图/视频独立(L33032)✅。**待确认**：原同步分支(L33076)是否仍可能拿到裸 CDN url 未落盘（红线 §3.2 严禁裸存）？需走 L33076 起的逻辑确认。
2. **`pending_confirmation` 卡死（独立已知问题，已闭合见 `T2.6`）**：`markNeedsConfirm`(L32980) 把节点置 await_confirm 并 return，**轮询终止不落盘**。已坐实：网关默认 AUTO_CONFIRM=true（main.py L34）仅减少触发；前端 `requestConfirm`/`makeConfirmAdapter` 全文件 **0 调用**、`confirmTaskId` 等 **0 读取** → 确认 UI 完全缺失，是真实死锁 bug（PROJECT_LOG 已记"未修"）。"网关默认与前端是否一致"的疑问已解答：卡死取决于前端是否暴露确认 UI，与网关 env 无直接耦合。
3. **`R` 变量作用域污染**：Jn 作用域内 `R`=网关基址（L32490 生图回调局部）；另有多处**组件局部** `R`（资源面板区 L29149 读文件、3D 属性面板区 L23247 zoom 拖拽等），**无模块级 R 全局变量**。grep `R` 必须带行号+作用域，否则误锚。**建议后续审计把 Jn 内的 R 标注为「网关 base（局部）」**。
4. **统一同步 effect 行号漂移**：FUNCTION_MAP 记 L44246，实测统一同步 effect 在 **L44354–L44425**（L44246 是"发送到网站"逻辑）。所有引用本 effect 的文档（ARCHITECTURE X1.4 / TASKS P0）须修正行号。
5. **缩略图落盘**：L44397 `a(t.thumbnailUrl,'image','thumb')` 也走 uploadFile，但 localTool 缩略图为伪复制（模块2 P2），最终仍指向 copyFileSync 原图。
