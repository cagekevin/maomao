## 一、运转图景

前端把文件/资源持久化到 localTool(:18080)，并通过 rescan 把磁盘文件扫入 resources 表。核心动作：
- 上传落盘：`ii`(L1888) 统一入口 → 调 `Xr`(**uploadToLocalTool**, L1802) → POST `/api/files/upload` → 返回绝对路径（Xr 语义见 `符号冲突深度核查_DRAFT.md` 问题2，已坐实 L1802=uploadToLocalTool）
- 资源入库：`Sv`(L42838) → POST `/api/resources/save`
- rescan：`Ev`(L42883) → POST `/api/resources/rescan`（扫盘入库）
- 查询：`xv`(L42821) → GET `/api/resources`
- 删除：`wv`(L42857) → POST `/api/resources/delete`（**只删 DB**）
- 同步刷新：`we`(L43015) = Ev()+xv()；`rescanThrottledSync`(L43028) 3s 节流

本模块是「资源面板 / 素材 Tab / 生成结果落盘」的底座。P1（上传不入库、URL拖入不落盘）、P2（删不删磁盘、缩略图伪复制）均在此。

## 三、关键数据流

### 流1：文件上传落盘（画布拖入/粘贴/面板）
```
ii(e,{subfolder})(L1888)
  → 字符串URL? 直返 : Xr(e,{subfolder:'canvas',...})(L1802) 落盘
  → POST /api/files/upload → {url,thumbnailUrl}（绝对路径）
  → 返回前端
```
边界：返回绝对路径已修（d5d48dd）；宿主 `chrome-extension://` 下不再破图。

### 流2：资源入库（生成完成 / 手动保存）
```
Ev()(L42883) → POST /api/resources/rescan → 扫盘补 resources 表
Sv(e)(L42838) → POST /api/resources/save → upsert 单条
xv(e)(L42821) → GET /api/resources?page&pageSize&sortBy=timestamp
we()(L43015) = Ev()+xv()  → 资源面板「刷新」
rescanThrottledSync(L43028) 3s 节流防循环
```
跨层事件：`mutiwindow-task-completed`(L31426 等) → Ev() rescan → 资源刷新。

### 流3：删除（P2 缺陷）
```
wv(id)(L42857) → POST /api/resources/delete?id=...
  → 只删 DB 记录；磁盘文件 orphan（长期占用）
  → 清空 POST /api/resources/clear?deleteFiles=true 才联动删磁盘
```

### 流4：P1「上传不入库」真因（实测）
```
资源面板 z 回调(L29159): t=await R(e)(读文件) → O(t.map(source:'upload'))
  → O = transitItems setter（内存）
  → 未调 Sv()/Ev() → 刷新/重启丢（仅内存 transitItems）
  → 需手动点「同步」(we/Oi) 才 rescan 入库
```
URL 拖入(L29181–29184 paste 的 text/plain 分支)：`O([{url:r,source:'paste'}])` 同样只写内存，未落盘（TASKS P1「URL 拖入不落盘」真锚点）。

4. **缩略图伪复制（P2）**：`ri`(L1856) 是前端内存缓存；真实缩放伪装在 localTool `files.ts` L104–142（仅 copyFileSync）。修复在 localTool 侧接 sharp，不在 App.js。
