## 结论（高置信）

**当前 `USE_LOCAL_ENGINE=true`（config.js L36），双服务并存设计正常：生图走 9004 网关、文件落盘走 18080 localTool，二者不冲突。**
**但存在 P2 级脆弱点：一旦 `USE_LOCAL_ENGINE=false`，文件上传 base `Hr` 会变成 9004 网关，而网关无 `/api/files/upload` 路由（仅有 `/v1/uploads/images` 走 Lovart CDN），导致 `ii`/`Xr` 文件上传全部失败。**

## 一、双服务 base 实测

### 1. 文件落盘 base：`Hr`
- `App.js:L1732` `var Hr = localEngineBase()`
- `config.js:L42` `localEngineBase()` 定义：`return USE_LOCAL_ENGINE ? LOCAL_ENGINE.base : REMOTE_BASE`
  - `LOCAL_ENGINE.base` = `http://127.0.0.1:18080`（config.js L12–18）
  - `REMOTE_BASE` = `http://127.0.0.1:9004`（config.js L39）
- 即：`Hr` = `USE_LOCAL_ENGINE ? :18080 : :9004`

### 2. 文件上传调用点（均用 `Hr`）
- `App.js:L1812` `fetch(`${Hr}/api/files/upload`)`（`Xr`=uploadToLocalTool 内）
- `App.js:L1832` `fetch(`${Hr}/api/files/upload`)`（`Zr`=uploadHelper 姐妹内）
- 走 localTool 的 `useLocalTool` 另用 `Bc`=`LOCAL_ENGINE.port`(L19049) → `http://127.0.0.1:${Bc}/api/files/upload`(L19102)，**硬编码 18080，不受开关影响**

### 3. 生图/轮询 base（9004 网关）
- `R` / `_` / `g` 为各生图回调的网关 base（来自 ENDPOINTS/DEFAULT_ENDPOINT=:9004）
- `App.js:L33005` `zc(`${R}/v1/tasks/${taskId}`)` 轮询
- `App.js:L33525` `zc(`${_}/v1/tasks/${k}`)`；L34197 `zc(`${g}/v1/tasks/${k}`)`
- rescan：`vv`=`LOCAL_ENGINE.base`(L42808) → `fetch(`${vv}/api/resources/rescan`)`(L42885)

### 4. 路由归属核查（关键）
| 路由 | 提供方 | 证据 |
|------|--------|------|
| `/api/files/upload` | **localTool :18080 独有** | `localTool/src/index.ts:L135` `if (pathname === '/api/files/upload' && method === 'POST') return await handleUpload(...)` |
| `/api/resources/rescan` | localTool :18080 | `Ev`(L42883) 用 `vv`=LOCAL_ENGINE.base |
| `/v1/tasks/{id}`（轮询） | **网关 :9004** | `apimart-gateway/main.py` 路由 |
| `/v1/uploads/images` | 网关 :9004（但走 Lovart CDN，**非本地文件服务**） | `main.py:L900` `upload_image` → `client.upload_file(...)`（Lovart CDN） |
| `/v1/tasks/{id}/confirm` | 网关 :9004 | `main.py:L897` `return ok({"status":"confirmed"})` |

**网关 :9004 完全没有 `/api/files/upload` 路由** → `Hr` 指向 9004 时该 fetch 必 404/失败。

## 二、错配触发条件

```
USE_LOCAL_ENGINE = true  （当前 config.js L36 默认）
  → Hr = :18080
  → ii/Xr/Zr 上传 → localTool /api/files/upload ✅
  → 生图 R/_/g → :9004 网关 /v1/tasks ✅
  双服务并存，正常。

USE_LOCAL_ENGINE = false
  → Hr = REMOTE_BASE = :9004
  → ii/Xr/Zr 上传 → fetch :9004/api/files/upload ❌ 网关无此路由 → 失败
  → 生图 R/_/g → :9004 网关 ✅
  ⚠️ 文件落盘全失效（拖入文件、截图落盘、缩略图 URL 化全部失败）
```

注意：`useLocalTool`(L19102) 仍硬编码 `127.0.0.1:${Bc}`(Bc=18080)，不受开关影响 —— 即部分文件操作仍走 18080，与 `ii`/`Xr` 走 `Hr` 的行为**不一致**，开关切 false 时会出现"有的文件操作成功、有的失败"的撕裂态。

---

## 三、潜在 Bug / 存疑（交后续 AI 对抗审计）

1. **P2 · 开关切 false 文件上传失效**：`Hr` 受 `USE_LOCAL_ENGINE` 切换，但 `/api/files/upload` 仅 18080 有。修复建议：`Hr` 对文件类操作应**永远指向 LOCAL_ENGINE.base**，与开关解耦（或 REMOTE_BASE 也代理该路由）。当前因默认 true 不触发，但属隐含脆弱点。
2. **撕裂态**：`ii`/`Xr`/`Zr` 用 `Hr`（受开关），`useLocalTool` 用 `Bc`(硬编码 18080)。开关 false 时二者 base 不一致，行为难预测。建议统一文件 base 常量。
3. **`REMOTE_BASE`(:9004) 语义混淆**：命名暗示"远程基址"，实际指向同为本地 9004 网关。若未来真接远程网关，文件上传逻辑需重审。
