## X3. 配置加载与双服务 base URL 桥接（模块1+模块2+模块3）

```
config.js: USE_LOCAL_ENGINE=true (L36) → localEngineBase() (config L42)
  → Hr = localEngineBase() (App L1732)  [随开关变 18080/9004]
  → vv = LOCAL_ENGINE.base (App L42808) [固定 18080]
  → 上传 Zr/ii 走 Hr；写库 Sv/wv/Ev 走 vv
```

⚠️ **交叉风险点3（grep 坐实）**：`USE_LOCAL_ENGINE=false` 时 `Hr=9004` 但 `vv=18080`。上传走 9004 无 `/api/files/upload` 路由 → 失败；写库走 18080 仍安全。这与 TASKS P2-5「ResourceAdded base URL 不一致」同源（仅 false 触发）。

**Q：mutiwindow-task-completed 真的驱动 rescan 吗？** A：grep 派发 L38481，监听 L31428 `window.addEventListener('mutiwindow-task-completed',e)`；但 rescan 实际是 `we`(L43015) 在 `handleSyncLocal`(L44426) 和生成完成链路触发（`mutiwindow-sync-local` 系误造事件名，源码 0 命中，真实为 `handleSyncLocal` 函数）。监听 L31428 是另一用途（多窗口同步）。✅ 接缝存在但非单一驱动。
