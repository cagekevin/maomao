## 二、模板 → 万能节点的注入路径
1. 用户保存模板：`onSaveTemplate`(L35579 `(e,t)=>{...}`) 收集配置 → `onAddCustomNodeTemplate`(ne)。
2. `ne` 写入 `te`(全局 `customNodeTemplates`) 并（经 `Q.setObject(Z.CUSTOM_NODE_TEMPLATES,...)`）持久化到本地引擎。
3. 启动时 L43152 反向加载 `Tn(e)` 回填 `te`。
4. 画布"万能节点"菜单项（`customNode`，L35791）拖出后，从 `te` 取模板预填 `data.config`（L13781 `r(e,{config:n})`）。
5. 运行：`onGenerateCustom`(L35577，仅 `customNode` 绑定) → L13784 → `_r`(L35078) 执行 API/逻辑。

## 三、边界契约
- 模板持久化依赖本地引擎（`Q.setObject` → localTool :18080）；若 `USE_LOCAL_ENGINE=false`，`Q` 降级到 localStorage，模板仍可读写（见 02 模块 `Q` StorageManager 降级）。
- 运行时产物统一走 `ii`(L13789) 落盘 `canvas/upload`，受 B1/B2 入库存疑影响同款风险。
