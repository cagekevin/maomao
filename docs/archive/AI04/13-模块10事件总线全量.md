## 一、三类通信机制

### 1.1 同窗口 CustomEvent（`window.dispatchEvent`）
用于多面板联动（设置弹窗、任务完成回调等）。事件名前缀 `mutiwindow-`。

### 1.2 跨标签页 BroadcastChannel（`maomao_canvas_sync`）
仅用于画布保存跨标签页通知。L31542 `new BroadcastChannel('maomao_canvas_sync')`，收 `CANVAS_SAVED`(L31544，比对 `projectId` + 排除自身 `tabId`)。

### 1.3 跨窗口剪贴板（`localStorage['mutiwindow-clipboard']`）
节点/图片复制跨窗口传递。写入 L16376/L16503 `localStorage.setItem('mutiwindow-clipboard', t)`；读取 L36034 `getItem('mutiwindow-clipboard')`。复制 payload 带 `type:'mutiwindow-nodes'`(L35918) 或 `'mutiwindow-images'`(L36037)。

### 1.4 iframe postMessage（`window.parent?.postMessage`）
L23121 `window.parent?.postMessage({...})` 用于导演台全景图等向父框架发消息（type `storyai:director-desk-captures-sent`，见 L23122）。

## 三、关键收发点锚点
- **任务完成广播**：L38481 `new CustomEvent('mutiwindow-task-completed',{...})`（生图完成后 nodeId 关联刷新）。
- **重跑**：L44343 `dispatchEvent(new CustomEvent('mutiwindow-rerun-task',{...}))`。
- **设置面板统一监听**：L43774 `addEventListener('mutiwindow-open-builtin-settings',...)` + `addEventListener('mutiwindow-open-schedule-settings',...)`。
- **剪贴板写入**：L16376 `localStorage.setItem('mutiwindow-clipboard', t)`。
- **跨窗口节点粘贴**：L35918 `if(t&&t.type==='mutiwindow-nodes')` → 重新生成 id + 相对定位 + 自动连线(L35955 `originalIds`)。
- **tabId 生成**：L19344 `a = 'mutiwindow-'+Date.now()+random`（用于 BroadcastChannel 排除自身）。

## 五、存疑点（仅记录）
- **B15（待评估）**：`resourceAdded` 的发送源未在 App.js 内 grep 到 `dispatchEvent`（仅在 L43527 接收）。说明它由 localTool 通过某通道推入（可能经 `zc` 代理或 WebSocket）。发送源缺失 → 跨进程事件契约未闭环，建议查 localTool 推送机制。
- **B16（待评估）**：`mutiwindow-task-completed` 在 L38481/L43640/L43676/L43697/L44406 共 5 处发送，但接收仅 L31428/L36970。若某发送点 `nodeId` 缺失（`i.nodeId && dispatch`）则静默丢弃——失败任务无兜底通知。
- **B17（已知限制）**：BroadcastChannel 仅通知"已保存"，不传输画布数据，接收端仅置脏标记 `Yt(true)`（L31544），实际重载依赖用户手动操作，非实时多端同步。
