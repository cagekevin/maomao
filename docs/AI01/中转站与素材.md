## 1. 这是什么：中转站与素材库的定义、两者关系、本地引擎角色

「maomao」中"资源"在代码里实际表现为**两层（且已基本统一落库）**：

1. **中转站（transit）** —— 历史上"网页图片/视频被捕获后暂存"的一层。
   - 持久化键：`Z.TRANSIT_RESOURCES`（真实字符串 `"transitResources"`，见 `src/config/storageKeys.js:11`）。
   - 现在仍由浏览器扩展 `background.ts` 写入 `chrome.storage.local['transitResources']`（最多保留 5 条，见 `src/background.ts:97-99`）。
   - 当前版本里，中转站主要作为**启动期引导/兼容层**：应用启动后把其中的历史资源"播种/迁移"进素材库（`migrated` 文件夹），之后实时捕获已直接落素材库（见 §4、§3）。
   - 面板展示层面，中转资源与素材共用同一资源面板 `Qn`（`src/App.js:94`），以 `transitItems`/`transitResources` 注入（`src/App.js:135-136`、`42374-42375`）。

2. **素材库（resources / materials）** —— 正式资源库。
   - 由 `src/services/resourceStore.js` 通过**本地引擎 REST API** 增删查改。
   - 本地引擎 = `LOCAL_ENGINE`，base = `http://127.0.0.1:18080`（`src/config/constants.js:39` 引用 `src/config.js:14` 的 `LOCAL_ENGINE.base`）。`config.js:36` 固定 `USE_LOCAL_ENGINE = true`，故 `vv` 恒为本地引擎 base。
   - 服务端落库到 SQLite 表 `resources`（`localTool/src/routes/resources.ts`）；文件实体存于本地引擎数据目录 `~/.maomao-localtool`（启动日志 `localTool/src/index.ts:250`），按 `tasks/`、`migrated/` 等子目录组织。
   - UI 用两个 tab 区分来源：`生成`(folder=`tasks`) 与 `素材`(folder=`migrated`)（`src/App.js:195-200`、`39885`）。

**两者关系**：中转站是素材"落库前的暂存/引导层"；实时产生的资源（网页捕获、AI 生成、剪贴板、local-tool 上传）现在**直接**经 `Sv`（saveResource）进入素材库 `resources` 表（folder=`migrated` 或 `tasks`），不再长期停留在 `TRANSIT_RESOURCES`。`TRANSIT_RESOURCES` 主要承载"扩展侧尚未被侧边栏接收的历史条目"及"启动期播种"。

**本地引擎角色**：所有素材的元数据与文件都归本地引擎管（REST + SQLite + 磁盘）。`USE_LOCAL_ENGINE=true` 时不存在独立的"在线模式"素材通路；`LOCAL_MODE_ALLOW_ALL=true`（`src/config.js:90`）仅影响某些功能的本地模式放行，素材读写始终走本地引擎。

## 4. 中转站 → 素材的迁移（启动播种 + 本地化）

## 5. 素材库的增删查改

封装在 `src/services/resourceStore.js`，全部打 `vv`（`LOCAL_ENGINE.base`）：

- **规范化** `yv`（`resourceStore.js:5-12`）：`isFavorite` 布尔化、`timestamp` 转数字、空串字段删除（`pageUrl/pageTitle/source/folder/name`）。
- **构造查询串** `bv`（`resourceStore.js:13-16`）：`page/pageSize/sortBy/sortDir/search/filters`（filters 为 JSON）。
- **列表** `xv`（`resourceStore.js:17-33`）：GET `/api/resources?...`，默认 `sortBy=timestamp&sortDir=DESC`；返回 `{items,total,page,pageSize,totalPages}`。
- **保存** `Sv`（`resourceStore.js:34-46`）：POST `/api/resources/save`，返回 ok/false。
- **收藏** `Cv`（`resourceStore.js:47-52`）：复用 `Sv` 并带 `isFavorite`。
- **删除** `wv`（`resourceStore.js:53-61`）：POST `/api/resources/delete?id=`。
- **清空** `Tv`（`resourceStore.js:62-78`）：POST `/api/resources/clear`，body `{folder, deleteFiles}`，返回 `{deleted}`。
- **重扫描** `Ev`（`resourceStore.js:79-88`）：POST `/api/resources/rescan`，返回 `{count}`。

前端调用：
```text

src/App.js:39883-39889  be() 构造 filters：folder=eqOrPrefix(tasks|migrated[/子目录])、type、isFavorite
src/App.js:39909-39920  we()=统一同步：先 Ev()(rescan 清理孤儿) 再拉全量(分页300)
src/App.js:41028-41034  mi() 收藏：本地置位 + Cv

src/App.js:41203-41204  yi() 删除：wv(id) + 本地过滤
```

**rescan 孤儿清理（服务端）**：`handleResourcesRescan`（`resources.ts:37-133`）遍历 upload 子目录（`tasks/migrated/...`），把磁盘文件/文件夹以 `source='local-tool'` 录入；随后删除"库中存在但磁盘已不存在"的记录（`resources.ts:120-130`，`orphanDeleted`）。

> ⚠️ `handleResourcesClear` 实际行为（`resources.ts:211-238`）：
> - 带 `folder`：执行 `DELETE FROM resources WHERE folder = ?`；`deleteFiles=true` 时 `fs.rmSync(folderPath,{recursive:true})` **删除整个文件夹目录**。
> - 不带 `folder`：清空整张表；`deleteFiles=true` 时删除 upload 目录下所有子目录文件。
> - 注意：SQL **未**按 `is_favorite` 过滤。`hi()` 的 confirm 文案称"收藏的资源保留"，但服务端 clear 按 folder 整删，收藏项一并移除——这是事实代码行为，改码前需留意。

## 8. 门控 / 降级

- **未连接本地引擎**：新建文件夹、打开目录、刷新/rescan、拖动移动均要求 `S.status.isConnected`，否则提示"请先连接本地引擎"（`src/App.js:282`、`307`、`592`、`688`）。
- **非插件（chrome 扩展）环境**：发送到活动标签提示"发送失败：非插件环境"（`src/App.js:41087`）。
- **资源未连接本地引擎**入口提示"资源未连接本地引擎"（`src/App.js:592`、`688`）。
- 配置侧：`USE_LOCAL_ENGINE=true`（`src/config.js:36`），`LOCAL_MODE_ALLOW_ALL=true`（`src/config.js:90`）——素材读写恒走本地引擎，无在线降级通路。
- **引擎连接探测**：`useLocalTool` hook（`src/hooks/useLocalTool.js`）每 5s（未连）/15s（已连）轮询 `/api/status` 维护 `isConnected`；所有素材文件操作（`uploadFile`/`createFolder`/`moveFile`/`getKV`）在 `!isConnected` 时直接抛错或提示，是比 App 顶层 `S.status` 更细粒度的门控来源。
- **未连接引导卡**：未连本地引擎时，资源面板渲染"本地引擎未连接"引导卡（安装 local-companion、启动 `:18080`、重试连接），`src/App.js:39934-39996`（行号随构建漂移）。

## 9. 与云同步的关系（重点核查）

核查 `syncToCloud`（`src/App.js:40798-40847`）推送键集合，核心 for 循环（`src/App.js:40802`）：
```text
src/App.js:40802
  for (let t of [`app_settings`, `api_configs`, `projects`, `presetPrompts`,
                 `customNodeTemplates`, `modelSchedules`, `cloud_storage_config`, `local_templates`]) {
```
**结论（带证据）**：被推送的键集合**不包含** `transitResources`（即 `Z.TRANSIT_RESOURCES`，真实字符串 `"transitResources"`），也**不包含**任何素材库 `resources` 数据。GAS 云端仅同步配置类数据（设置/API/项目/预设/节点模板/模型计划/云存储配置/本地模板）。

因此：**素材/中转站不随 GAS 跨设备同步，仅存于本地引擎**（SQLite `resources` 表 + 磁盘 `~/.maomao-localtool` 目录）。

> 补充（避免混淆）：`Z.TRANSIT_RESOURCES` 出现在 `src/utils/storage/index.js` 的 `syncToLocalTool`/`syncAllToLocalTool` 键列表中（`index.js:313-330`、`323`），但那只是**本地多存储引擎之间**的对齐（localTool ↔ chrome.storage ↔ localStorage ↔ localforage），**不是** GAS 云端同步。GAS 推送路径里它从未出现。

### 9.1 远程基址与切换（补充）
`src/config.js:39` 定义 `REMOTE_BASE = 'http://127.0.0.1:9004'`（旧在线模式地址）；`src/config.js:43` `localEngineBase()` 在 `USE_LOCAL_ENGINE` 为真时返回 `LOCAL_ENGINE.base`、否则返回 `REMOTE_BASE`。当前 `USE_LOCAL_ENGINE=true`（`config.js:36`），故 `vv`/`Hr` 恒为 `:18080` 本地引擎，9004 在线通路未启用。

### 9.2 `local_templates` 推送时剥离二进制字段（补充）
GAS 推送集合里的 `local_templates` 在 `syncToCloud`（`src/App.js:40818-40848`）发送前会 **strip 掉** `coverUrl` 及节点内的 `imageUrl/thumbnailUrl/images/videoUrl/...Ref/...Uploaded` 等二进制/大体积字段，只保留纯 JSON 结构再上传。这进一步印证"GAS 刻意只同步轻量配置类数据，素材/中转站（文件型）不在其列"。

## 11. 改码前必查

1. **本地引擎端点一致性**：前端 `resourceStore.js` 的 `/api/resources*` 必须与 `localTool/src/index.ts:182-199` 路由完全一致；新增端点须同步改两端。
2. **`vv` 基准**：所有资源请求走 `vv`（`LOCAL_ENGINE.base`）。改 base/端口要同步 `src/config.js` 的 `LOCAL_ENGINE` 与 `localTool` 的 `PORT`（默认 18080）。
3. **`TRANSIT_RESOURCES` 键一致性**：真实字符串 `"transitResources"` 在 `storageKeys.js:11`、`background.ts:99`、`App.js:3429/39960/6774`、`storage/index.js` 多处引用，改名须全量替换。
4. **迁移逻辑幂等**：启动播种用 `kv` 标记（`src/App.js:39959`）与 localforage/KV 合并去重（`39960-39966`）；改动前确认不会重复入库或漏播。
5. **`Zr` 本地化铁律**：任何外部地址（http/data:/blob）入库前必须转内部 `/files/migrated/...`，否则跨设备/跨环境破图（`src/App.js:40415-40417`、本地化失败整条丢弃 `40422`）。
6. **rescan 孤儿清理**：`resources.ts:120-130` 会在 rescan 时删除磁盘已失的 `local-tool` 记录；改动删除逻辑前确认不会误删用户手动放入的文件。
7. **delete 与 clear 对文件的实际影响**：`wv`/delete 仅删 DB 行**不删磁盘文件**（`resources.ts:202-209`）；`Tv`/clear 带 `deleteFiles=true` 会删除**整个 folder 目录**（`resources.ts:215-237`），且 `hi()` 的 confirm 文案与实际"按 folder 整删（含收藏）"不符——改动前务必对齐文案与语义。
8. **GAS 同步范围**：若希望素材跨设备同步，必须在 `syncToCloud` 的 for 循环（`src/App.js:40802`）显式加入对应键，并确认云端 schema 兼容 `resources` 的大体积/二进制字段（当前刻意排除，原因见 §9）。
9. **门控条件**：任何新增资源操作需检查 `S.status.isConnected`（本地引擎，来自 `useLocalTool` 或 `S.status`）与插件环境（`y`），避免无连接时空操作。
10. **`useLocalTool` 端点一致性**：画布/节点组件走 `useLocalTool.js`（`/api/files*`、`/api/kv*`），顶层走 `localToolClient.js`/`resourceStore.js`（`/api/resources*`），二者都打 `LOCAL_ENGINE.base`；改端口只动 `src/config.js` 的 `LOCAL_ENGINE`，莫在调用处硬编码（`resourceStore.js` 文件头注释"统一使用 vv，不改 9004"）。
11. **导出排除对齐**：本地导出 `Ri`（`src/App.js:41671`）显式排除 `transitResources`；若新增"中转站/素材应进备份"的需求，须同步改此处与 `syncToCloud` 键集合（§9），并注意本地引擎文件本体并不随 JSON 导出。
12. **`REMOTE_BASE` 切换**：`localEngineBase()`（`config.js:43`）按 `USE_LOCAL_ENGINE` 在 `:18080` 与 `:9004` 间切换；当前恒本地，改动开关前确认 9004 在线服务是否仍可用。
