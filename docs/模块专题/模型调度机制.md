## 一、这是什么：模型调度机制的定义

maomao 是基于 React + Vite 的前端画布 / AI 工作流应用。每个生成类节点（绘图 / 文生文 / 视频 / sd2Video / 折扣视频 等）都要选一个"模型"去调用 API。节点 `selectedModel` 有两种形态：

1. **普通模型选择**：`selectedModel` 是一个具体模型名，来自该节点类型对应的配置列表（换行分隔的模型名字符串），如 `drawingModel` / `textModel` / `videoModel` / `sd2VideoModel` / `discountVideoModel`。
2. **模型调度（schedule）**：`selectedModel` 可为 `schedule:<id>` 形式的引用，指向一条"调度配置"。调度配置把多个模型按 steps（每步 `{ model, retries }`）排成一条**失败重试 / 故障转移序列**，运行期按顺序逐个尝试，某个模型失败就自动切下一个。

一句话：**一条调度 = 一个按序尝试的"故障转移 / 重试"序列**。调度只决定"尝试哪些模型、什么顺序、每个试几次"；真正的 API 调用由运行期生成函数（`sr/cr/ur/dr/pr`）逐个发起；而每个模型能否真正调用，仍受模型权益（entitlements）门控（见第六章）。

调度配置本身不调用任何 API，它只决定"尝试哪些模型及顺序"。

适用节点类型（分两层，详见 §9）：
- **选择器 UI 已暴露调度选择（含蓝色"调度"徽标 + "配置 ›"入口）**：绘图(image)、文生文(text)、AI 空间文本(director, text) 三类。
- **运行期已接入调度故障转移**：上述三类 + 视频(video)、sd2Video、折扣视频(discountVideo) 共 5 类生成函数。

### 2.2 一条调度对象的字段（`oa` 归一化，`:28-44`）

```text

  name:      string,                    // 默认 "未命名调度"
```

- `id`：`sa()` 生成 `sch_${Date.now().toString(36)}_${Math.random().toString(36).slice(2,8)}`（`:45-47`）。
- `category`：决定该调度出现在哪类节点的选择器里；`oa` 只允许 `text|image|video`，否则落 `image`（`:31`）。

### 2.3 `schedule:<id>` 引用约定（`:108-113`）

```text
function ga(e) { return `${ra}${e}`; }                       // 构造 'schedule:<id>'
function _a(e) { return e && e.startsWith(`schedule:`) ? e.slice(9) : null; }  // 解析出 id
```

- 节点 `selectedModel` 存的就是 `ga(id)`；`_a` 切片去掉前 9 个字符（`schedule:` 长度）取回 id。改前缀字符串必须同步 `ga`/`_a`/`ra` 三处。

### 2.4 steps 校验规则（核心刚性约束，`aa`/`ia`，`:7-27`）

```text
function ia(e) {            // 重试数归一化 1–3
  let t = Math.round(Number(e) || 1);
  return t < 1 ? 1 : t > 3 ? 3 : t;
}
function aa(e) {            // steps 校验：最多 5 步、总 retries ≤ 10
  if (!Array.isArray(e)) return [];
  let t = [], n = 0;
  for (let r of e) {
    if (t.length >= 5) break;                       // 最多 5 个模型
    let e = typeof r?.model == `string` ? r.model.trim() : ``;
    if (!e) continue;
    let i = ia(r?.retries);
    if (n + i > 10 && (i = 10 - n), i <= 0) break;  // 总重试超过 10 则截断
    t.push({ model: e, retries: i }), n += i;
  }
  return t;
}
```

- 单步 `retries` 归一到 1–3（`ia`）。
- 模型步数最多 5 个；累计 `retries` 超过 10 时，剩余步被丢弃（`aa` 第 20 行 `i = 10 - n` 后再判 `i<=0` 则 break）。
- 空 model / 非字符串被 `continue` 跳过；`oa` 在 `r.length === 0` 时返回 `null`，拒绝非法调度。
- UI 层同样做上限保护：加入第 6 个模型或累计重试达 10 时禁用"加入"按钮；每步 `retries` 的 +/− 步进按钮在 1/3 处禁用；上限提示文案固定为"同一模型最多重试 3 次，最多 5 个模型，总调度不超过 10 次"。

`va` 扁平化（按 retries 重复展开，上限 10，`src/services/modelSchedules.js:114-121`）：

```text
function va(e) {
  let t = [];
  for (let n of e.steps) for (let e = 0; e < n.retries; e++) {
    if (t.length >= 10) return t;
    t.push(n.model);
  }

```

**机制结论**（即"故障转移 / 重试"）：
1. `or(r)` 把 `schedule:<id>` 解析成一条 `enabled` 调度对象；解析不到（非 `schedule:` 前缀、id 不存在、被禁用）则返回 `null`。
2. 生成函数（`sr/cr/ur/dr/pr`）先 `or(r)`，命中则调用 `ar(e, 调度对象, 单模型生成回调)`，否则直接按原模型名生成。
3. `ar` 内部 `va(调度)` 把 `steps` 展开成"模型名按 retries 重复"的有序数组（如 step A retries=2、step B retries=3 → `[A,A,B,B,B]`，最多 10 个）。
4. `for` 循环逐个 `await n(模型)` 发起真实生成；每轮后 `ir(e)` 检查节点错误态：无错=成功并 `return`，用户中断=结束，否则打印"失败，切换下一个"继续。
5. 全部尝试失败则打印"调度结束：全部 n 次尝试均失败"。
6. `rr.current`(Set) 防同一节点重入。

## 六、与模型权益（modelEntitlements）的衔接

调度**只决定"尝试哪些模型及顺序"**；最终每个模型能否真正发起调用，仍受 `src/services/modelEntitlements.js` 的权益门控。两者是"顺序编排 + 单模型可用性判定"的叠加关系：

- 顶层放行 `ki(model)` 返回 `{ access: 'allowed' | 'denied' | 'quota_exceeded', ... }`（`:26-48`）；`LOCAL_MODE_ALLOW_ALL_MODELS` 为真时直接返回 `{ access: 'allowed' }`（`:29-31`）。
- `Xi(i)` 区分内置/三方，`Ai(i)` 取 deny reason；被禁用的模型在普通选择列表里就不可点（UI 禁用与提示）。
- 在调度运行期 `ar` 的循环里，每个 `await n(模型)` 回调（如 `Jn`/`tr`/`er`/`Qn`/`$n`）内部仍会走权益校验：若某候选模型因权益不足/额度耗尽而失败，`ir(e)` 读到错误态，循环照常切到下一个候选——即**权益失败也会触发故障转移**，但最终能否成功仍取决于队列里是否存在一个"权益允许"的模型。

因此整体语义：调度负责"排好候选顺序"，entitlements 负责"逐个判定能不能用"；二者互不影响字段，但运行期共同决定一次生成能否成功。

## 八、本地模式

调度是**纯本地配置**（仅存于 `modelSchedules` 键，随 GAS 同步但本质是本地草稿）。本地模式（无官方账号 / `LOCAL_MODE_ALLOW_ALL`）下：

- 权益层 `ki` 在本地模式直接返回 `{ access: 'allowed' }`（`modelEntitlements.js:26-32`，`Mi` 置 `LOCAL_MODE_ALLOW_ALL_MODELS = true`）。
- 因此调度序列里每个模型在本地模式都会被放行，不受官方账号权益限制；调度机制照常按序尝试、失败转移。
- 即：本地模式 = 用模型名直接调用 + entitlements 本地全放行，调度在此之上额外提供"多模型故障转移"能力，二者正交、可叠加。

## 十、改码前必查

1. **键名三处一致**：`constants.js`(`ta`)、`storageKeys.js`(`MODEL_SCHEDULES`)、`localStorage` 键、`Q` 键必须都等于 `modelSchedules` / `modelSchedules:change`，否则读不到、推送/合并错位。
2. **`schedule:` 前缀编解码对称**：`ga` 拼、`_a` 切（固定 slice(9)，前缀长度 9）。改前缀字符串必须同步改两处及 `ra` 常量。
3. **retries / steps 上限**：`ia` 归一化 1–3，`aa` 限制最多 5 步、总 retries ≤ 10；`va` 扁平化再卡一次 ≤ 10（`:20,117`）。UI 禁用阈值（`:37744,37776` 等）与这两个常量必须保持一致，否则会"能加但运行期被截断"或反之。
4. **`va` 上限 10 与 `aa` 总 retries ≤ 10 必须同源**：二者都基于 `retries` 累加，改其一需同步复核。
5. **GAS 合并按 id 去重、云端覆盖本地**：`ba`（`:125-130`）顺序为本地先、云端后覆盖。若需"本地优先"须改此处；拉取分支 `:41129` 与推送分支 `:41062` 必须都包含 `modelSchedules`，漏一个就不同步；推送务必用 `la()` 现读，不能用内存旧值。
6. **自定义事件 + storage 事件双订阅的清理**：`ha` 返回的清理函数必须被组件 `useEffect` 返回调用（`:2116`/`:3415`/`:23835`/`:37872`），面板监听 `:40900`/`:40905` 同理，否则切节点/切页会内存泄漏或串数据。
7. **`or` 只取 enabled 调度**：运行期故障转移仅在 `enabled===true` 时触发（`:32357`）；禁用后节点回落到"按原始字符串当模型名直接调用"，不要误以为会报错。
8. **category 仅 `text|image|video`**：`oa` 把非法 category 落 `image`（`:31`）；UI 三选一标签来自 `v_`，新增类目（如 `discountVideo`）需同步改 `oa` 白名单与 `S_` 的 category 选项，否则调度不会出现在对应节点里。
9. **视频/sd2Video/折扣视频 选择器未接调度 UI**：当前只有 `sr/cr/ur/dr/pr` 运行期支持；若要在这些节点 UI 暴露徽标/配置入口，需补 `la().filter(category===...)` 订阅、`ga()` 构造、`mutiwindow-open-schedule-settings` 配置按钮三件套（参考绘图节点 `:2115`/`:2890`/`:2954`/`:2959`）。
10. **本地模式放行**：调度序列每步最终受 entitlements 门控；本地模式 `LOCAL_MODE_ALLOW_ALL_MODELS` 才全放行，勿在调度层绕过权益检查。
