## 〇、这是什么

- **存储键**：`local_templates`（localTool KV，定义 `src/App.js` `LOCAL_TEMPLATES_KEY='local_templates'` ≈29043）
- **粒度**：多个节点的 subgraph + 它们内部相连的 edges
- **UI 入口**：左侧节点栏底部「模板库」按钮（`un()` ≈29882/36147）→ `sg` 面板；画布选中 ≥2 节点右键「创建模板」（`ln()` ≈29870/36245）
- **本质**：把自己框选好的节点组合存起来，之后从模板库拖回画布复用

> ⚠️ 易混：左侧栏「自定义」分组是**另一套**（自定义节点模板 `customNodeTemplates`），不在本专题范围内，也不在本面板里出现。

## 一、完整生命周期

### 1.1 创建（保存）
入口：框选 ≥2 节点 → 右键「创建模板」→ `ln()` 校验（必须已登录 `V`、有选中节点且 `type!=='group'`）→ 弹窗录名称（≈28955-29014）→ 确认调 `qr()`（≈34783）：
1. 取选中节点 + 它们**内部相连**的 edges（`$t.current` 过滤两端都在选中集）；
2. 以左上角节点为原点**归一化坐标**（减最小 x/y），删运行时字段（`loading/progress/errorMessage/imageUrlRef/imageUrlThumbRef/imageUrlUploaded/hasChanged`）；
3. **资源云端化**：`Wr(data, Hr)` 递归遍历节点 data，把 `url/Url/images/resultData` 等字段里的图片 blob/dataURL/远程 URL，经 `Hr`→`Zh()` POST 到 `${endpoint}/upload/asset`（folder=`templates`）转成 https；**视频资源保持原样**（与弹窗"仅云端化图片"一致）；
4. 取首个 https 图作 `coverUrl`；
5. 调 `Qh`（≈29054）写入：`{id:Date.now(), name, category:Gr(由节点类型推断 image/video/text), coverUrl, graphData:{nodes,edges}, nodeCount, isPublic:false, reviewStatus:'approved', createdAt}`，`unshift` 到数组头，`Q.setObject(LOCAL_TEMPLATES_KEY, arr)`。

### 1.2 读取（浏览）
- 左侧「模板库」按钮 → `sg` 面板（≈29133），默认 tab `mine`；
- `mine` → `$h()`（≈29061）：从 `local_templates` 读全部，按 `category` + `keyword` 过滤；
- 卡片渲染 `coverUrl`（`Yh` 加 `?x-tos-process=image/resize,w_300` 缩略）、`category` 角标、`name`；支持 `draggable`（dragstart 写 `text/plain` + 自定义 MIME）。
- 历史"公开/广场" tab 走 `eg()`（≈29070）——**直接 `return []`，本地模式无广场**，UI 上该 tab 已基本不渲染。

### 1.3 使用（插入画布）
从模板库**拖拽**到画布或点「使用」→ `Jr()`（≈34823）：
1. 遍历 `graphData.nodes`，**重新生成 id**（`${type}-${Date.now()}-${rand}`），建旧→新 id 映射；
2. 坐标 = 落点 `(t.x,t.y)` + 原相对坐标；
3. **插入时资源再处理**：`Wr(data, Ur)`——`Ur`（≈34743）把远程 https（`Vr` 判定）经 localTool `Zr(...,subfolder:'canvas/template')` **下载落盘到 18080** 再回贴 URL；本地 blob/dataURL 保持；
4. edges 按 id 映射重建（`selected:false`）；
5. `W(...concat(nodes))` + `G(...concat(edges))` 注入 React Flow（`setNodes/setEdges`）；
6. `rg(e.id)`（≈29089）记使用次数——**本地模式为空实现 no-op**。

### 1.4 修改 / 删除 / 公开
- 设为公开/私有：`tg(e, !isPublic)`（≈29073）翻转 `isPublic` 后写回；但广场 `eg` 恒空，**翻转后界面不可见**，属功能残桩。
- 删除：`ng(e)`（≈29081）按 id `splice` → `_saveLocalTemplates` 写回。
- **无编辑更新接口**：要改只能删了重建。

### 1.5 一句话
`框选节点 → 录名+图片云端化 → 写 local_templates(KV) → 模板库浏览/拖入 → 重 id+远程资源落盘18080 → 注入画布`；无编辑更新、无有效广场、无使用计数。

## 二、存储与全局持久化

- 落盘经 `Q.setObject / Q.getObject`（StorageManager），优先级 **localTool KV(:18080) → chrome.storage.local → localStorage/localforage**（见 `src/utils/storage/index.js` `wr`/`Mr`/`Q`）。
- 进 `exportData`/`importData` 兼容键集合（`41786` 列 `local_templates`）。
- 启动**不预载**全量，模板库打开时才 lazy 读 `$h`。

### 2.1 数据持久化的真实风险（已根因定位 · 2026-07-22）

> 现象：「昨天保存到我的模板，今天不见了」。根因来自 localTool 的落盘机制，与画布模板代码本身无关。

localTool 是一个 Node 进程（监听 `:18080`），内部用 **sql.js（WASM SQLite）内存数据库**：

- **写入只改内存**：`localTool/src/routes/kv.ts` 的 `handleKvSet`（≈22-34）执行 `run(db, DELETE...)` + `run(db, INSERT...)` 后只 `return json(res,{ok:true})`，**不调用任何落盘函数**。也就是说 `Qh`/`_saveLocalTemplates` 每次保存只是把数据写进进程内存，**磁盘上的 `localtool.db` 没变**。
- **落盘只在优雅退出**：`localTool/src/db/database.ts` 里 `saveDb()`（≈51-57）做 `db.export()` + `fs.writeFileSync(dbPath)`，但 `grep` 确认 **`saveDb()` 仅被 `closeDb()`（≈71-77）调用**；而 `closeDb()` 只在 `src/index.ts` 的 `shutdown`（≈271-278，绑定 `process.on('SIGINT'/'SIGTERM', shutdown)`）里触发。
- **结论 / 触发丢数据的路径**：
  - ✅ localTool 正常 `Ctrl+C` 退出 → `SIGINT` → `shutdown` → `closeDb` → `saveDb` → 内存落盘 → 今天能读到。
  - ❌ 被关终端窗口 / 任务管理器杀掉 / 电脑睡眠后进程消失 / 崩溃 → **没走 `shutdown`** → 内存里的 `local_templates` 永远没写进磁盘 → 下次启动读到的是旧的 `localtool.db` → **模板消失**。
- **数据文件位置**：`~/.yimao-localtool/localtool.db`（`getDataDir`，可用 `YIMAO_DATA_DIR` 环境变量覆盖）。

> 这正是「昨天有、今天没」的最可能原因：**localTool 上次不是优雅退出，内存里的模板没落盘**。画布模板的读写代码（`Qh`/`$h`/`_saveLocalTemplates`）都是对的，锅在 localTool 的「只在退出时落盘」设计。

## 三、改码前必查的雷区（红线级）

1. **别把自定义节点模板混进来**：本套只动 `local_templates` / `Qh`/`$h`/`Jr`/`tg`/`ng`；万能节点"保存为模板"写的是 `customNodeTemplates`，不是这里。
2. **资源落盘绝对路径**：`Jr`/`Wr`/`Ur` 处理模板资源时，远程 URL 必须经 localTool `Zr(...,subfolder:'canvas/template')` 落 18080 再回贴；严禁直接把 CDN/远程 URL 写进节点 data（违反 §3.2 异步契约）。
3. **保存时清理运行时字段**：`qr` 里 `delete loading/progress/errorMessage/imageUrlRef/...` 不能删错或漏删，否则模板带脏状态。
4. **KV 写入拒绝空值**：`Q.setObject` 对空 `{}`/`[]`/空串直接 warn 拒写（`storage/index.js` ≈239-248），写前确保 `graphData.nodes.length>0`。
5. **已知无害残留（别修）**：`eg` 广场恒 `[]`、`tg` 公开翻转无效、`rg` 使用计数 no-op、`RootErrorBoundary` 噪音——属本地模式功能残桩，非 Bug。
6. **新增模板字段**：若加字段，在 `exportData/importData` 兼容逻辑里登记。
7. **localTool 非优雅退出会丢数据（红线）**：`Qh`/`_saveLocalTemplates` 每次保存只进 localTool 内存，**不等落盘**；只有 localTool 收到 `SIGINT/SIGTERM` 优雅退出才 `saveDb`。不能假设"保存成功返回 ok 就持久化了"。
