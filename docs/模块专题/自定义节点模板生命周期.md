## 〇、这是什么

- **存储键**：`customNodeTemplates`（localTool KV，定义 `src/config/storageKeys.js` `CUSTOM_NODE_TEMPLATES='customNodeTemplates'`）
- **粒度**：单个万能节点（`customNode`）的 `config` 参数对象（含 `variables`）
- **UI 入口**：左侧节点栏 **「自定义」**分组（`te.map` 渲染，≈36098）
- **本质**：把万能节点配好的 API 参数存成预设，之后点一下就在画布上加一个同配置节点——是**节点级参数复用**，不进画布 subgraph、不碰资源

> ⚠️ 易混：画布模板库（`local_templates`，框选多节点的 subgraph）是另一套，不在本分组、不读本存储键。万能节点配置面板里"保存为模板"写的是**这里**（`customNodeTemplates`），不是模板库。

## 一、完整生命周期

### 1.1 创建（保存）
入口：万能节点（`customNode`）配置面板"保存为模板"按钮（`y()` ≈12463）→ 弹 `window.prompt` 要名称 → 调 `onSaveTemplate(name, config)`（≈33883）→ 回调 `ne`（来自父组件 `onAddCustomNodeTemplate`）；
最终落到 `Si()`（≈42178）：
- `e.id ||= Date.now().toString()`；
- `wn = [...wn, e]`（`wn` 即 `customNodeTemplates` 状态）；
- `Tn(t)` 更新 React 状态 + **`Q.setObject(Z.CUSTOM_NODE_TEMPLATES, t)`** 落盘。
- 写入结构：`{id, name, config}`（`config` 是该万能节点的 API 参数对象，含 `variables`）。

### 1.2 读取（浏览）
- 启动 `loadAppSettings`（≈41187）从 KV 读 `customNodeTemplates` → `Tn(e.customNodeTemplates)` 注入全局 `wn`；
- 左侧节点栏 **「自定义」**分组（`te.map`，≈36098）渲染为下拉列表（`te = customNodeTemplates`）。

### 1.3 使用（实例化节点）
点列表项（≈36100）→ 以 `e.config` 实例化一个 `customNode`：`Z('customNode', …, {label:e.name, config:e.config, configMode:false, variables:e.config.variables})`，属**节点级参数复用**，不进入画布 subgraph，也不处理资源。

### 1.4 删除
`Ci()`（≈42183）：`confirm` 后 `wn.filter(id!==e)` → `Tn` + `Q.setObject(Z.CUSTOM_NODE_TEMPLATES, t)` 落盘。
- **无编辑更新接口**：要改只能删了重建。

### 1.5 一句话
`万能节点内"保存为模板" → {id,name,config} 写 customNodeTemplates(KV) → 自定义分组列表 → 选中回填 config 实例化节点 → 可删除`；仅存配置、不处理资源、无编辑更新。

## 二、存储与全局持久化

- 落盘经 `Q.setObject / Q.getObject`（StorageManager），优先级 **localTool KV(:18080) → chrome.storage.local → localStorage/localforage**（见 `src/utils/storage/index.js` `wr`/`Mr`/`Q`）。
- 进 `exportData`/`importData` 兼容键集合（`42387` 列 `customNodeTemplates`；老格式 `42431` 把 `t.customNodeTemplates` 映射到 `Z.CUSTOM_NODE_TEMPLATES`）。
- 启动即预载（经 `loadAppSettings` → `Tn`），不像画布模板那样 lazy 读。

## 三、改码前必查的雷区（红线级）

1. **别把画布模板混进来**：本套只动 `customNodeTemplates` / `Si`/`Ci`/`Tn` / 万能节点 `onSaveTemplate`；框选多节点存的是 `local_templates`，不是这里。
2. **不处理资源**：本套只存 `config` 参数，任何资源 URL 由节点运行时自己处理，模板层不应做云端化/落盘（那是画布模板 `Wr`/`Hr`/`Ur` 的职责）。
3. **`id` 生成**：`Si` 用 `Date.now().toString()`，删除/去重都靠它，改动时保持唯一性。
4. **KV 拒绝空值**：`Q.setObject` 对空 `{}`/`[]`/空串 warn 拒写（`storage/index.js` ≈239-248），写前确保 `config` 非空。
5. **已知无害残留（别修）**：列表删除用 `window.confirm` 原生弹窗属历史实现；`RootErrorBoundary` 噪音不改。
6. **新增字段**：若给模板加字段，在 `exportData/importData` 兼容逻辑里登记，并保持 `loadAppSettings` 的 `Tn` 注入一致。
