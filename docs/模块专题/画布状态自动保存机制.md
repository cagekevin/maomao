# 画布状态自动保存机制（模块专题 · 按需查阅）

> 事实锚点：grep `src/utils/storage/index.js`、`src/config/storageKeys.js`、`src/App.js`（行号快照 2026-07-23）坐实。
> 行号随构建漂移，主引用用函数语义名 + 真实路径；动手前回源码复核当前行号。
> 本文只写代码里已存在的机制，不写方案、不写待决策。

---

## 〇、这是什么

画布上的节点 / 连线 / 视口，按「当前项目」分桶，自动（防抖）落盘到 localTool，保证刷新 / 切项目不丢当前工作。键格式 `canvas-state-v1-<projectId>`。

---

## 一、存储键与落盘层

- 键构造 `Cr(e)` = `canvas-state-v1-${e}`（`storage/index.js:7-9`）；前缀 `CANVAS_STATE_PREFIX`（`storageKeys.js:16`）。
- `saveCanvasState`/`loadCanvasState`（≈356/360）→ `setLocalforage`/`getLocalforage`：先写 `wr`（localTool），再写 `Pr`（localforage）**双写**；读时 `wr` 优先、缺失回退 `Pr`。
- `saveCanvasStateWithVersion`/`getCanvasVersion`（≈364/385）：带版本号防并发覆盖。

---

## 二、自动保存触发（已验证事实）

- 自动保存函数 `gn`（`App.js:29526`，依赖 `W`=setNodes / `M`=setEdges / `H.saveKV`）：内部调 `saveCanvasStateWithVersion(en.current, r, Date.now())`（≈29609-29627），版本号取 `Date.now()` 时间戳。
- **触发 1 · 防抖（主路径）**（`App.js:29736-29742`）：`useEffect` 依赖 `[U(nodes), pe(edges), gn]`，`setTimeout(gn, 1800)`；`ge`（已加载）为假或 `vt.current`（撤销/重做中）则跳过。
- **触发 2 · 跨标签页广播**（`App.js:29742-29754`）：监听 `CANVAS_SYNC_REQUEST`，同项目则触发 `gn()`，保证多标签页画布同步。
- **非保存按钮触发**：当前实现无显式「保存」按钮，纯自动。
- **版本冲突处理**（≈29609-29627）：若 `getCanvasVersion` 返回远程版本 > 本地 `Date.now()`，拒绝覆盖并 `getObject` 回拉最新节点/边重灌（提示「其他页面已更新画布」）。

---

## 三、读取 / 加载

- 画布组件以 `projectId` 为 prop（`en.current = e`，`App.js:29447`）；加载 `useEffect` 依赖 `[e, …]`（≈29670）调 `loadCanvasState(Cr(e))` 灌入 `Qt.current`/`W`/`G`。
- 空画布（`nodes` 为空）`saveCanvasStateWithVersion` 直接 `skipped`，不写（≈366）。

---

## 四、一句话

`节点/边变更 → 1800ms 防抖（或跨页广播）→ gn → saveCanvasStateWithVersion(canvas-state-v1-<projectId>, 状态, Date.now()) → 双写 wr+Pr；远程版本更新则拒绝覆盖并回拉`。

---

## 五、改码前必查

1. **键必须带 projectId**：任何读 / 写都要走 `Cr(projectId)`，否则会串项目。
2. **空画布拒写也作用于此**：`saveCanvasStateWithVersion` 内对 `canvas-state-v1-*` 还额外校验 `nodes.length>0`（`storage/index.js:243-245`），无节点不落盘。
3. **与「本地 JSON 存储」的关系**：本机制走 `setLocalforage`（wr+Pr 双写），而 `Q.setObject` 走 `wr` 单写 + 空值拒写——两者都受 localTool「仅优雅退出落盘」风险影响（见 `本地JSON存储机制.md` §三）。非优雅退出仍可能丢最近一次自动保存。
4. **撤销/重做中勿触发**：`vt.current` 守卫防抖，改动相关逻辑勿破坏。
