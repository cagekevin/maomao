## 〇、这是什么

提示词管理是**纯本地、多入口、单组件**的体系。所有预设存一个 KV 数组，三个入口打开同一个 `PromptLibrary` 组件。

| 入口 | 触发方式 | 行为 |
|------|---------|------|
| 节点面板 → 「提示词库」按钮 | 下拉中点击 | 浏览全部、搜索、分类筛选、使用提示词 |
| 节点面板 → 「提示词」下拉 | 直接点下拉项 | 快速选一条填入节点 |
| 设置 → 「预设提示词」tab | 点击左侧 tab | 打开 PromptLibrary，可增删改 |

> 对比旧版：原来有 3 个独立入口（设置面板内联编辑器 + 下拉 `Ja` + 弹窗 `qa`），现在统一为一个 `PromptLibrary` 组件。远程广场/标签/收藏残桩已全部移除。

## 二、数据流

```
KV(presetPrompts) ← Q.getObject/Q.setObject
        ↑↓
App.js jr 状态 (Mr setter)
        ↑ yimao:presetsChanged 事件
        │
PromptLibrary ──saveAndNotify(newPresets)──→ Q.setObject + dispatchEvent
        ↑ props: presetPrompts=jr
        │
  PromptDropdown(5 处节点面板)
        ↑ 分发
  App.js 启动加载 → 注入 jr
```

### saveAndNotify（新增）

`promptManager.saveAndNotify(presets)` → 两步原子操作：
1. `Q.setObject(Z.PRESET_PROMPTS, presets)` → 写 KV
2. `window.dispatchEvent(new CustomEvent('yimao:presetsChanged', { detail: presets }))` → 通知 App.js 刷新 `jr`

**为什么用事件而不是 props 传 `Mr`**：5 处节点面板是模块级函数，无法访问 `App` 函数内的 `Mr`(setJr)。事件总线解耦了状态更新。

## 三、PromptLibrary 组件（替代旧 `qa` 弹窗）

**Props:**
```
open, onClose, onUse?, onToast?, defaultCategory?, presetPrompts
```

**内部功能：**
- 两 tab：「我的提示词」（本地预设卡片网格）、「最近使用」（localStorage 驱动）
- 分类筛选：全部/文本/生图/视频（`Ua`）
- 搜索：标题 + 内容
- 新建：表单（标题/类型/内容） → `saveAndNotify`
- 编辑：点击卡片编辑按钮 → 表单 → `saveAndNotify`
- 删除：编辑表单内删除按钮 → `saveAndNotify`
- 使用：点击卡片 → `recordRecent(id)` → `onUse(content)`
- `onUse` 为空时（从设置面板打开）：复制到剪贴板

**无远程代码**：不再调用 `Fa`/`Pa`/`Ia`/`La`/`Ra`/`za`，无标签行，无收藏按钮。

## 六、危险操作（红线）

1. **两张写路径**：防抖 1s（≈40989）+ 导入导出（≈41943）。新增的 `saveAndNotify` 是第三条路，写 KV 后必须 dispatch 事件通知 `jr` 更新，否则 UI 和防抖保存都不会触发。
2. **`enabled` 语义是 `!== false`**：`mapToLibraryCards` 过滤 `p.enabled !== false`，消费端一致。
3. **KV 拒绝空值**：`Q.setObject` 对空 `{}`/`[]`/空串 warn 拒写。空 `[]` 只 warn 不报错。
4. **localTool 非优雅退出丢数据**：内存 sql.js，只有 SIGINT/SIGTERM 才 `saveDb()`。
5. **编辑靠下标**：旧数据可能无 `id` 字段，`mapToLibraryCards` 用 `presetIndex`（原数组下标）定位编辑目标。删除/新增导致下标漂移时，编辑中的项可能映射到错误位置。
6. **事件名不要乱改**：`yimao:presetsChanged` 是 PromptLibrary 和 App.js 之间的通信协议，改一边另一边也改。

---

## 七、一句话

> 提示词管理 = **一个 KV 数组 `presetPrompts`** + **一个 PromptLibrary 组件**（三个入口统一）+ **一条 localStorage 最近使用**。纯本地，无远程残桩。数据变更走 `saveAndNotify`（写 KV + 事件通知 jr 刷新），落盘依赖 localTool 优雅退出 + 防抖 1s 自动保存。
