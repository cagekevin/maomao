## 〇、这是什么

提示词管理是**多入口、单存储**的体系：所有预设存在一个 KV 数组里，三个入口读写它。

| 子体系 | 唯一存储 | 编辑入口 | 消费入口 | 本地可用性 |
|--------|----------|----------|----------|------------|
| **本地预设提示词** `presetPrompts` | KV 键 `presetPrompts`（localTool） | 设置 → 「预设提示词」tab | 节点「提示词」下拉、提示词库弹窗本地项 | ✅ 完整可用 |
| **提示词库弹窗** `qa` | 复用上面的 KV + localStorage 最近使用 + 远程（残桩） | 仅浏览/使用 | **唯一打开点**：节点下拉的「提示词库」按钮 | ⚠️ 仅本地/最近项可用 |
| **节点「提示词」下拉** `Ja` | 复用上面的 KV + 远程收藏（残桩） | 「本地配置」按钮跳设置 | 5 处节点面板（image/text/video×3） | ✅ 完整可用 |
| **最近使用** | localStorage `yimao:promptRecent` | 自动记录 | 弹窗「最近使用」tab | ⚠️ 本地模式实质失效（见 §1.3） |
| **广场 tab** | 远程 API（残桩） | — | 弹窗广场 tab | ⚠️ 远程恒空，但兜底渲染本地预设 |
| **标签** | 远程 API（残桩） | — | 弹窗标签栏 | ❌ 纯残桩 |
| **收藏** | 远程 API（残桩） | — | 卡片收藏按钮 | ❌ 纯残桩 |

> 易混：`src/config/options.js` 里的内置提示词技巧 `mi`（约 40 条静态文案/构图/运镜建议）是**静态展示素材**，不可编辑、不落 KV、不受本专题管理。

## 一、完整数据流（从前到后）

### 1.1 存储层（单一事实来源）

- 存储键：`Z.PRESET_PROMPTS`（值 `'presetPrompts'`）定义在 `src/config/storageKeys.js:10`。
- 读写函数：`Q.getObject` / `Q.setObject`（`src/utils/storage/index.js`）。
  - ⚠️ **`Q` 只走 localTool 引擎 `wr`，无任何回退链**。`Q.getObject`→`jr()`→`wr.get`；`Q.setObject`→`wr.setObject`（仅 `wr`）。`Mr`(chrome.storage **引擎**，见 `storage/index.js`) / `Nr`(localStorage) / `Pr`(localforage) 三引擎虽定义，但 `Q` 的读写完全不用它们——`getStatus()` 只是报告能力。（⚠️ 此处的 `Mr` 是 storage 引擎，与下文 §1.2 启动加载里 `Mr(e)` 的 App.js `presetPrompts` 状态 setter 同名但**不是同一个符号**。）
  - 即：**`presetPrompts` 只有一份拷贝在 localTool（sql.js 内存库），无 chrome/localStorage 兜底**。这与「localTool 非优雅退出会丢预设」是同一事实的两面。
  - `Q.setObject` 拒绝空 `{}`/`[]`/空串（`index.js` ≈239-248），预设列表为空 `[]` 时只 warn 不报错。

### 1.2 启动加载（读）

- 锚点：`Q.getObject(Z.PRESET_PROMPTS)`（当前 `src/App.js` ≈40546）。
- 逻辑：`Q.getObject(Z.PRESET_PROMPTS).then(e => e && e.length > 0 && Mr(e))` —— 读 KV 注入全局 `jr`（`presetPrompts` 状态数组；这里的 `Mr` 是 **App.js 内**的 `presetPrompts` 状态 setter，勿与 storage/index.js 的 chrome.storage 引擎 `Mr` 混淆）。**空数组 / 键不存在则不注入**，列表为空。
- 全局 `jr` 经 React 树下发到各节点组件作 `presetPrompts` prop，再传给 `Ja` 下拉与 `qa` 弹窗。纯内存读取，无预载特殊处理。

### 1.3 编辑（设置面板「预设提示词」tab）

- 入口 UI 锚点：中文串 `预设提示词`（当前 `src/App.js` ≈42921/42954/43760，即设置面板里该 tab 的标题文案）。
- 数据结构（每项）：`{ id?, title, type: 'all'|'text'|'image'|'video', prompt, enabled: true }`。**旧 KV 项可能无 `id`**，渲染与比对靠数组下标，不是 `id`。
- 添加 / 逐字段编辑 / 删除：混淆函数 `Ai`/`ki`/`Fi`（当前 ≈42959/43075/43013，行号易漂移，改码时用功能语义定位，勿依赖行号）。
  - `enabled` 语义是 `!== false`：复选框锚点 `e.enabled !== false`（当前 ≈42974）；禁用项在消费端被 `filter(e=>e.enabled!==false)` 过滤，但仍留数组。
  - `type` 用于消费端按节点类型归类：`all` 通用 / `text` / `image` / `video`。
- 数量上限：会员权益 `lr.presets` 的超限判断被 `false &&` 短路（当前 ≈43019），**当前实际无上限**。

### 1.4 落盘（两条独立写路径，都要覆盖）

- **路径 A：防抖自动保存**。监听 `jr` 变化的 `useEffect`（当前 ≈40984-40991）1 秒后 `Q.setObject(Z.PRESET_PROMPTS, jr)` 全量覆盖写；同时顺手落 `API_CONFIGS`/`CUSTOM_NODE_TEMPLATES`/`CLOUD_STORAGE_CONFIG`。无显式保存按钮。
- **路径 B：导入/导出**。在 `exportData`/`importData` 流程里也有 `await Q.setObject(Z.PRESET_PROMPTS, jr)`（当前 ≈41943，与路径 A 并列的独立写点）。
- ⚠️ 改保存逻辑时**两条路径都要改**，漏掉任一都会导致导入导出后预设不回写或防抖写被覆盖。
- 落盘终点依赖 localTool 优雅退出才写磁盘（详见 `画布模板生命周期.md` §2.1）。

### 1.5 消费入口 A：节点「提示词」下拉 `Ja`

- 定义锚点：`function Ja({category, presetPrompts, onApply, onToast})`（当前 ≈2278）。挂载于 5 处节点面板，分别传 `category`：`image`/`text`/`video`×3（对应 onApply 追加逻辑在 ≈2438/4273/7901/9035/11001）。
- 列表 = 远程收藏 `m`（`La()` 恒 `[]`，本地永远空）+ 本地预设 `p`（原始 `presetPrompts`，过滤 `e.enabled !== false` ≈2288，按 `type` 归类：匹配项在前、其余在后 ≈2289-2290）。
- 点击：
  - 本地预设项 `onClick: () => h(e.prompt)`（≈2343，`h` 即调 `onApply`，字段是原始 `.prompt`）；
  - 远程收藏项 `onClick: () => h(e.content)`（≈2332，字段 `.content`）。
  - `onApply` 统一把内容**以 `, ` 分隔追加**到节点当前 prompt（`${cur}, ${t}`，见 5 处节点 ≈2438/4273/7901/9035/11001），**不是替换**。
- 底部两按钮：
  - `本地配置` → `Ha()`（当前 ≈2353，派发 `Va=yimao:openPromptSettings` 事件）→ 设置面板监听处打开 `settings`+`basic` tab；
  - `提示词库` → `s(true)`（当前 ≈2359）→ 打开 `qa` 弹窗（也是 `qa` 唯一打开点，≈2366）。

### 1.6 消费入口 B：提示词库弹窗 `qa`

- 定义锚点：`function qa({open, onClose, onUse, ...})`（当前 ≈2059，定义在 `Ja` 之前）。**唯一打开点**：`Ja` 内「提示词库」按钮（当前 ≈2356-2363 → `s(true)` → ≈2366）。
- 本地预设映射 `N`（当前 ≈2092-2101）：`q.filter(e=>e.enabled!==false).map(e=>({ id:'preset-'+e.title, title:e.title, content:e.prompt, category: e.type==='all'?'':e.type, isLocal:true }))`。注意：这里把原始 `.prompt` **重映射成 `.content`**，所以弹窗内本地项字段是 `.content`。
- Tab：
  - `提示词广场`（square）：远程 `Fa` 恒 `[]` → UI 兜底 `l.length > 0 ? l : N`（≈2109），广场空时直接显示本地 `N`。**广场 tab 本地可用、能看到你自己的预设，非残桩**。
  - `最近使用`（recent）：`Ga()` 读 localStorage `yimao:promptRecent`，与 `[...l, ...m]`（l=广场远程、m=收藏远程，均本地空）取交集（≈2103-2107）。
- 标签过滤：推荐 + `Pa()` 恒 `[]`（残桩，只剩硬编码「推荐」）。
- 收藏态：`Ia()`/`La()` 恒 `[]`；卡片收藏按钮调 `Ra`/`za` 本地均失败（`Ra`→`{ok:false,error:'本地模式不支持'}`、`za`→`false`）。
- 使用：
  - 本地项 `onClick: () => e.isLocal ? n(e.content) : j(e)`（≈2261，`n`=`onApply`）。
  - 远程项 `j(e)`：`Ka(e.id)` 记最近（写 `yimao:promptRecent`，上限 50）+ `n(e.content)`。
  - ⚠️ **「最近使用」tab 本地模式实质失效**：`Ka` 只在远程项分支调用，本地项直接走 `n(e.content)` 不调 `Ka`；远程恒空 → 本地永不触发 `Ka` → 该 tab 永远空。修复需让本地项也调 `Ka(e.id)` 并让 recent 查找表（`[...l,...m]`）能命中本地 id `preset-<title>`。

## 二、存储与全局持久化（结论）

- **主存储**：`presetPrompts` 走 `Q.getObject/Q.setObject`，**只落 localTool（`wr`），无 chrome/localStorage 兜底**（见 §1.1）。`Q.setObject` 拒空值（≈239-248）。
- **导入导出**：预设含在导出键集合（`presetPrompts`/`local_templates` 等，当前 ≈41369），且导入导出路径会写回 KV（≈41943）。
- **最近使用**：不走 KV，直接 `localStorage['yimao:promptRecent']`（`Ka` 写 / `Ga` 读，键 `Wa` 在 `src/services/auth.js:125`，上限 50），纯前端、不进 localTool。
- **远程数据**：`Pa/Fa/Ia/La/Ra/za` 全部 `src/services/auth.js` 首行 `return` 桩值（行号：Pa=47/Fa=52/Ia=59/La=69/Ra=80/za=106），即使接远端也不落本地 KV——远程收藏与广场是独立服务端状态。预设本身也不经 `auth.js`，是两套独立通道。

### 2.1 真实风险（localTool 落盘同源）

`presetPrompts` 最终落盘依赖 localTool KV。localTool 是 Node 进程，用 sql.js（WASM SQLite）**内存库**：写入只进内存，**仅 `SIGINT/SIGTERM` 优雅退出时 `saveDb()` 才写磁盘**（详见 `画布模板生命周期.md` §2.1；端口/数据路径以 localTool 实际配置为准）。

- ✅ localTool 正常退出 → 内存落盘 → 下次能读。
- ❌ 被杀/关窗口/崩溃 → 没走 shutdown → 内存预设没落盘 → **预设消失**（「昨天加的提示词今天没了」）。
- 数据文件：`~/.yimao-localtool/localtool.db`（`YIMAO_DATA_DIR` 可覆盖）。
- 比画布模板温和：预设高频防抖自动保存，丢的只是一小段时间窗内的编辑。

## 三、改码前必查的雷区（红线级）

1. **三套入口别混写**：`presetPrompts`（KV 可编辑）、`mi`（静态技巧）、远程广场（残桩）是三个不同东西。动列表只动 `jr`/`presetPrompts`。
2. **编辑靠下标不是 `id`**：旧项可能无 `id`，**不要**改成按 `id` 查找而不兼容缺 `id` 项，否则老数据错位/删错。
3. **`enabled` 语义是 `!== false`**：过滤统一用 `e.enabled !== false`（三处：N≈2092、Ja≈2288、复选框≈42974）。新增字段/导出别写成 `=== true`。
4. **保存是整组覆盖写**：每次 `jr` 变化 1s 后全量 `Q.setObject(Z.PRESET_PROMPTS, jr)`。注意**两条写路径**（防抖≈40989 + 导入导出≈41943），改保存逻辑两条都要动。
5. **KV 拒绝空值**：`Q.setObject` 对空 `{}`/`[]`/空串 warn 拒写（≈239-248）。
6. **数量上限是死代码**：`false &&` 短路（≈43019），当前实际无上限。启用前先确认 `lr.presets` 本地真实取值。
7. **远程能力区分对待**：`Pa`(标签)/`Ia`/`La`/`Ra`/`za`(收藏) 是**纯残桩**，属本地模式设计不是 Bug，别修也别依赖。但 **`Fa`(广场) 虽恒 `[]`，弹窗广场 tab 以本地 `N` 兜底渲染**（`l.length>0?l:N`），本地可用、能看到自己预设——改 UX 别误以为广场永远空白。接远端前先打通网关基址（`auth.js` 内 `${$n}${er}`）与鉴权。
8. **localTool 非优雅退出会丢预设（红线）**：不能假设「改完 1 秒就持久化」——那 1 秒后只是进 localTool 内存，磁盘落盘仍在退出时。
9. **新增字段要在导出登记**：确认 `exportData/importData`（≈41369）含 `presetPrompts`，并兼容老格式（老项可能缺 `type`/`id`/`enabled`）。
10. **`Va` 事件耦合设置面板**：节点「本地配置」按钮经 `Ha()` 派发 `Va`（`yimao:openPromptSettings`，常量在 `src/config/constants.js:19`）→ 设置面板监听处打开 `settings`+`basic`。改事件名或设置 tab 结构两边一起改。
11. **「最近使用」tab 本地模式实质失效（改前须知）**：`Ka` 只在 `qa` 远程项分支调用，本地项走 `n(e.content)` 不调 `Ka`；远程恒空 → 永不触发 → tab 永远空。修复要点见 §1.6。

## 五、一句话给改码人

> 提示词管理 = **一个可编辑的 KV 数组 `presetPrompts`**（设置里增删改、两条路径落盘：防抖≈40989 + 导入导出≈41943）+ **两个只读消费入口**（`Ja` 下拉 / `qa` 弹窗）+ **一条 localStorage 最近使用记录（本地失效）**；远程广场/标签/收藏是本地模式残桩。动它记住：编辑按下标、启用看 `!==false`、保存是全量覆盖且两条路径都要改、落盘赌 localTool 优雅退出、行号会漂改用稳定字符串锚定。
