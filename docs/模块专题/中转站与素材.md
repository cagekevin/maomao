## 1. 这是什么：中转站与素材库的定义、两者关系、本地引擎角色

「maomao」中"资源"在代码里实际表现为**两层（且已基本统一落库）**：

1. **中转站（transit）** —— 历史上"网页图片/视频被捕获后暂存"的一层。
   - 持久化键：`Z.TRANSIT_RESOURCES`（真实字符串 `"transitResources"`，见 `src/config/storageKeys.js:13`）。
   - 仍由浏览器扩展 `background.ts` 写入 `chrome.storage.local['transitResources']`（最多保留 5 条，`.slice(0,5)`，见 `src/background.ts:97`）。
   - 当前版本里，中转站主要作为**启动期引导/兼容层**：应用启动后把其中的历史资源"播种/迁移"进素材库（`migrated` 文件夹），之后实时捕获已直接落素材库（见 §3、§4）。
   - 面板展示层面，中转资源与素材共用同一资源面板 `Qn`，以 `transitItems`/`transitResources` 注入。

2. **素材库（resources / materials）** —— 正式资源库。
   - 由 `src/services/resourceStore.js` 通过**本地引擎 REST API** 增删查改。
   - 本地引擎 = `LOCAL_ENGINE`，base = `http://127.0.0.1:18080`（`src/config/constants.js:39` 引用 `src/config.js:12-18` 的 `LOCAL_ENGINE.base`）。`config.js:36` 固定 `USE_LOCAL_ENGINE = true`，故 `vv`/`Hr` 恒为本地引擎 base。
   - 服务端落库到 SQLite 表 `resources`（`localTool/src/routes/resources.ts`）；文件实体存于本地引擎数据目录 `~/.maomao-localtool/uploads`（`localTool/src/db/database.ts:14,18`，可用环境变量 `MAOMAO_DATA_DIR` 覆盖），按 `tasks/`、`migrated/` 等子目录组织（`localTool/.env.example:10-13`）。
   - UI 用两个 tab 区分来源：`生成`(folder=`tasks`) 与 `素材`(folder=`migrated`)。

**两者关系**：中转站是素材"落库前的暂存/引导层"；实时产生的资源（网页捕获、AI 生成、剪贴板、local-tool 上传）现在**直接**经 `Sv`（saveResource）进入素材库 `resources` 表（folder=`migrated` 或 `tasks`），不再长期停留在 `TRANSIT_RESOURCES`。`TRANSIT_RESOURCES` 主要承载"扩展侧尚未被侧边栏接收的历史条目"及"启动期播种"。

**本地引擎角色**：所有素材的元数据与文件都归本地引擎管（REST + SQLite + 磁盘）。`USE_LOCAL_ENGINE=true` 时不存在独立的"在线模式"素材通路；`LOCAL_MODE_ALLOW_ALL=true`（`src/config.js:90`）仅影响某些功能的本地模式放行，素材读写始终走本地引擎。

## 4. 中转站 → 素材的迁移（启动播种 + 本地化）

**本地化落库（`Zr` 把外部地址转 `/files/migrated/...` 后 `Sv` 入库）在源码中共有 4 处调用点**：

1. **启动挂载迁移 ①**：`src/App.js:3428-3447`（`Y.useEffect` 内 `Q.getObject(Z.TRANSIT_RESOURCES)`）。
2. **节点挂载迁移 ②**：`src/App.js:6774-6793`（节点 effect，结构同上，`ne && Q.getObject(...)`）。
3. **首启动"播种"（幂等）**：`src/App.js:40196` 起（`if (await Q.getConfig(kv)) return;` 保护；合并 localTool KV 与 localforage 两份 `TRANSIT_RESOURCES` 去重，`:40197-40198`），逐条 `Zr`+`Sv`，成功打 kv 标记避免重复。
4. **实时捕获落库**：`resourceAdded` 消息处理器（`src/App.js:40634-40671`，`Zr` 本地化在 `:40647`）。

前 3 处属"启动/播种期读 `TRANSIT_RESOURCES` 做迁移"，第 4 处属"网页捕获实时落库"。以 `src/App.js:3429-3447` 为例：

## 5. 素材库的增删查改

封装在 `src/services/resourceStore.js`，全部打 `vv`（`LOCAL_ENGINE.base`）：

- **规范化** `yv`（`resourceStore.js:5-12`）：`isFavorite` 布尔化、`timestamp` 转数字、空串字段删除（`pageUrl/pageTitle/source/folder/name`）。
- **构造查询串** `bv`（`resourceStore.js:13-16`）：`page/pageSize/sortBy/sortDir/search/filters`（filters 为 JSON）。
- **列表** `xv`（`resourceStore.js:17-33`）：GET `/api/resources?...`，默认 `sortBy=timestamp&sortDir=DESC`；返回 `{items,total,page,pageSize,totalPages}`。
- **保存** `Sv`（`resourceStore.js:34-46`）：POST `/api/resources/save`，返回 ok/false。
- **收藏** `Cv`（`resourceStore.js:47-52`）：复用 `Sv` 并带 `isFavorite`。
- **删除** `wv`（`resourceStore.js:53-61`）：POST `/api/resources/delete?id=`。
- **清空** `Tv`（`resourceStore.js:62-78`）：POST `/api/resources/clear`，body `{folder, deleteFiles}`，返回 `{deleted}`。
- **重扫描** `Ev`（`resourceStore.js:79-88`）：POST `/api/resources/rescan`，返回 `{count}`。资源面板每次打开（`active` 变 true）自动 `rescan` 一次，手动刷新按钮同理，生成完成也触发。

前端调用：`xe()`(列表分页)、`be()`(构造 filters)、`we()`(统一同步：先 `Ev` 清孤儿再拉全量)、`mi()`(收藏)、`hi()`(清空)、`yi()`(删除)、`vi()`(复制) 等，均在 `src/App.js` 资源面板 `Qn` 内。

**rescan 孤儿清理（服务端）**：`handleResourcesRescan`（`resources.ts:37-133`）遍历 upload 子目录（`tasks/migrated/...`），把磁盘文件/文件夹以 `source='local-tool'` 录入；随后删除"库中存在但磁盘已不存在"的记录（`resources.ts:120-130`，`orphanDeleted`）。

> ⚠️ **服务端对"删文件"的真实语义（改码前必看）**：
> - `handleResourcesDelete`（`resources.ts:202-209`）：`DELETE FROM resources WHERE id = ?` —— **只删 DB 行，不删磁盘文件**。
> - `handleResourcesClear`（`resources.ts:211-238`）：带 `folder` 时 `DELETE FROM resources WHERE folder = ?`；`deleteFiles=true` 时 `fs.rmSync(folderPath,{recursive:true,force:true})` —— **删除整文件夹目录**。不带 `folder` 则清整张表；`deleteFiles=true` 时删除 upload 目录下所有子目录文件。
> - **SQL 未对 `is_favorite` 过滤**。前端清空确认框文案称"收藏的资源保留"，但服务端 clear 按 folder 整删，收藏项一并移除——这是事实代码行为，改码前需对齐文案与语义。

## 8. 门控 / 降级

## 11. 改码前必查

1. **本地引擎端点一致性**：前端 `resourceStore.js` 的 `/api/resources*` 必须与 `localTool/src/index.ts:182-199` 路由完全一致；`useLocalTool.js`/`localToolClient.js` 的 `/api/files*`、`/api/kv*` 必须与 `index.ts:123-159` 一致；新增端点须同步改两端。
2. **`vv` 基准**：所有资源请求走 `vv`（`LOCAL_ENGINE.base`）。改 base/端口要同步 `src/config.js` 的 `LOCAL_ENGINE`（host/port，默认 18080）与 `localTool` 的 `PORT`（`index.ts:22`）。`useLocalTool.js` 直接拼 `http://127.0.0.1:${Bc}`，`Bc`=`LOCAL_ENGINE.port`，与 `vv` 同源，改端口只动 `config.js`。
3. **`TRANSIT_RESOURCES` 键一致性**：真实字符串 `"transitResources"` 在 `storageKeys.js:13`、`background.ts:99`、`App.js:3429/6774/40197`、`storage/index.js` 多处引用，改名须全量替换。导出排除（`App.js:41671`）与本地引擎同步包含（`storage/index.js:323`）要保持"不同步 GAS、可同步本地引擎"的对齐。
4. **迁移逻辑幂等**：启动播种用 kv 标记（`src/App.js:40196`）与 localforage/KV 合并去重；改动前确认不会重复入库或漏播。
5. **`Zr` 本地化铁律**：任何外部地址（http/data:/blob）入库前必须转内部 `/files/migrated/...`，否则跨设备/跨环境破图（`src/App.js:40643-40645`、本地化失败整条丢弃 `:40650-40653`）；`source==='local-tool'` 跳过下载。
6. **rescan 孤儿清理**：`resources.ts:120-130` 会在 rescan 时删除磁盘已失的 `local-tool` 记录；改动删除逻辑前确认不会误删用户手动放入的文件。
7. **delete 与 clear 对文件的实际影响（重点）**：`wv`/delete 仅删 DB 行**不删磁盘文件**（`resources.ts:202-209`）；`Tv`/clear 带 `deleteFiles=true` 会删除**整个 folder 目录**（`resources.ts:215-219`），且 SQL **未按 `is_favorite` 过滤**——前端清空确认框"收藏的资源保留"与实际"按 folder 整删（含收藏）"不符，改动前务必对齐文案与语义。
8. **GAS 同步范围（键集已修正为 10 键）**：当前 `syncToCloud` 推送 `app_settings, api_configs, users, membership, projects, presetPrompts, customNodeTemplates, modelSchedules, cloud_storage_config, local_templates`（`App.js:41062`）。若希望素材跨设备同步，必须在该 for 循环显式加入对应键，并确认云端 schema 兼容 `resources` 的大体积/二进制字段（当前刻意排除，原因见 §9）。
9. **门控条件**：任何新增资源操作需检查 `S.status.isConnected`（本地引擎，来自 `useLocalTool` 或 `S.status`）与插件环境（`chrome.runtime`），避免无连接时空操作。`resourceStore` 底层失败仅 `console.error` 不抛异常（`:43,59,76,86`），UI 侧自行处理。
10. **`useLocalTool` 端点一致性**：画布/节点组件走 `useLocalTool.js`（`/api/files*`、`/api/kv*`），顶层走 `localToolClient.js`/`resourceStore.js`（`/api/resources*`），二者都解析到 `LOCAL_ENGINE.base`；改端口只动 `src/config.js` 的 `LOCAL_ENGINE`，莫在调用处硬编码。
11. **导出排除对齐**：本地导出（`App.js:41671`）显式排除 `transitResources`；若新增"中转站/素材应进备份"的需求，须同步改此处与 `syncToCloud` 键集合（§9），并注意本地引擎文件本体并不随 JSON 导出。
12. **`REMOTE_BASE` 切换**：`localEngineBase()`（`config.js:42-44`）按 `USE_LOCAL_ENGINE` 在 `:18080` 与 `:9004` 间切换；当前恒本地，改动开关前确认 9004 在线服务是否仍可用。
