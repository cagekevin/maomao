## 〇、这是什么

把全量配置 + 各项目画布状态导出为单个 JSON 文件（备份 / 迁移），或导入还原。入口在 `ImportExportPanel`，经 `exportData`/`importData` 暴露。

---

## 一、导出 `Ri`（`App.js:41966`）

- 结构 `{localforage:{}, kvStore:{}}`。
- **kvStore 键**（遍历白名单，≈41972）：`users, membership, old_membership, projects, lastOpenedProject, app_settings, presetPrompts, customNodeTemplates, cloud_storage_config, api_configs`。
- **localforage**：除 `img_*` 与 `transitResources` 外的全部键（≈41978）；并额外把每个项目的 `Cr(t.id)`（`canvas-state-v1-<id>`）并入 kvStore（≈41992）。
- 产物 `Blob` → 下载 `yimao-workflow-backup-<YYYY-MM-DD>.json`（≈41995-42000）。
- 另：`syncAllToLocalTool`（≈41944-41960）会把 localforage 画布 + 中转文件迁移进 localTool，是导出前的数据对齐动作（非导出本身）。

---

## 二、导入 `Bi`（`App.js:42005`）

- 解析文件，判定**老格式**（无 `localforage`/`kvStore` 但含 `users/projects/.../apiUrl/apiKey` 等扁平字段）。
- **老格式兼容**（≈42016 / 42018）：
  - kvStore 映射：`users→USERS`、`projects→PROJECTS`、`presetPrompts→PRESET_PROMPTS`、`customNodeTemplates→CUSTOM_NODE_TEMPLATES`、`membership→MEMBERSHIP`、`cloud_storage_config→CLOUD_STORAGE_CONFIG`、`lastOpenedProject`、`apiConfigs→API_CONFIGS`；`app_settings` 由一堆扁平字段（`defaultTextModel`/`*ApiConfigId`/`videoDurations`/`globalPolling*`/`transitGridCols`/`sd2Token`/各 `ApiUrl`/`ApiKey`）拼回。
  - localforage：`transitResources`、`canvas-state-v1-default`（老 `canvasState`/`canvasStateV1`）等。
- **新格式**（≈42019）：直接取 `t.localforage` / `t.kvStore`。
- 写回：localforage 部分 `Sr.default.setItem`（≈42022）；kvStore 部分 `Q.setObject`（≈42026，需 localTool 已连）。
- 完成后 `setTimeout(reload, 1500)`（≈42028）重载页面生效。

---

## 三、一句话

`导出：白名单 KV 键 + 全量 localforage（除 img_/transitResources）+ 各项目 canvas-state-v1-* → 单 JSON；导入：老格式自动展平兼容 → localforage.setItem + Q.setObject → reload`。

---

## 四、改码前必查

1. **白名单同步**：新增持久化存储键时，若需进备份，必须在 `Ri` 的 `kvStore` 遍历列表（≈41972）登记，否则导出漏字段。
2. **老格式兼容别删**：`Bi` 的扁平映射（≈42016/42018）是历史备份的还原通道，改动后须保留兼容分支。
3. **`transitResources` 不进 kvStore 导出**：导出 localforage 时显式跳过（≈41978），但老格式导入会带 `transitResources`；中转文件本体在 localTool 18080（见 `中转资源本地落盘机制.md`），导出文件不含实际二进制。
4. **导入后必 reload**：`Bi` 依赖页面重载生效，改动勿去掉 `setTimeout(reload)`。
