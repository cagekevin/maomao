## 〇、这是什么

与 `资源上传机制.md`（云端 `upload/asset` 返回公网 URL）互补：本篇讲**本地**文件落盘——把远程 / 生成的图片、视频、音频下载到 localTool `:18080` 文件系统，回贴本地绝对 URL，满足红线「严禁直接用 CDN/远程 URL，必须先落 18080」（CLAUDE.md §3.2）。

---

## 一、存储键

- `Z.TRANSIT_RESOURCES` = `transitResources`、`Z.TRANSIT_GRID_COLS` = `transit_grid_cols`（`storageKeys.js:13-14`）——中转资源**索引**（KV），但**文件本体在 18080 文件系统，不在 KV**。

---

## 二、核心函数 `Zr`（uploadRemoteUrlToLocalTool）

- 定义 `src/services/localToolClient.js:67`；从 App.js 经 `localToolClient` 导入（App.js:23 解构 `Zr`）。
- 行为：`new FormData()` 附 `fileUrl` + `subfolder`（默认 `canvas`）；`fetch(`${Hr}/api/files/upload`, POST)`，`Hr` = `http://127.0.0.1:18080`；返回 `{url, thumbnailUrl, path}` 或 null。
- 调用点：画布模板插入 `Ur`（`App.js:34326`）→ `Zr(e, {subfolder:'canvas/template'})`（≈34333），把远程 https 落 18080 再回贴。

---

## 三、localTool FILE api（区别于 KV api）

- 上传 `POST /api/files/upload`（`useLocalTool.js:63`）：`file`(File/Blob) 或 `fileUrl` 二选一 + `subfolder`；返回相对 `/files/<subfolder>/<basename>` → 前端补 `http://127.0.0.1:18080` 前缀（useLocalTool.js:71-73）。
- 服务端 `localTool/src/routes/files.ts:14` `handleUpload`：multipart 或 `fileUrl`（服务端 `fetch` 下载后落盘）；`subfolder` 默认 `canvas`（files.ts:28）；落盘返回 `/files/...`。
- 对比 KV api：`/api/kv/set`·`/api/kv/get?key=`（`useLocalTool.js:79`/`96`）——`StorageManager Q` 走这条；中转文件走 FILE api，是两套通道。
- 其它 FILE 接口：`/api/files/thumbnail`、`/api/files/mkdir`、`/api/files/move`、`/api/files/read`、`/api/files/open`、`/api/status`。

---

## 四、与 AI 生成流水线的关系

- AI 结果取回后（`App.js` 轮询 SUCCESS 分支 ≈13486）经 `Xr(t,{subfolder:'tasks'})`（同源 18080 上传）下载结果到 18080，再 `addTransitResource(url, type, 'generated')`（≈13522）登记进 `transitResources`，并回写节点 `data`（≈13514-13521）。

---

## 五、一句话

`远程/生成资源 → Zr → POST 18080 /api/files/upload（subfolder 区分 canvas/template、tasks）→ 落文件系统返回本地绝对 URL → 回贴节点 data + 登记 transitResources`。

---

## 六、改码前必查

1. **红线**：节点 data 里严禁留 CDN/远程 URL，必须经 `Zr`/`Xr` 落 18080 再回贴（CLAUDE.md §3.2）。
2. **FILE vs KV 别混**：中转**文件**走 `/api/files/upload`，中转**索引**（`transitResources`）走 KV；改存储层逻辑别把文件当 KV 写。
3. **subfolder 规范**：模板插入用 `canvas/template`、AI 结果用 `tasks`；新增落盘点须带语义化 subfolder，避免平铺。
4. **落盘即时性不同**：FILE api 收到即写盘（与 KV「仅退出落盘」不同），但 localTool 进程被杀仍可能中断在途上传。
