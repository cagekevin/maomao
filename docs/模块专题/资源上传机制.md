## 一、这是什么

为某些第三方 API 网站（节点生成、外部接口）设计的云端资源上传。这些网站不接受 `data:image/...` base64 或本地 blob，必须给一个公网可访问的 https URL。这套机制把本地/生成的图片、视频、音频上传到云端，返回 https URL 供其消费。

---

## 二、完整生命周期

- **入口函数 `Zh(file, filename, opts)`**（`src/App.js` ≈29029）：
  1. 前置校验 `Oa()`（必须已登录，否则抛「请先登录」）；
  2. 构造 `FormData`：`file` / `folder`（`opts.folder`）/ `bizType`（`opts.bizType`）；
  3. `fetch(`${qh}/upload/asset`, {method:'POST', headers:Jh(), body:FormData})`；
  4. 解析 `{success, data:{url}}`，失败抛「素材上传失败 (status)」；
  5. 返回云端 `url`（https）。
- **端点 `qh`**：`var qh = ${$n}${er}`（≈29016），拼出云端网关基址 + 路径前缀；`Jh()`（≈29017）注入登录鉴权头（`Authorization: Bearer <token>` 或空）。
- **典型调用点 `urlifyAsset`**（各节点消费，如 `ImageNode` ≈1772、`PromptNode` ≈3058、`VideoNode` ≈7664、`CustomNode` ≈12505、`InfiniteCanvas.handleFileUpload` ≈34173）：节点要对外传资源时，先 `urlifyAsset` → 失败则 fallback base64。画布模板保存时 `qr()` 也走 `Hr`→`Zh` 把图片云端化（folder=`templates`）。
- **消费面**：节点 data 里的 `url/Url/images/resultData`，以及 `fileToUrlNode` / `urlToImageNode` 等需要公网 URL 的场景。

---

## 三、一句话

`本地/生成资源 → FormData → POST {qh}/upload/asset（带登录 token）→ 返回 https URL → 喂给不能传图的 API 网站`。

> 这套是**文件/二进制**上云，返回 URL，不是存 JSON 配置。视频资源在模板场景保持原样不强制云端化。

---

## 四、改码前必查

1. **别动 `Zh` 的鉴权头 `Jh()`**：资源上传依赖登录 token，改坏会导致所有节点对外传图失败。
2. **`qh` 是云端网关基址**：改动端点拼接（`${$n}${er}`）会影响所有资源上传调用点。
