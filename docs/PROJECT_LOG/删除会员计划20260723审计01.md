## 一、🔴 致命：Item 7「删除整个请求 pipeline」会破坏本地基建

**计划原文**：删 `Tg/Cg/Sg/wg/Dg/Eg/Og/kg/Ag/Mg/jg` 全家桶，称"整个请求 pipeline 全部打给 `$n`（官方服务器）"。

**实测反驳**：
1. 函数清单本身写错：`Mg` 不是函数，是 L35890 的别名对象 `{get:Eg,post:Dg,...,setAuthToken:ka,...}`；真正函数是 `Sg/Cg/wg/Tg/Eg/Dg/Og/kg/Ag/jg`（无 `Mg` 函数，且少了说明 `Mg` 是别名）。
2. **`Eg` 被本地插件清单加载使用**：`Ng` 在 L35902 调 `Eg(`${localEngineBase()}/plugin/manifest.json`, {useLicenseServer:true, skipAuth:true})`——这是**本地引擎**插件清单，走绝对本地 URL，与云端无关。
3. **扩展更新函数也走 `Eg`**：`Kg/Jg/Gg`（L36320-36559）经 license server 调 `Eg`，属本地插件/扩展管理。
4. `Tg/Cg/wg/Sg` 是**通用基建**：`Cg`(35789) 注入头、`wg`(35797) 是带超时的 `fetch`、`Sg`(35786) 做 URL 解析、`Tg`(35805) 是核心包装（含 401 处理）。它们对本地/云端一视同仁。

**结论**：不能删 pipeline。正确做法是**保留 `Tg/Cg/Sg/wg/Eg/Og/kg/Ag/jg/Mg` 基建，只删云端调用点**：
- 删 7 个 `Dg(...)` 调用（36158/36187/36213/36974/37486/39682/39761）；
- 删云端 `Eg(...)` 调用：`/workflow-apps/*`（36853/36866/37108/37241/37436/39331/39739）、`/user/info`（37436）；
- 这些调用点删除后，pipeline 仅被本地插件清单 `Eg` 使用，无副作用。

## 二、🔴 严重：跨文件耦合未标注（auth.js ↔ App.js）

**计划把以下列为独立删除项**：Item 4 删 auth.js `Oa/ka/Aa/Na`；Item 7 删 `jg`/`Mg`。

**实测耦合**：
- `Cg`（App.js L35792-35793）调用 `Oa()` 注入 `Bearer`；
- `jg`（L35888）`return Oa()`；
- `Mg` 别名对象（L35896-35898）的 `setAuthToken:ka / clearAuthToken:Aa / getCurrentToken:jg` 全指向 auth.js。
- 顶栏登出 `Pe(false), ..., ka(''), ..., Q.remove(Z.AUTH_TOKEN)`（L41002）也调 `ka`。

**风险**：删 auth.js 的 `Oa/ka/Aa/Na` 后，App.js 内 `Cg`/`jg`/`Mg` 变 `ReferenceError`，打包/运行即崩。

**修正**：
- 改 `Cg`：去掉 `Oa()` 分支，直接 `return new Headers(e.headers || sr)`（本地调用全带 `skipAuth:true`，无需 Bearer）；
- 改 `jg`：返回 `''` 或删除该别名项；
- 改 `Mg` 别名：删 `setAuthToken/clearAuthToken/getCurrentToken` 三项（保留 get/post/put/delete/patch）；
- L41002 登出里移除 `ka('')` 与 `Q.remove(Z.AUTH_TOKEN)`。

## 三、🔴 严重：urlTools 删除范围过宽（Item 8）

**计划**：删 `$n/er/nr/sr` + `Jh/Zh/Xh/qh` 依赖。

**实测**：
- `sr` 被 `Cg`（L35790 `new Headers(e.headers || sr)`）使用 → **本地管线依赖 `sr`**，不能删；
- `$n`/`er` 仅被 `Sg`（L35787）用于相对 URL 解析到官方，本地调用传绝对 URL 不经此分支，删 `$n/er` 不影响 `Sg` 本地行为，可删；
- `nr` 6 处全是云端跳转（见§〇），可删，但需确认 L34924/L41770 是否在活跃 UI（见§四.3）。

**修正**：`sr` 必须保留（或把其默认值内联进 `Cg`）；`$n/er/nr` 可删；`Jh/Zh/Xh/qh` 需先 grep 确认除 App.js 外无其他引用再删。

## 四、🔴 严重：Item 9 与 改造项 A 自相矛盾（Ue）

**计划冲突**：Item 9 要删 `Ue` onLoginSuccess（L40150-40188）；改造项 A 要保留 `Ue({nickname:'本地用户',...})` 本地注入。

**实测**：L40150-40188 正是 `Ue` 的**函数定义**（`Ue = e => {...}`），本地注入在 L40286 调用的就是这个 `Ue`。二者是同一函数体，删 40150-40188 会把本地注入一起删掉。

**本地注入实测**（L40284-40299）：
- L40286 `Ue({nickname:'本地用户',username:'local',...,modelApiTokenKey:''})`；
- L40294 `Le({balance:{totalBalance:0},proxyBalance:{totalBalance:0},...})` 兜底，防顶栏 `Ie.balance` 崩；
- **关键**：本地分支 L40284 内**没有** `Pe(true)`（40164 的 `Pe(true)` 在在线写入分支内）。即本地模式 `Ne`(登录态) 恒为 false，会员态由 `Wn` 推 UNLIMITED（见§五.2）。

**修正**：保留 `Ue` 函数壳，仅裁切在线写入部分（L40165-40187 的 `Oa()`/`ji(cr(...))`/`modelApiTokenKey`/`Q.setObject(Z.MEMBERSHIP)`/`Wn(t)`），保留 L40286-40299 本地注入。

## 五、🔴 严重：Item 1 / Item 20 的 `Z.USERS` 并非云端桩

**计划**：Item 1 把 `Q.setObject(Z.USERS)` 当成云端提示词桩一并删；Item 20 删 `Z.USERS` 存储键。

**实测**：`Z.USERS` 是**本地用户列表 KV 存储**，与云端无关：
- L40192 `Q.getObject(Z.USERS)` 读本地用户列表（喂顶栏用户下拉 `ze/Be`）；
- L40680 / L40896 / L41636 `Q.setObject(Z.USERS, ze)` 写本地用户列表（多用户切换功能）。

**风险**：删 `Z.USERS` 键会让上述 4 处写入 `.catch` 吞错、读返回空，**破坏本地"多用户管理/用户下拉"**。这与"只删云端、保留本地"目标冲突。

**决策点（需你确认）**：本地用户列表是否保留？
- 若保留：Item 20 不能删 `Z.USERS`，Item 1 不能删其读写；
- 若删除：需把 L40192/L40680/L40896/L41636 的 `USERS` 读写一并移除，并确认顶栏用户下拉因此失能是可接受的。

## 六、🟠 重要遗漏与修正清单

### 6.1 `Zr` 重载冲突（Item 10）
`Zr` 在 L25 从 `localToolClient` 导入，用于**资源本地化**（`await Zr(resource.url,...)` 在 3435/6780/34018/40208/40647）；同时 L41001 有本地 `Zr = async () => {}` 登出函数（重赋值遮蔽 import）。删登出 `Zr` 时**只能删 L41001 的赋值 + 按钮 L42054**，绝不能动 import 或本地化调用，否则资源迁移/本地化全崩。

### 6.2 本地 UNLIMITED 注入必须保留（Item 17 误伤风险）
`type:'UNLIMITED'` 实测仅出现在 L40302（位于 L40284 本地分支内），是 `Wn`(会员态) 的 UNLIMITED 注入点，是本地模式核心。
- Item 17 要删"会员过期闸门 L40272-40279"（读 `Z.MEMBERSHIP` 做过期判断）——这段是兜底的存储恢复，删掉无妨（本地不落 `Z.MEMBERSHIP`），但**L40302 的 `Wn({type:'UNLIMITED'})` 必须保留**。
- 执行前请先读 L40300-40305 确认 UNLIMITED 注入完整保留。

### 6.3 `nr()` 活跃 UI 待确认（Item 27）
6 处 `nr()` 全是云端 URL，但 L34924(`href:nr('/docs')`) 与 L41770(`window.open(nr(),'_blank')`) 是否在 `false &&` 禁用区内**未核实**（41932-41998 确在禁用区 41887-42001 内，但 34924/41770 不在该范围）。若处在活跃 UI，删 `nr` 会白屏，需改为空函数或一并删 UI。

### 6.4 `modelEntitlements.js` 改造陷阱（Item E/13/14/15）
- Item 15 要删 `xi/bi` 在线数据变量，但 `xi`/`bi` 被 `ki`(L32/L34)、`Ni/Pi`、`Bi`、`Xi`、`Qi/$i/ea`、`j_()` 大量引用。删变量声明会让这些保留函数 `ReferenceError`。
  **修正**：保留 `let xi=[]; let bi=null;` 声明，只删 `ji`/`Mi` 函数及其在线赋值。
- L41002 登出调用 `Mi()` 重置权益；删 `Mi` 后需把该调用移除。
- `LOCAL_MODE_ALLOW_ALL_MODELS` 初始化在 L9（来自 config），保留。

### 6.5 登出串 `Pe(false)` 连带（Item 10/11）
L41002：`Pe(false), Le(null), ka(''), Mi(), Q.remove(Z.AUTH_TOKEN)`。删 token/权益后，需同步移除 `ka('')`、`Mi()`、`Q.remove(Z.AUTH_TOKEN)` 三项，否则 `ReferenceError`/无效调用。

### 6.6 在线分支内的其它耦合（删 40150-40188 时一并处理）
- L40172 `tr($n)` / `ar(e,true)`：替换 apiConfig url 里的 `{VITE_API_BASE_URL}` 为官方 `$n`，属在线分支，随删；
- L40165 `ji(cr(``), t, true)`：`cr()` 取 base URL，删 `ji` 后移除。

### 6.7 `Ne`(isLoggedIn) 门控（Item 12）
`Ne` 在 L40146 `let [Ne, Pe] = Y.useState(false)` 是 App 根级登录态，本地模式常驻 false。计划称 `!Ne` 仅门控 workflow-apps（云端，本就删）。**执行前需精确 grep 大写 `Ne`（非小写 `ne`/`Ne` 局部变量）全量用法**，确认无本地功能被 `!Ne` 门控而误杀。注意其他文件 import 的 `Wt as Ne` 是另一语义，别混淆。
