## 前置验证结论（已 grep 源码坐实）

### 余额：安全删除
`Ie.balance` / `Ie.proxyBalance` 只在被 `false &&` 停用的顶栏展示区被读取（App.js L41887-41934 余额区块、L41980-42001 会员信息区块）。**代码中无任何地方在任务提交前校验余额**。

### 请求链路：关键发现
- `Tg()`/`Dg()`/`Eg()` 整个请求 pipeline（L35786-35899）统一走 `Sg` 构造 URL，`Sg` 默认 base 是 `$n`（官方服务器），`Cg`(L35789) 在 `!skipAuth` 时注入 `Bearer ${Oa()}`
- 9004 网关（`config.js`：`DEFAULT_ENDPOINT='http://127.0.0.1:9004'`）是独立 endpoint，生成请求经 `R/_/g` base（非 `$n`），Bearer 用 `apiConfigs` 模型 key，**与 `Oa()`/token 体系解耦** → **删 `Oa()/ka()/Aa()` 不影响 9004 生成**
- `Dg()` 调用点：`/auth/send-code`、`/auth/login-phone`、`/auth/login-password`、`/workflow-apps/*`、`/user/change-password`，全部打给官方服务器
- ⚠️ **`$n` 不能删**：它是应用部署基址（`urlTools.js` 中 `$n=Vn()` 来自 `entry.js`），被本地功能依赖——`Sg`(L35787 默认请求 base)、`ar`(L12702 aiApp URL 重写)、`tr`/`cr`。删 `$n` 会让所有经 `Sg` 的请求与 aiApp 重写崩溃。只删纯跳转 `nr`，保留 `$n/er/sr/tr/ar/cr`。

### 登出按钮
`Zr` at App.js L41001-41003（`Zr = async () => { Pe(false), Le(null), ka(``), Mi(), Q.remove(Z.AUTH_TOKEN) }`），未在 `false &&` 内，但用户确认已隐藏。
⚠️ **`Zr` 名称复用风险**：L40208/3435/6780/34018/40647 的 `Zr(resource.url,{subfolder})` 是资源本地化调用（带参），与 L41001 无参登出共用变量名。删除时**只删 L41001-41003 的登出赋值体**，必须确认资源本地化真定义在删除后仍然可用，禁止全局按名删 `Zr`。

## 七、首次执行翻车记录（2026-07-23）

### 翻车 #1：验证命令用错

**现象**：全程用 `npx vite build` 验证，全部通过。但 `npm run build` 报 `invalid JS syntax`。

**根因**：`npx vite build` 比 `npm run build`（经 `cross-env NODE_OPTIONS=--max-old-space-size=1024` 包装）更宽松，能放过 import 不存在的导出、模块级孤儿语句等语法错误。

**对策**：CLAUDE.md 已更新，**改码后验证必须用 `npm run build`，禁止 `npx vite build`**。

### 翻车 #2：Node 脚本 brace 计数漏删孤儿代码

**现象**：`cleanup-membership.cjs` 用 brace 计数删除 `Zh` 函数和过期闸门，但：
- `Zh` 的 `async function` 体删了函数头，留了 `if (!Oa())` / `let r = new FormData()` 等模块级孤儿语句
- 过期闸门的 Promise 链 `.then(e => { ... })` 同样被部分删除，留下孤儿 `.then`

**根因**：混淆代码中 async/await 和复杂 Promise 链导致 brace 计数错位。

**对策**：
1. 脚本删完后**必须跑 `npm run build` + 检查 acorn 解析**，不能单靠脚本日志
2. 对 async 函数或复杂 Promise 链，改用 `replace_in_file` 手动精确删除
3. 每次脚本执行后立即 `npm run build`，过不了就停

### 翻车 #3：死 UI 删除脚本破坏语法结构

**现象**：`cleanup-dead-ui.cjs` 用 `indexOf` + `slice` 删除 `false &&` UI 块时，切掉了兄弟元素之间的逗号分隔符，导致 JSX 语法断裂。

**对策**：死 UI 块在 `false &&` 内不渲染也不影响运行时，**不要用脚本批量删**。用 `replace_in_file` 手动处理或直接保留。

### 执行经验总结

| 教训 | 下次怎么做 |
|------|-----------|
| 验证命令 | 只用 `npm run build` |
| 批量删除脚本 | 跑完立刻 `npm run build`，过不了就 `git checkout -- src/App.js` |
| async/Promise 函数 | 不用 brace 计数，手动 `replace_in_file` |
| 死 UI 块 | 保留不删，或手动精确替换 |
| 小步提交 | 每个独立改动单独 commit，便于回退 |
