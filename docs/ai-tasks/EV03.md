# EV03 — 事件总线字典提取（6 个）

> 标注每个 CustomEvent 的派发方和消费方。

## 方法
1. grep 事件名在 App.js + src/ 其他文件中的出现
2. 找到 dispatchEvent → 派发方
3. 找到 addEventListener → 消费方
4. 说明事件携带的 detail 字段

## 输出格式

```
open-shortcuts-modal:
  派发: 设置菜单按钮 onClick
  消费: App.js → Le(true) 打开快捷键面板
  detail: 无
```

### `mousemove`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生（DOM 事件，**非 CustomEvent**；全仓无 `new CustomEvent('mousemove')` 派发） |
| 消费 | `src/App.js` 多处 `window.addEventListener('mousemove', handler)`，均为拖拽/缩放/选区交互。代表点：L1033（画布 hover）、L1918-1920、L1999-2001（节点缩放）、L4513、L4594、L5363、L20767-20769、L21021（画布平移/框选）、L27684-27686、L28015、L28352、L34893、L39247 |
| detail | 原生 `MouseEvent`（`clientX`/`clientY`/`button`/`buttons`/`shiftKey`/`ctrlKey` 等），无自定义 `detail` |

### `mouseup`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生（DOM 事件，**非 CustomEvent**；全仓无 `new CustomEvent('mouseup')` 派发） |
| 消费 | `src/App.js` 多处 `window.addEventListener('mouseup', handler)`，与 `mousemove` 配对用于结束拖拽。代表点：L1918、L1999、L4513、L4594、L5363、L20767、L21021、L27684、L28015、L39247 |
| detail | 原生 `MouseEvent`（同上），无自定义 `detail` |

### `mutiwindow-open-builtin-settings`
| 属性 | 值 |
|------|----|
| 派发 | `src/App.js` 多处按钮 `onClick`：`window.dispatchEvent(new CustomEvent('mutiwindow-open-builtin-settings'))`，无 detail。代表点：L3019、L3967、L7580、L24205（设置/节点菜单按钮） |
| 消费 | `src/App.js` L41164 `window.addEventListener('mutiwindow-open-builtin-settings', e)` → 处理 `e`：`Te('settings'); De('builtin')`（打开设置面板并切到内置子页）；L41168 cleanup 中 `removeEventListener` |
| detail | 无（`new CustomEvent` 未传 `detail`） |

### `mutiwindow-open-schedule-settings`
| 属性 | 值 |
|------|----|
| 派发 | `src/App.js` 节点「模型调度」配置按钮 `onClick`：`window.dispatchEvent(new CustomEvent('mutiwindow-open-schedule-settings'))`，无 detail。代表点：L2980（绘图）、L3928（文生文）、L24168（AI 空间文本） |
| 消费 | `src/App.js` L41164 `window.addEventListener('mutiwindow-open-schedule-settings', t)` → 处理 `t`：`Te('settings'); De('builtin'); setTimeout(() => dispatchEvent('builtin-panel-switch-schedule'), 50)`（打开设置→内置子页→切到调度页）；L41168 cleanup 注销 |
| detail | 无（`new CustomEvent` 未传 `detail`） |

### `mutiwindow-rerun-task`
| 属性 | 值 |
|------|----|
| 派发 | `src/App.js` `Ei` 函数（资源面板「重新执行」按钮）L41779：`window.dispatchEvent(new CustomEvent('mutiwindow-rerun-task', { detail: { task: e } }))`；`e.nodeId` 不存在则 `Br('该任务无关联节点,无法重新执行')` 后早退，不广播 |
| 消费 | `src/App.js` L34421 `window.addEventListener('mutiwindow-rerun-task', e)` → 读取 `e.detail.task`，按 `task.nodeId` 重新派发生成任务；cleanup 中 `removeEventListener` |
| detail | `{ task: <task 对象> }`（task 含 `nodeId`；无 `nodeId` 则派发方早退） |

### `mutiwindow-task-completed`
| 属性 | 值 |
|------|----|
| 派发 | `src/App.js` **5 处**，且均仅在 `e.nodeId` 存在时派发：<br>• L36294（AI 轮询完成/失败）：`{ taskId, nodeId, resultUrl, type, status, errorMsg }`<br>• L41030（资源落盘失败）：`{ taskId, nodeId, resultUrl:undefined, type, status:'failed', errorMsg:'任务未找到或已被清理' }`<br>• L41066（资源落盘成功）：`{ taskId, nodeId, resultUrl, type, status:'completed', errorMsg:undefined }`<br>• L41087（资源落盘失败）：`{ taskId, nodeId, resultUrl:undefined, type, status:'failed', errorMsg }`<br>• L41842（统一同步后）：`{ taskId, nodeId, resultUrl, thumbnailUrl, type, customOutputType, status, errorMsg }` |
| 消费 | `src/App.js` L29213 `window.addEventListener('mutiwindow-task-completed', e)` → 触发后端 rescan 统一入库（`Ev()`，L29211 调用）；L29214 cleanup 中 `removeEventListener`。是资源面板刷新的唯一主驱动 |
| detail | 公共字段 `{ taskId, nodeId, resultUrl, type, status, errorMsg }`；L41842 额外携带 `thumbnailUrl`、`customOutputType` |
