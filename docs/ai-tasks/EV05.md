# EV05 — 事件总线字典提取（3 个）

> 标注每个 CustomEvent 的派发方和消费方。

## 方法
1. grep 事件名在 App.js + src/ 其他文件中的出现
2. 找到 dispatchEvent → 派发方
3. 找到 addEventListener → 消费方
4. 说明事件携带的 detail 字段

## 输出格式

```
open-shortcuts-modal:
  派发: 设置菜单按钮 onClick
  消费: App.js → Le(true) 打开快捷键面板
  detail: 无
```

### `pointerup`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生派发（指针抬起），代码中无 `dispatchEvent('pointerup')`。 |
| 消费 | 1) `src/App.js:5526-5527` 画布自由绘制拖拽工具：以 `window.addEventListener('pointerup', l)` 注册拖拽结束回调 `l`，结束 `drawing` 状态并落笔（`V(), ie(), oe()`）。<br>2) `src/App.js:20821` 检视器 range 滑块：`h`(pointerdown) 中 `window.addEventListener('pointerup', m)`，抬起后由 `m` 移除监听并调用 `endInteraction(p)` 结束交互。 |
| detail | 无。原生 `PointerEvent`，无自定义 `detail`；坐标等数据直接由 `event.clientX/clientY` 提供。 |

### `resize`
| 属性 | 值 |
|------|----|
| 派发 | `src/App.js:17432` 全景节点组件 `Yc`（`Y.memo`）在宽高 / `panoType` 变化后，通过 `window.dispatchEvent(new Event('resize'))` 主动派发（延迟 10/50/150/300ms 多次），强制下游重算布局/画布尺寸。 |
| 消费 | 多个尺寸测量 hook 以 `window 'resize'` 作为 `ResizeObserver` 不可用时的兜底：<br>`src/App.js:23837`（元素尺寸测量 hook）、`24906`、`24956`（多 ref 自适应）、`25966`（元素尺寸）、`29183`（`Tt` 重新测量 `xe` 元素 `getBoundingClientRect`）。收到事件后重算尺寸/布局。 |
| detail | 无。`new Event('resize')` 未携带 `detail`。 |

### `seeked`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生派发（`<video>` 完成 `currentTime` seek 后），代码中无 `dispatchEvent('seeked')`。 |
| 消费 | 视频抽帧/缩略图逻辑，均 `addEventListener('seeked', r)` 等待 seek 完成后读取帧数据：<br>`src/App.js:14347`（按时间 seek 后 `drawImage` 截缩略图并 `toDataURL`）、`14371`（16×16 像素采样 `getImageData`）、`14761`（`Is()` 辅助函数 seek 到指定时间后 `resolve`）。 |
| detail | 无。原生 `Event`，帧数据直接读取 `video` 元素，不通过 `detail` 传递。 |
