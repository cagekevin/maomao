# NC03 — 节点组件契约提取（6 个）

> 提取每个节点的 props、回调、产出 data 字段、上下游连接关系。

## 方法
1. 在 App.js 中找到对应函数定义（grep variable name）
2. 读取函数参数（prop 解构）→ 得到 props
3. 搜索 onClick/onChange/onSubmit → 得到 callbacks
4. 搜索 W(setNodes) 或 updateNodeData → 得到 output 字段
5. 搜索 onAddImage/onPushImagesToImageBox 等 → 判断是否 AI 生成类

## 输出格式

```
imageNode (li):
  props: {id, data{imageUrl,label,thumbnailUrl}, selected}
  callbacks: onCrop(), onZoom(), onEdit(), onSendToActiveTab()
  output: {imageUrl, thumbnailUrl, loading, errorMsg}
  AI生成: 是 → handleImageGenerate → :9004 → 回写 imageUrl
  connected: 接收 promptNode/urlToImageNode → 输出到 gridSplitNode
```

### rhWebappNode (`Ms`)
| 属性 | 值 |
|------|----|
| props | `{id, data{webappId, aiAppApiUrl, aiAppApiKey, schema[], values{}, uploadStatus{}, uploadError{}, webappName, webappDesc, webappTags[], covers[], schemaLoading, schemaError, loading, status, errorMessage, taskId, lastResultTaskId, taskCostTime, consumeMoney, finalPrice, preDeductAmount, openAppSelectorOnMount, membershipType, addTransitResource, updateGlobalTasks, onShowToast}, selected}` |
| callbacks | `onRun()` 校验+POST `/run` 发起任务并启动轮询；`onStop()` 停止轮询置 IDLE；`onSelectApp(appId)` 重载 schema；`onOpenMarket()` 打开应用市场；`onUploadFile(field,file)` 本地上传并自动建 imageNode/audioPlayerNode 连边；`onDisconnectVar(field)` 删除 `var-{field}` 边；`onValueChange(field,val)` 写 `values[field]`；`pollTask(taskId)` 轮询 `/task/{id}` 成功则 spawn 结果节点 |
| output | `webappId, schema, webappName, webappDesc, webappTags, covers, preDeductAmount, values, uploadStatus, uploadError, uploadSourceSig, schemaLoading, schemaError, loading, status, errorMessage, taskId, lastResultTaskId, taskCostTime, consumeMoney, finalPrice`；结果经 `ee()` 用 `setNodes` 直接新建 `imageNode`/`audioPlayerNode`/`textNode`（带 `hasChanged`） |
| AI生成 | 是 → `onRun` → `POST rr(p,'/run')` → 轮询 `rr(p,'/task/{id}')` → 成功 spawn 结果节点（不走固定端口回写，动态建节点） |
| 上游连接 | target handle `var-${fieldName}`（每个 schema 字段一个：`IMAGE`/`VIDEO`/`AUDIO` 接收对应图片/视频/音频节点 `imageNode`/`videoNode`/`audioPlayerNode`；其余类型接收 `textNode` 等文本/数值节点） |
| 下游连接 | source handle（右侧，无 id）；运行结果以新建 `imageNode`/`audioPlayerNode`/`textNode` 形式出现在右侧；同时经 `addTransitResource` 推入中转站 |

### videoExtractNode (`Ns`)
| 属性 | 值 |
|------|----|
| props | `{id, data{mode('count'\|'interval'\|'smart'\|'first_last'\|'manual'), intervalSec, frameCount, sensitivity, hiddenIndices[], videoUrl, videoName, errorMessage, loading, progress, extractedImages[], allExtractedImages[], onShowToast, onExtractFrames}, selected}` |
| callbacks | `onFileUpload(file)` 选本地视频写 `videoUrl/videoName`；`onManualCapture()` 手动模式 canvas 截当前帧；`onExtract()` 按模式抽帧写 `allExtractedImages/extractedImages/progress`；`onCopy()` 复制已提取帧到剪贴板/缓存；`onConfigToggle()` 展开/收起参数 |
| output | `videoUrl, videoName, errorMessage, loading, progress, extractedImages, allExtractedImages, hiddenIndices, mode, intervalSec, frameCount, sensitivity, onExtractFrames` |
| AI生成 | 否（纯前端 canvas 抽帧） |
| 上游连接 | target handle（左侧，无 id）→ 接收 `videoNode` / `imageNode`(含视频 url) / `textNode`(含 url) 的视频 |
| 下游连接 | source handle（右侧 id `main-output`）→ 输出 `allExtractedImages/extractedImages`，供 `imageCompressNode`/`faceMosaicNode`/`imageBoxNode` 等图片消费节点连接 |

### videoToGifNode (`Gs`)
| 属性 | 值 |
|------|----|
| props | `{id, data{fps, maxSize, colors, speed, videoUrl, videoName, errorMessage, loading, progress, resultInfo{width,height,frameCount,size}, onShowToast, onAddImage, onSpawnImageNode}, selected}` |
| callbacks | `onFileUpload(file)` 选本地视频 → `onAddImage` 推入；`onGenerate()` 调 `Ls()` 转 GIF，写 `resultInfo`，`onSpawnImageNode` 新建 GIF 节点；`onShowToast` |
| output | `videoUrl, videoName, errorMessage, loading, progress, resultInfo{width,height,frameCount,size}, fps, maxSize, colors, speed` |
| AI生成 | 否（前端 gif.js 合成 `Ls`） |
| 上游连接 | target handle（左侧，无 id）→ 接收 `videoNode`/`imageNode`(video url)；`Ws()` 取首个含 `videoUrl` 的源节点 |
| 下游连接 | source handle（右侧 id `main-output`）；运行时 `onSpawnImageNode` 新建 `imageNode` 承载 GIF（url 为 blob） |

### imageCompressNode (`nc`)
| 属性 | 值 |
|------|----|
| props | `{id, data{maxSize, quality, format, targetKB?, imageUrls[], loading, progress, errorMessage, resultInfo{count,totalOriginal,totalSize}, onShowToast, onAddImage, onSpawnImageNode, onPushImagesToImageBox}, selected}` |
| callbacks | `onFileUpload(files)` 多选图片 → `onAddImage` 推入；`onCompress()` 调 `Xs()` 压缩每张写 `resultInfo`，优先 `onPushImagesToImageBox` 否则 `onSpawnImageNode` 逐个新建；`onShowToast` |
| output | `imageUrls[](tc 聚合), loading, progress, errorMessage, resultInfo{count,totalOriginal,totalSize}, maxSize, quality, format, targetKB` |
| AI生成 | 否（前端 canvas 压缩 `Xs`） |
| 上游连接 | target handle（左侧，无 id）→ 接收 `imageNode` 等图片节点（`tc()` 聚合所有源 `imageUrl`） |
| 下游连接 | source handle（右侧 id `main-output`）；运行后通过 `onPushImagesToImageBox`（下游为 `imageBoxNode` 时）或 `onSpawnImageNode` 新建压缩图节点 |

### faceMosaicNode (`Cc`)
| 属性 | 值 |
|------|----|
| props | `{id, data{mode('mosaic'\|'blur'\|'pixel'\|'bar'\|'grid'), strength, color, imageUrls[], loading, progress, errorMessage, resultInfo{count,faceTotal}, resultUrls[], onShowToast, onAddImage, onSpawnImageNode, onPushImagesToImageBox, onZoom}, selected}` |
| callbacks | `onFileUpload(files)` 多选 → `onAddImage`；`onModeChange(mode)` 切打码模式；`onStrengthChange`/`onColorChange` 强度/颜色；`onManualMosaic()` 手动打码编辑器 `bc` 写 `resultUrls`；`onAIMosaic()` 调 `_c()` 人脸检测打码写 `resultInfo{faceTotal}` 并输出；`onZoom(url)` 放大查看；`onShowToast` |
| output | `imageUrls[](Sc 聚合), loading, progress, errorMessage, resultInfo{count,faceTotal}, resultUrls[], mode, strength, color` |
| AI生成 | 是（AI 打码路径 `onAIMosaic` → `_c()` 人脸检测+打码；另含手动打码 `onManualMosaic`） |
| 上游连接 | target handle（左侧，无 id）→ 接收 `imageNode` 等图片节点（`Sc()` 聚合 `imageUrl`） |
| 下游连接 | source handle（右侧 id `main-output`）；输出通过 `onPushImagesToImageBox`（下游 `imageBoxNode`）或 `onSpawnImageNode` 新建打码图节点 |

### compareNode (`Lc`)
| 属性 | 值 |
|------|----|
| props | `{id, data{split(0~1), orientation('v'\|'h'), labelA, labelB, onShowToast, onSpawnImageNode}, selected}` |
| callbacks | `onDragSplit(x,y)` 拖动分隔条调 `split`；`onToggleOrientation()` v/h 切换；`onReset()` split=0.5、视频归零；`onPlayPause()` 播放/暂停（视频）；`onSyncTime()` 同步两端视频时间；`onExportImage()` `Mc()` 合成对比图 → `ii()` 上传 → `onSpawnImageNode` 新建对比图节点；`onRecordVideo()` `Pc()` 录制对比视频 → `onSpawnImageNode` 新建对比视频节点（提示可连 `videoToGifNode`）；`onShowToast` |
| output | `split, orientation`（持久化）；运行结果以新建 `imageNode`(对比图)/`imageNode`(对比视频) 形式输出，无 data 结果字段回写 |
| AI生成 | 否（前端 canvas 合成 `Mc` / 录制 `Pc`） |
| 上游连接 | target handle（左侧，无 id）→ 接收 2 个图片/视频节点（`imageNode`/`videoNode`），取前 2 个不同 url |
| 下游连接 | source handle（右侧 id `main-output`）；运行结果经 `onSpawnImageNode` 新建 `imageNode`(对比图)/`imageNode`(对比视频)；对比视频节点常连 `videoToGifNode` |
