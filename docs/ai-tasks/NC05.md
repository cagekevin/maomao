# NC05 — 节点组件契约提取（2 个）

> 提取每个节点的 props、回调、产出 data 字段、上下游连接关系。

## 方法
1. 在 App.js 中找到对应函数定义（grep variable name）
2. 读取函数参数（prop 解构）→ 得到 props
3. 搜索 onClick/onChange/onSubmit → 得到 callbacks
4. 搜索 W(setNodes) 或 updateNodeData → 得到 output 字段
5. 搜索 onAddImage/onPushImagesToImageBox 等 → 判断是否 AI 生成类

## 输出格式

```
imageNode (li):
  props: {id, data{imageUrl,label,thumbnailUrl}, selected}
  callbacks: onCrop(), onZoom(), onEdit(), onSendToActiveTab()
  output: {imageUrl, thumbnailUrl, loading, errorMsg}
  AI生成: 是 → handleImageGenerate → :9004 → 回写 imageUrl
  connected: 接收 promptNode/urlToImageNode → 输出到 gridSplitNode
```

### stickyNoteNode (`Uh`)
> 定义：`// [✔ 已确认] Uh — TextOverlayEditor`（App.js:27558）
> 本质：富文本便签/文字叠加标注节点，纯前端状态，`updateNodeData` 自动回写，无任何 AI 调用。

| 属性 | 值 |
|------|----|
| props | `{id, data}`<br>data 读取字段：`html`(n.html)、`text`(n.text，经 `Hh()` 转 html)、`fontSize`(??24)、`bold`(??false)、`color`(??Rh[0].value)、`bgColor`(??Lh[0].value)、`width`(??400)、`height`(??400)。<br>新建默认 data：`{text:'', fontSize:24, bgColor:'rgba(180,160,60,0.25)', textColor:'rgba(255,255,255,0.92)'}`（App.js:33609）。<br>注：函数签名未解构 `selected`。 |
| callbacks | `onDoubleClick`(j) 进入编辑态；`onInput`(F) contentEditable 实时写 html；`onBlur`(N) 提交并退出；`onKeyDown`Esc(M) 取消；`onBgColorChange`(p) 改底色；`onColorChange`(R) 改字色；`onBoldToggle`(z) 加粗；`onFontSizeChange`(ee/B) 字号增减；`onEmojiInsert`(te) 插 emoji；`onContextMenu`(P) 右键菜单；`onResize`(ne) 右下角拖拽调宽高。`Rh`/`Lh`/`zh`/`Bh` 为色板与字号/emoji 预设数组（App.js:27498-27546）。 |
| output | 通过 `useEffect` 自动 `updateNodeData(e, {html, fontSize, bold, color, bgColor, width, height})` 回写（App.js:27582-27592）。即产出 data 字段：`html, fontSize, bold, color, bgColor, width, height`。 |
| AI生成 | 否（无 AI 生成，无 `:9xxx` 调用） |
| 上游连接 | 无（组件内无 `Handle` 组件，不接收连线） |
| 下游连接 | 无（无 source handle，不输出连线） |

### group (`Eh`)
> 定义：`// [✔ 已确认] Eh — GroupNode`（App.js:26324）
> 本质：节点容器/编组节点，可折叠，通过 `setNodes` 改写自身样式与 `parentId` 关系，非数据流节点。

| 属性 | 值 |
|------|----|
| props | `{id, data, selected}`<br>data：`name`(t?.name \|\| `编组`)、`collapsed`(t?.collapsed \|\| false)、`expandedWidth`、`expandedHeight`。<br>样式从 `t.style?.width/height` 或 `t.measured` 读取；新建默认 `data:{name:'编组'}`（App.js:34610）。 |
| callbacks | `onNameChange`(d) 名称输入；`onNameCommit`(f) 经 `updateNodeData(e,{name})` 回写名称；`onKeyDown`Enter(p) 提交名称；`onToggleCollapse`(m) 折叠/展开：用 `setNodes` 改写 `style`+`data.collapsed/expandedWidth/expandedHeight`（App.js:26348-26390）；`onDoubleClick` 进入名称编辑态。 |
| output | `updateNodeData({name})`；`setNodes` 改写 `{collapsed, expandedWidth, expandedHeight, style}`（折叠时 width:`max-content` height:40，展开时还原）。 |
| AI生成 | 否（无 AI 生成） |
| 上游连接 | 有隐藏 target handle（Left，`!opacity-0`，不可见），理论上可接收上游连线 |
| 下游连接 | 有隐藏 source handle（Right，`!opacity-0`，不可见），理论上可向下游连线；实际主要作为容器分组（将选中节点设 `parentId` 归入该 group）而非数据连线（App.js:34597-34619）。 |
