# EV02 — 事件总线字典提取（6 个）

> 标注每个 CustomEvent 的派发方和消费方。

## 方法
1. grep 事件名在 App.js + src/ 其他文件中的出现
2. 找到 dispatchEvent → 派发方
3. 找到 addEventListener → 消费方
4. 说明事件携带的 detail 字段

## 输出格式

```
open-shortcuts-modal:
  派发: 设置菜单按钮 onClick
  消费: App.js → Le(true) 打开快捷键面板
  detail: 无
```

### `keydown`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生（window 自动派发 `KeyboardEvent`）；App.js 中无任何主动 `dispatchEvent` |
| 消费 | App.js 中多处 `window.addEventListener('keydown')`：<br>• L1981–1984 / L4606–4609 / L5202–5205 / L15814–15817 / L41172–41175：监听 `Escape` 关闭对应弹窗/面板<br>• L5534–5544（capture 阶段）：全局快捷键分发，输入框/可编辑元素内忽略，命中则触发绑定动作<br>• L26129–26133（capture 阶段）：画布 `Ctrl/Cmd + C/V/Z/Y` 复制/粘贴/撤销/重做<br>• L27985–27991（与 `keyup` 配对）：预览可见且焦点不在输入框时，`Space` 触发播放 `L(true)`<br>• L34804–34808（与 `paste` 配对）：快捷键触发粘贴 `kr()` |
| detail | 原生 `KeyboardEvent`（`key`、`code`、`ctrlKey`/`metaKey`/`shiftKey`、`target` 等），无自定义 `detail` |

### `keyup`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生（`KeyboardEvent`），无主动 `dispatchEvent` |
| 消费 | App.js L27992–27996：`window.addEventListener('keyup', t)`（与 `keydown` 配对）；`Space` 松开时执行 `L(false)`、`R(false)` 停止播放 |
| detail | 原生 `KeyboardEvent`，无自定义 `detail` |

### `load`
| 属性 | 值 |
|------|----|
| 派发 | `FileReader` 读取完成后由浏览器自动派发（`Np` 函数 L24369–24372：`new FileReader().addEventListener('load', …)`），无主动 `dispatchEvent` |
| 消费 | App.js L24372：`FileReader` 实例监听 `load`；读取成功时将 `reader.result`（模型文件文本）经 `Promise.resolve` 返回，否则 `reject(Error('模型文件读取失败'))` |
| detail | 无 `detail`；数据通过事件目标 `reader.result` 取得（字符串） |

### `maomao:presetsChanged`
| 属性 | 值 |
|------|----|
| 派发 | `src/services/promptManager.js` L62–67 `saveAndNotify(presets)`：`await savePresets(presets)` 后 `window.dispatchEvent(new CustomEvent('maomao:presetsChanged', { detail: presets }))` |
| 消费 | App.js L40868–40871：`window.addEventListener('maomao:presetsChanged', e => e.detail && Array.isArray(e.detail) && Mr(e.detail))`，调用 `Mr` 同步最新预设数据 |
| detail | `presets` —— 保存后的预设数组（与 `savePresets` 入参一致） |

### `mousedown`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生（`MouseEvent`），无主动 `dispatchEvent` |
| 消费 | App.js 中多处 `document`/`window.addEventListener('mousedown')`：<br>• L20653–20662（与 `keydown` 配对）：点击下拉外部则关闭 `s(false)`<br>• L26821–26830（capture，延时注册）：点击缩略图菜单外部（非 `[data-thumb-menu]`/`[data-thumb-menu-portal]`）关闭菜单<br>• L38678–38685（延时注册）：点击 popover 外部则关闭 `m(false)`<br>• L2178 / L3481 / L6851 / L7852 / L9498 / L24061（capture，条件注册）：点击外部关闭各类菜单/弹层 |
| detail | 原生 `MouseEvent`（`target`、`clientX`/`clientY` 等），无自定义 `detail` |

### `mouseleave`
| 属性 | 值 |
|------|----|
| 派发 | 浏览器原生（`MouseEvent`），元素鼠标移出时自动派发（元素 `e.addEventListener('mouseleave', …)`） |
| 消费 | App.js L1030–1034：鼠标移出自定义元素时，将位移样式 `--cust-shift-x` / `--cust-shift-y` 重置为 `0px`（与 `mousemove` 配对，用于拖动偏移回弹） |
| detail | 原生 `MouseEvent`，无自定义 `detail` |

---

> 备注：除 `maomao:presetsChanged` 为代码主动 `dispatchEvent` 的 `CustomEvent` 外，其余 5 个均为浏览器原生事件，派发方为 DOM/浏览器，代码中未发现主动 `dispatchEvent` 调用。以上消费方行号基于 `src/App.js`；`src/components/common/ResizableTextarea.js` 等处仅以 React `onKeyDown` 合成事件形式消费，未使用 `addEventListener`，故未列入。
