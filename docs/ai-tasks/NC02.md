# NC02 — 节点组件契约提取（6 个）

> 提取每个节点的 props、回调、产出 data 字段、上下游连接关系。

## 方法
1. 在 App.js 中找到对应函数定义（grep variable name）
2. 读取函数参数（prop 解构）→ 得到 props
3. 搜索 onClick/onChange/onSubmit → 得到 callbacks
4. 搜索 W(setNodes) 或 updateNodeData → 得到 output 字段
5. 搜索 onAddImage/onPushImagesToImageBox 等 → 判断是否 AI 生成类

## 输出格式

```
imageNode (li):
  props: {id, data{imageUrl,label,thumbnailUrl}, selected}
  callbacks: onCrop(), onZoom(), onEdit(), onSendToActiveTab()
  output: {imageUrl, thumbnailUrl, loading, errorMsg}
  AI生成: 是 → handleImageGenerate → :9004 → 回写 imageUrl
  connected: 接收 promptNode/urlToImageNode → 输出到 gridSplitNode
```

### videoNode (`Do`)
| 属性 | 值 |
|------|----|
| props | `{id, data, selected, width}`，其中 data 关键字段：`prompt, aspectRatio, size, customSize, selectedSeconds, selectedModel, videoModel, videoDurations, selectedContextResources, apiFormat, expanded, videoUrl, thumbnailUrl, imageAvailable, loading, progress, errorMessage, inputWidth, inputHeight, presetPrompts, label` |
| callbacks | `onGenerateVideo(id, prompt, aspectRatio, size, model, seconds, apiFormat)`、`onAddImage(id, url)`(参考图上传)、`onDelete()`、`onStop(id)`、`onShowToast(msg)`；点击卡片切换 `expanded` |
| output | 回写 data：`prompt, aspectRatio, customSize, selectedModel, selectedSeconds, expanded, inputWidth, inputHeight, imageAvailable, thumbnailUrl, errorMessage, loading, progress, videoUrl` |
| AI生成 | 是 → 触发 `onGenerateVideo` → 经网关(`http://127.0.0.1:9004`) → 回写 `videoUrl`(+`thumbnailUrl/loading/progress`) |
| 上游连接 | `target`(左) 接收 `imageNode / videoExtractNode / imageBoxNode` 的图片，以及其它节点文本；`@` 选中的素材写入 `selectedContextResources` |
| 下游连接 | `source`(右) 输出 `videoUrl`，供下游视频类/导出节点消费 |

### sd2VideoNode (`Ao`)
| 属性 | 值 |
|------|----|
| props | `{id, data, selected}`，data：`prompt, size, selectedSeconds, selectedModel, sd2VideoModel, videoDurations, selectedContextResources, internalResources, uploadedAssets, expanded, videoUrl, thumbnailUrl, loading, progress, errorMessage, inputWidth, inputHeight, presetPrompts, label` |
| callbacks | `onGenerateSD2Video(id, prompt, size, model, seconds)`、`onUploadAsset(url, type)`(素材上传网关)、`onDelete()`、`onStop(id)`、`onRefresh(id)`、`onShowToast(msg)`；上传文件自动创建并连线 `imageNode / videoExtractNode / audioNode` |
| output | 回写 data：`prompt, size, selectedModel, selectedSeconds, expanded, inputWidth, inputHeight, internalResources, selectedContextResources, uploadedAssets, videoUrl, thumbnailUrl, loading, progress, errorMessage` |
| AI生成 | 是 → 触发 `onGenerateSD2Video` → 网关(`:9004`) → 回写 `videoUrl` |
| 上游连接 | `target`(左) 接收含 `imageUrl / videoUrl / audioUrl` 的连线节点 + `@` 选中素材 + 节点内 `internalResources`；区分 images/videos/audios/texts 四类 |
| 下游连接 | `source`(右) 输出 `videoUrl` |

### discountVideoNode (`os`)
| 属性 | 值 |
|------|----|
| props | `{id, data, selected}`，data：`prompt, size, resolution, selectedSeconds, selectedModel, discountVideoModel, videoDurations, selectedContextResources, uploadedAssets, expanded, videoUrl, thumbnailUrl, loading, progress, errorMessage, inputWidth, inputHeight, presetPrompts, label` |
| callbacks | `onGenerateDiscountVideo(id, prompt, aspectRatio, model, seconds, resolution)`、`onUploadAsset(url, type)`、`onDelete()`、`onStop(id)`、`onRefresh(id)`、`onShowToast(msg)`；上传文件自动创建并连线 `imageNode / videoExtractNode / audioNode` |
| output | 回写 data：`prompt, size, resolution, selectedSeconds, selectedModel, expanded, inputWidth, inputHeight, selectedContextResources, uploadedAssets, videoUrl, thumbnailUrl, loading, progress, errorMessage` |
| AI生成 | 是 → 触发 `onGenerateDiscountVideo` → 网关(`:9004`) → 回写 `videoUrl`；生成前按模型规格 `Go/he` 校验比例/分辨率/时长 |
| 上游连接 | `target`(左, `variant:large`) 接收含 `imageUrl / videoUrl / audioUrl` 的连线节点 + `@` 素材；文本仅接收 `textNode / audioNode / textConcatNode` |
| 下游连接 | `source`(右, `variant:large`) 输出 `videoUrl` |

### audioNode (`cs`)
| 属性 | 值 |
|------|----|
| props | `{id, data, selected}`，data：`prompt, maxDuration, pauseGap, audioUrl, audioName, chunks, text, loading, errorMessage, audioApiUrl, audioApiKey, audioModel` |
| callbacks | 内部 `onGenerateAudio`(断句/听音，调用 `ss()` 转录 whisper-1)；文件上传 `onChange` 写入 `audioUrl/audioName`；`copy JSON`；`onShowToast(msg)` |
| output | 回写 data：`audioUrl, audioName, chunks`(断句结果 JSON)、`text`(`JSON.stringify(chunks)`)、`loading, errorMessage` |
| AI生成 | 是(听音断句) → 用自备 `audioApiUrl + audioApiKey + audioModel`(默认 whisper-1) 调用 `ss()` → 回写 `chunks/text`；非中央网关，而是用户配置的转录 API |
| 上游连接 | `target`(左) 接收含 `audioUrl / videoUrl(音频) / imageUrl(音频) / text(含音频URL)` 的连线节点，自动提取首个音频 URL 写入 `audioUrl` |
| 下游连接 | `source`(右) 输出 `text / chunks`，供 `textNode / textConcatNode` 等使用(label 显示「听音断句结果」) |

### audioPlayerNode (`ps`)
| 属性 | 值 |
|------|----|
| props | `{id, data, selected}`，data：`audioUrl, audioName` |
| callbacks | 文件上传 `onChange` → `set audioUrl/audioName`、清除 `v()`、下载 `x()`；`onShowToast` 未使用 |
| output | 回写 data：`audioUrl, audioName`；本地将上游/远程音频 fetch 为 blob 得 `playUrl` 用于 `<audio>` 播放 |
| AI生成 | 否（纯播放器，无 AI 生成） |
| 上游连接 | `target`(左, `variant:small`) 经 `ls(data)` 解析连线节点的音频 URL（或 `@` 选中）；无音频时提示从左侧连线接入 |
| 下游连接 | `source`(右, `variant:small`) 输出 `audioUrl` |

### customNode (`ms`)
| 属性 | 值 |
|------|----|
| props | `{id, data, selected}`，data：`config{apiUrl,method,headers,body,outputType,executionMode,resultPath,taskIdPath,polling*,rawTextOutput,variableFormats}, configMode, resultData, loading, progress, errorMessage, label` |
| callbacks | `onAIAssist(prompt, config)`(AI 辅助生成配置)、`onSaveTemplate(name, config)`(保存为模板)、`onGenerateCustom(id)`(开始处理)、`onStop(id)`、文件上传 `x`(变量素材)、完成配置 `v()` 写回 `config/configMode` |
| output | 回写 data：`config, configMode, resultData`(文本/图片/视频/音频 URL)、`loading, progress, errorMessage`；每个变量有独立输入值存于 `variables` |
| AI生成 | 是(万能节点调用自定义 API，可能经网关) → 触发 `onGenerateCustom` → 按 `config` 发请求（sync 或 async 轮询 `polling*`）→ 回写 `resultData`；可选 `onAIAssist` 自动生成 `config` |
| 上游连接 | 每个变量对应一个 `target`(左, `variant:small`)，handle `id="var-{name}"`；图片/音频/视频/文件类变量可连线或本地上传 |
| 下游连接 | `source`(右, `variant:small`) 输出 `resultData` 供下游节点消费 |
