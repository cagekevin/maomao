# AB05 — 节点函数体重写（3 个）

> ⚠️ 铁律：改写后功能必须和原来完全一样。只改局部变量名和加注释，不改任何逻辑、不删任何代码、不变任何行为。

> 在 App.js 内重写函数体。不改文件结构，写坏只影响一个节点。

## 规则

1. **不改函数签名和模块级变量名**（ImageNodeComp 等已语义化）
2. **重命名函数体内局部变量**：`a,b,c` → `loading, setLoading`
3. **加段落注释**：`// ── 状态 ──` `// ── 回调 ──` `// ── 渲染 ──`
4. **保持 vendor/React 用法不变**：不动 `Y.useState`、`X.jsx` 等
5. **不改 JSX 结构**：不动 children 树
6. **输出到本文件：在任务表格中填写「重写后代码」列

## 示例

```js
// 重写前
var ImageNodeComp = Y.memo(({ id, data, selected }) => {
  let [a,b] = Y.useState(false);
  let c = Y.useRef(null);
  let d = Y.useCallback(() => { b(true); upload(); b(false); }, []);
  return X.jsxs("div", { ... });
});

// 重写后
var ImageNodeComp = Y.memo(({ id, data, selected }) => {
  // ── 状态 ──
  const [loading, setLoading] = Y.useState(false);

  // ── 上传 ──
  const handleUpload = Y.useCallback(() => {
    setLoading(true);
    uploadImageToEngine();
    setLoading(false);
  }, []);

  // ── 渲染 ──
  return X.jsxs("div", { ... });
});
```
