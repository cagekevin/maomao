## 0. 安全护栏（最高优先级，任何执行前先读）

本计划所有动作都建立在以下护栏之上，违反任意一条即视为"不安全"，停止执行：

- **禁区（绝对不改）**：`src/App.js`、`src/vendor/`（含 `vendor.js`/`rolldown-runtime.js`）、`dist/`、`*.css`、`src/entry.js`、`src/main.tsx`（已删）、`reference/App.original.js`、`captureVideoFrame-*.js`。
- **已知噪音（不修）**：`9004` 未实现 API 的 `404`、`RootErrorBoundary` 的 `null` 异常、`18080` 硬编码与中文乱码。
- **版本隔离**：只跑 V1，禁止引入任何 V2 代码/逻辑。
- **可回退**：每个文件级改动前先 `git status` 确认工作区干净；任何新增配置都用独立新文件，**不重写既有可运行脚本**，避免破坏现有启动链路。
- **小步提交**：一次只落一类变更，便于 `git revert` 单点回滚。

**保护措施（来自对照计划补强）**
1. **专用分支隔离**：从 `main` 切 `git checkout -b chore/standardize`，所有改动只落该分支；出问题 `git checkout -- <file>` 或整分支丢弃即可回滚，绝不污染 `main`。
2. **基线可构建态锁定**：当前 `npx vite build` 已通过（HANDOFF 记载 commit `176437b`）。每完成一步都必须重新 build 通过才算有效，以该 commit 为可恢复基线。
3. **奥卡姆剃刀**：每项改动先回答"删掉它有影响吗"，答不上来就不做；不建包装层、不建桶文件、不引入新依赖（除非 P0/P1 必须）。

> 若执行中发现某项会触达禁区或噪音，直接跳过该项并记入 §6「计划外风险」。

## 3. 执行顺序与回滚

**分支与提交约定（来自对照计划补强）**
1. 起手：`git checkout -b chore/standardize`，所有改动只落该分支。
2. 推荐提交顺序（每步独立 commit，便于单点回滚）：
```
P0-3 (.editorconfig) → P0-4 (Prettier) → P0-2 (README) → P0-1 (cross-env)
→ P1-1 (pyproject×2) → P1-2/1-3 (requirements) → P1-4 (split_server 归位)
→ P1-5 (.nvmrc) → P2-1 (ESLint) → P2-2/2-3/2-4 (按需、可延后)
```
3. 提交信息清晰：`feat(ci): 跨平台构建与基础规范` / `build(py): 依赖可复现` / `docs: README 与红线摘要`。
4. **合并闸门**：合并 `main` 前再跑一次全量 `npm run build`（Windows/Mac）与 `ruff check`/`eslint`，通过后提请 Review，再合并。

**回滚方式**：
- 配置类（P0-3/4、P1-1/2/3/5、P2-1）：`git revert <commit>` 或删文件即可，零运行影响。
- 脚本改动（P0-1）：保留原 bash 写法，回滚即切回原字段。
- 文件移动（P1-4、P2-3）：`git revert` 后 `git grep` 确认引用复原。

**风险 → 缓解 → 回滚对照表**

| 风险点 | 缓解 | 回滚 |
|---|---|---|
| 改 `build` 脚本导致本机/CI 失败 | 先在 `chore/standardize` 分支验证双平台 | `git checkout -- package.json package-lock.json` |
| Prettier/ESLint 误扫混淆代码 | `ignorePatterns` 排除 `App.js`/`vendor`/`dist` | 删配置文件即可 |
| 移动 `ps_split.jsx`/`split_server.py` 致悬空引用 | 移动前 `git grep` 全仓引用 | `git checkout -- <原路径>` + 还原引用 |
| Python 依赖钉死过严装不上 | 用兼容上界而非硬 `==` | 还原 `requirements.txt` |
| 引入新依赖破坏现有链路 | 奥卡姆剃刀：非必须不加 | `git revert` 依赖改动 |

### 7.1 git 卫生预检（P0 前置，零风险）
- 动作：执行前先跑 `git ls-files | grep -E '\.(pyc|log)$'`，确认无历史误追踪的 `*.pyc`/`*.log`。
- 落点：仅核查，不改文件；若有命中，单独 commit 用 `git rm --cached` 剔除非必要项（保留 `.gitignore` 已忽略项）。
- 验证：命令无输出（或仅剩 `.gitignore` 已覆盖项）。

### 7.2 根级多进程编排脚本（P0/P1 新增）
- 来源：审计报告 §二 2.2（根缺 `start`/`serve`，多服务靠各自脚本）。
- 动作：根 `package.json` 新增编排脚本，例如 `dev:all`（用 `concurrently` 串联 `localTool` + `apimart-gateway`，前端仍走 `vite`）或加 `scripts/dev-all.mjs`。
- 安全说明：纯新增 npm script / 新文件，不重写现有启动链路；`concurrently` 作为 `devDependency` 加入。
- 验证：`npm run dev:all` 能拉起各服务（或至少脚本解析无误）。

### 7.3 脚本模块系统收敛（P2 新增）
- 来源：审计报告 §四 4.2（根 `"type":"module"` 但 `scripts/*.cjs` 为 CommonJS）。

### 7.4 启动器英文命名 / 别名（P2-2 强化）
