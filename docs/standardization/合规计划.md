## 0. 合规护栏（任何改动前先固化）

**禁止触碰（红线，违反即中止）**
- `src/App.js`、`src/vendor/`（含 `vendor.js`/`rolldown-runtime.js`）、`dist/`、`*.css`、`src/entry.js`
- 禁止重建已删除的 `src/main.tsx`；禁止引入 V2 代码

**保护措施**
1. 从 `main` 切出专用分支：`git checkout -b chore/standardize`，所有改动只落该分支，出问题 `git checkout -- <file>` 即可回滚。
2. 锁定**基线可构建态**：当前 `npx vite build` 已通过（HANDOFF 记载 commit `176437b`）。每完成一步都必须重新 build 通过才算有效。
3. ESLint/Prettier 的 `ignorePatterns` 必须显式排除 `src/App.js`、`src/vendor/`，防止对混淆代码误报。
4. 不修已知噪音：`9004` 未实现 API 的 `404`、`RootErrorBoundary` 的 `null` 异常——属红线记载的"别修"项。
5. 奥卡姆剃刀：每项改动先回答"删掉它有影响吗"，答不上来就不做；不建包装层、不建桶文件。

## 4. P2 — 渐进收敛（可控推进）

### 4.1 ESLint 引入
- 根 + `localTool` 加 ESLint flat config；`ignorePatterns` 排除 `src/App.js`、`src/vendor/`、`dist/`。
- 首轮只开"无错误"规则（如 `no-undef`、`no-dupe-keys`），不强制风格；对 `src/utils|services|components|hooks|contexts|config` 等已解耦模块逐步覆盖。
- 验证：`npm run build` 仍通过 + `eslint .` 在范围内无 error。

### 4.2 启动器统一
- 约定：每服务以 `npm run dev`/`npm start` 为唯一跨平台入口。
- 平台专用脚本（`.command`/`.bat`/`.sh`/`.ps1`）收敛到 `tools/launchers/` 或加统一英文前缀，避免四式并存。
- 回滚：脚本类改动不影响运行逻辑，删错即 `git checkout`。

### 4.3 游离文件归位
- `ps_split.jsx`（根）→ 移入 `src/` 或 `scripts/`，并更新任何引用（先 grep 确认无外部 import）。
- `split_server.py`（根，重 ML）→ 移入 `tools/layer-divider/` 并配 `requirements.txt`。
- 验证：移动后 `npm run build` 通过 + grep 确认无悬空引用。

### 4.4 TS 严格度对齐
- 新代码默认趋严；根 `tsconfig` 在非 `App.js` 区域可逐步开启 `noUnusedLocals`/`noUnusedParameters`。
- 保持根（bundler/ESNext）与 `localTool`（Node16）差异"有据可查"，仅在注释说明运行时不同即可。

## 5. 风险与回滚总览

| 风险点 | 缓解 | 回滚 |
|---|---|---|
| 改 build 脚本导致 CI/本机失败 | 先在分支验证双平台 | `git checkout -- package.json package-lock.json` |
| Prettier/ESLint 误扫混淆代码 | `ignorePatterns` 排除禁区 | 删配置文件即可 |
| 移动 `ps_split.jsx`/`split_server.py` 致悬空引用 | 移动前 grep 全仓引用 | `git checkout -- <原路径>` + 还原引用 |
| Python 依赖版本钉死过严装不上 | 用兼容上界而非硬 `==` | 还原 `requirements.txt` |
