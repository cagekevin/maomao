# 校准日志 2026-07-05

## 关键失误

### 1. 写五步法时走了自己的最短路径 — 直接套用原版框架

写 Skill 写作五步法时，我第一版直接平移了原版推理五步法的结构，没有考虑到"写 Skill"和"推理问题"是两个完全不同的场景。原版的核心机制（挂起等待确认）在 Skill 写作场景里水土不服。

- **最短路径**：五步法好用 → 改改描述就交差
- **遗漏的变量**：写 Skill 时用户不在旁边逐步骤确认，用户要的是完整产出物
- **教训**：不能因为一个框架好用就假设它能在所有场景工作，要先分析场景差异

### 2. 给示例导致 AI 照抄

写 Style Direction 说明时给了具体示例（"美式药妆风格，渐变灰白背景…"），被 Kevin 指出 AI 会直接照抄，不管用户实际产品是什么。

- **教训**：给示例要谨慎，AI 是字面理解者，示例=模板

### 3. 把"分步降维"写成 AI 听不懂的术语

自创了"分步降维"这个词，被 Kevin 质疑"AI 能听懂吗？"

- **教训**：写 Skill 时不要用自创术语，用 AI 能直接理解的大白话

### 4. 强推"通用原则" — 把图片和视频的垫图逻辑合并

试图把"垫图为主"提炼成通用原则，被 Kevin 纠正——图片只垫产品图，视频垫人物+场景+产品，两者不一样。

- **教训**：不要为了追求"通用"而抹平领域差异

## 正向记录

1. **State 0 加"为什么"** — 每条"我们不要"附带"为什么 AI 会犯这个错"，让 AI 理解原因而不是死记
2. **附录加入行业 Know-how** — 图片设计的"约束少风格多"、视频设计的"60-20-20"和"先控制再生成"
3. **破坏性校验机制** — State 5 模拟一个喜欢偷懒的 AI 去撞禁忌，测试防呆机制
4. **信任基座原则** — 绝对信任 AI 的基础能力，只保留扭转 AI 默认倾向的增量约束
5. **自检清单** — 产出的 Skill 文件末尾为未来执行者预留自检清单
6. **垫图为主理念** — 提示词不重复描述垫图已有的内容，把注意力放在垫图表达不了的地方
