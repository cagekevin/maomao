## Claude Code + Ollama 本地模型配置：最终发现正确的配法

### 问题现象
- `claude --model qwen3.5:9b` + 环境变量 → 报 `模型不存在: qwen3.5:9b`
- 直接 `curl Ollama 的 /v1/messages` 用 `qwen3.5:9b` → 正常
- 通过 `settings.json` 的 env 设 `ANTHROPIC_MODEL=qwen3.5:9b` → 能连上

### 根因
- **`claude --model xxx` 这个参数传到 Claude Code 内部后，模型名可能被改了（加了 ANSI 码或前缀）**
- 而通过 `settings.json` 的 `env.ANTHROPIC_MODEL` 设置模型名，Claude Code 会原样传给 Ollama

### 正确配置（settings.json）
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "ollama",
    "ANTHROPIC_BASE_URL": "http://localhost:11434",
    "ANTHROPIC_MODEL": "qwen3.5:9b",
    "CLAUDE_CODE_MAX_OUTPUT_TOKENS": "128000"
  }
}
```
- **必须用 `settings.json` 的 env 设 `ANTHROPIC_MODEL`，不能用 `--model` 参数**
- `ANTHROPIC_BASE_URL` 不加 `/v1`
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS` 设大一点防止输出截断

### 还存在的问题
- `ollama launch claude` 列表里能选自定义模型，但选完后发请求也报 `模型不存在`，跟 `--model` 参数同理
- 9B 模型（6.6GB）在 M5 Mac 上跑起来比较卡，响应慢

### 仍需验证
- 用 `my-qwen`（自定义 GGUF 文件）能不能走通？之前模板解析错误可能跟 settings.json 的 model 名设置方式也有关
- `claude launch codex` 的高峰期报错问题跟本地配置无关，是 Codex 云端服务的问题 — 有脚本优先用脚本

## Claude Code + Ollama 本地模型配置：最终发现正确的配法

### 问题现象
- `claude --model qwen3.5:9b` + 环境变量 → 报 `模型不存在: qwen3.5:9b`
- 直接 `curl Ollama 的 /v1/messages` 用 `qwen3.5:9b` → 正常
- 通过 `settings.json` 的 env 设 `ANTHROPIC_MODEL=qwen3.5:9b` → 能连上

### 根因
- **`claude --model xxx` 这个参数传到 Claude Code 内部后，模型名可能被改了（加了 ANSI 码或前缀）**
- 而通过 `settings.json` 的 `env.ANTHROPIC_MODEL` 设置模型名，Claude Code 会原样传给 Ollama

### 正确配置（settings.json）
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "ollama",
    "ANTHROPIC_BASE_URL": "http://localhost:11434",
    "ANTHROPIC_MODEL": "qwen3.5:9b",
    "CLAUDE_CODE_MAX_OUTPUT_TOKENS": "128000"
  }
}
```
- **必须用 `settings.json` 的 env 设 `ANTHROPIC_MODEL`，不能用 `--model` 参数**
- `ANTHROPIC_BASE_URL` 不加 `/v1`
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS` 设大一点防止输出截断

### 还存在的问题
- `ollama launch claude` 列表里能选自定义模型，但选完后发请求也报 `模型不存在`，跟 `--model` 参数同理
- 9B 模型（6.6GB）在 M5 Mac 上跑起来比较卡，响应慢

### 仍需验证
- 用 `my-qwen`（自定义 GGUF 文件）能不能走通？之前模板解析错误可能跟 settings.json 的 model 名设置方式也有关
- `claude launch codex` 的高峰期报错问题跟本地配置无关，是 Codex 云端服务的问题
