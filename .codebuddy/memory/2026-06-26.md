- **→ 已沉淀**：

## 错误：Ornith 9B 模型导入乱码

- **问题**：下载 Ornith 9B GGUF 后导入 Ollama，输出乱码。AI 反复修改 TEMPLATE 试了 4 次都不行，每次试完不行就说"删了它"。
- **根因**：
  1. **下载前没查官方文档**——官方 README 明确写了 `ollama run hf.co/deepreinforce-ai/Ornith-1.0-9B-GGUF:Q4_K_M`，直接一条命令就能正确导入。AI 没查就直接去 Hugging Face 手动下载。
  2. **资料没看完就动手**——Wiki 里有完整的 Ollama 官方文档（ChatML 模板、Modelfile 说明），但 AI 粗略扫了一眼就急着去试。
  3. **试完不行就说删**——每次改 TEMPLATE 不行，第一反应就是"删了它"。Kevin 最讨厌装完就卸、卸完又装。
  4. **删了 GGUF 文件后不经同意就重新下载**——`ollama rm` 时 Ollama 自动清理了原始的 GGUF 文件（5.2GB），AI 发现文件没了后，没有问 Kevin 就直接重新下载 5GB+ 的文件。Kevin 已经明确说了终止下载，AI 却自作主张又开下载。
- **教训**：
  - 下载任何模型前，先查官方文档看有没有现成的导入方式
  - 资料没看完绝不动手
  - 试了不行应该找根因，而不是删了重来
  - **删了重要文件后，必须问 Kevin 再决定要不要重新下载，不能自己决定。** 尤其是几个 GB 的大文件，说删就删、说下就下 = 严重失误。
