## 01画布项目总控

### 错误：$pid 变量冲突导致总控崩溃

- **问题**：总控.ps1 里用 `$pid` 作为变量名，但 `$pid` 是 PowerShell 系统保留变量（当前进程 PID），赋值时报 `VariableNotWritable` 错误
- **根因**：不了解 PowerShell 保留变量机制
- **修复**：改用端口检测（`netstat`）+ `$targetPid` 变量名，彻底避开保留变量
- **→ 已沉淀**：总控.ps1 使用端口检测方案

### 错误：wt 分屏参数多次尝试失败

- **问题**：想用 `wt` 分屏实现优雅启动，但 `--title` 参数在 PowerShell 中被解析为 `--` 运算符，路径含空格导致 cmd 找不到文件
- **根因**：PowerShell 对 `--` 前缀参数的特殊解析 + 路径含空格时引号嵌套层级混乱
- **修复**：放弃 wt 分屏，改用独立 PowerShell 窗口 + 后台隐藏模式 + 日志文件
- **→ 已沉淀**：总控.ps1 采用后台隐藏 + 日志方案

### 错误：菜单状态不实时更新，操作无反馈

- **问题**：按数字键直接执行，用户看不到操作确认，菜单状态也不会自动刷新
- **根因**：用 `ReadKey` 无回显，执行后没有暂停等待状态更新就进入下一轮循环
- **修复**：改用 `Read-Host` 回车确认 + 执行后显示操作提示 + `Start-Sleep` 等待后自动刷新菜单
- **→ 已沉淀**：总控.ps1 的交互模式

### 错误：项目配置写入了校准日志

- **问题**：Kevin 说"写入记忆"时，把项目配置信息写到了校准日志（本该只记 AI 错误）

### 错误：整理作图经验时方向跑偏，太啰嗦效率低

- **问题**：Kevin 让从他的 Skills 里提炼经验，我先是搞了太详细的自检清单（一条条打勾），又搞了太虚的方法论，还给了绝对化的规则（"能垫图就垫图"），结果都不对
- **根因**：
  1. 没搞清楚给谁用的 — 自检清单是给 AI 内部思维链用的，不是给人一条条打勾用的
  2. 总想搞复杂的、系统化的东西，反而忽略了简单直接
  3. 给绝对化的规则，没考虑实际场景的复杂性（垫图不是万能的，产品图适合垫图，场景图不一定）
  4. 不懂装懂，瞎指挥，没有尊重 Kevin 自己的经验和判断
- **修复**：停止整理，写入校准日志
- **→ 已归档**：
  - 整理经验要简单直接，不要搞复杂的系统
  - 不要给绝对化的规则，实际场景很复杂，分情况讨论
  - 不要瞎指挥，多听 Kevin 的，他比我懂

### 学到的：Kevin 的核心作图方法论（从 Skills 横向提炼）

- **系统化思维** — 先搭系统，再出图。不是写一段 prompt，是搭一套可复用的生产流程
- **解耦思维** — 样式和骨架解耦、内容和结构解耦。改哪里动哪里，不牵一发而动全身
- **防错优先** — 把坑在设计阶段就堵死，比出图后再修高效得多
- **分层拼装** — 提示词是乐高积木，分层拼装，条件注入，不是一坨浆糊
- **参考驱动** — 能用参考图锁定的，就不用文字描述。但参考图不是万能的，分场景用
- **闭环思维** — 输入确认 → 生成 → 质检 → 落盘，每一步都不能少
- **最小代价修订** — 出问题只改责任最小的那层，不全量返工
- **克制原则** — AI 对程度词容易走极端，用静态的、状态的、克制的描述
