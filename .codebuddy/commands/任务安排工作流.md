# 分配任务工作流

> ⚡ 奥卡姆剃刀：如无必要，勿增实体。每句删了能不能完成任务？能，就删。

## 一、角色定义

| 角色 | 做什么 |
|---|---|
| **主 AI（委派者）** | 清旧任务书 → 接收需求 → 探索代码 → 写任务书 → 归档 |
| **执行 AI（受命者）** | 按任务书执行，不会回来提问，必须独立完成 |
| **用户** | 给出原始需求 |

**核心纪律**：执行 AI 不会提问。任务书必须自包含，不留模糊空间。

## 二、工作流

### 0. 清理旧任务书

删除 `docs/AI0X/任务书/` 下所有 `TASK-*` 文件，为新一轮任务腾空编号空间。产出一个空白目录。

### 1. 理解需求

确认目标主题、是否禁止参考现有文档、是重写还是新增。产出：需求摘要。

> 需求不明确时标记 [信息缺口] 追问，不用假设填空。

### 2. 探索代码基底

grep 中英文关键词 → 读核心文件 → 标记混淆变量名语义 → 定位关键行区。产出：探索起点清单（路径+行号+grep 词+语义）。

### 3. 设计生命周期覆盖

按项目独特生命周期拆解（非 CRUD 通用模板），枚举"出现的所有地方"。产出：覆盖清单。

### 4. 起草任务书

按以下结构组装：项目背景 → 硬约束 → 探索起点 → 覆盖清单 → 输出规范 → 验收标准 → 铁律文件名（`docs/AI0X/主题.md`）。产出：`docs/AI0X/任务书/TASK-序号-任务短名.md`。

### 5. 归档

扫 `docs/AI0X/任务书/` 已有 `TASK-*`，取最高序号+1，写入。

### 6. 交付话术

输出一行，用户直接看文件路径发给对应 AI 即可：

`任务书：docs/AI0X/任务书/TASK-XXX-任务短名.md → 发给 AI0X`

## 三、绝对禁令清单

1. **文件名铁律**：每次委派必须指定唯一产出路径 `docs/AI0X/主题.md`，不同 AI 不能写同一文件。
2. **禁止参考**：如要求"不参考现有文档"，硬约束必须写明。
3. **探索起点真实**：必须来自本次会话的实际 grep，不能复用历史（行号会变）。
4. **验收标准强制**：每份任务书必须有可自测的验收标准。
5. **序号递增**：编号扫描目录已有 +1，禁止硬编码 `TASK-001`。
6. **不可交互**：任务书禁止出现"如有问题请联系"等假设执行 AI 能提问的表述。
7. **独立文件夹**：每个 AI 的产出放 `docs/AI0X/`，不混入 `docs/模块专题/`。

## 四、示例

> 用户：安排 AI01 重写会员机制文档，不参考现有版本。

- 产出文件：`docs/AI01/会员与登录机制-v2.md`
- 探索起点：grep `会员 / login / token / auth / membership`；读 `src/services/auth.js`、`src/services/modelEntitlements.js`、`src/config/storageKeys.js`、`src/App.js`（登录 L43849、401 登出 L36143、会员过期 L40586、登出 L41308）
- 覆盖清单：两条通路 → 存储键 → 登录协议 → token 收发 → 登录态校验/过期 → 会员等级来源 → 受限功能点 → 登出 → GAS 同步衔接 → 改码必查

## 五、输出前自检

- [ ] 旧任务书是否已清空？
- [ ] 产出文件名是否唯一且不与已有文件冲突？
- [ ] 硬约束是否写明"禁止参考现有文档"（如需）？
- [ ] 是否枚举了"出现的所有地方"（≥3 个，带路径+行号）？

## 六、多 Agent 并行规则（⚠️ 新增）

### 6.1 铁律：不能让多个 agent 同时改同一个文件

| 风险场景 | 正确做法 |
|----------|----------|
| 多人同时改 `src/App.js` | agent 输出到各自任务文件，用合并脚本统一写入 |
| 多人同时改 `docs/func-mapping.txt` | 各 agent 输出到 `docs/ai-tasks/AI0X.md` 表格，你跑 `scripts/merge-all.mjs` 合并 |
| 多人识别同一个代码区域 | 环形交叉分配：每函数 2 个 agent，出结果后对比 |

### 6.2 标准多 Agent 任务模板

```
1. 分配：每个 agent 领独立任务文件 docs/<task-dir>/XX01~05.md
2. 输出：agent 填各自文件的表格，不碰项目源码
3. 合并：你运行合并脚本，统一写入
4. 验证：npm run build && node scripts/check-build.cjs
```

### 6.3 环形交叉验证（识别类任务用）

需要识别函数/API/事件时，每个 target 分配 2 个 agent，交叉覆盖：
- 第 1 组 → AI01 + AI05
- 第 2 组 → AI01 + AI02
- ...环形交替
- 完成后对比：一致采纳，分歧人工裁决

### 6.4 禁止事项

- ❌ 多个 agent 的任务文件指向同一个产出路径
- ❌ agent 直接修改 `src/App.js`（始终走输出文件 + 合并脚本）
- ❌ agent 互相引用对方结果（每个任务文件加"请勿看别人结果"）
