# 智能视频切片引擎 (Intelligent Video Slicing)

## 1. 核心概述

本 Skill 旨在将批量源视频自动切割为适合后期剪辑的短片段，标准时长为 4 至 8 秒。对于难以切割的视频，系统具备自动调参及重试的机制。

该工具已封装为“深模块（Deep Module）”。其内部的所有复杂实现（包含切片、分析、调参、抽帧、改名及清理）均隐藏在 `scripts/` 目录下。用户仅需通过统一的入口文件 `main.py` 调用四个高层命令即可完成操作，无需了解内部 6 个脚本的协作细节。整个 Skill 目录支持整体迁移，具有高度的自包含性。

## 2. 触发条件

当接收到以下意图时，应调用本 Skill：

- “把这个目录的视频切一下”
- “帮我分割画面”
- “素材切片，每段 4-8 秒”
- 任何涉及对批量 MP4 视频进行场景检测与切割的请求

## 3. 标准工作流（Quick Start）

工作流包含四个核心步骤，除第 3 步需人工介入外，其余均全自动执行。

### 步骤 1：自动切片与收敛 (`cut`)

全自动执行切片操作，直到所有片段时长收敛至 4-8 秒/段。系统会自动分析切片时长，对欠切（过长）或过切（过碎）的视频动态调整阈值，并删除需重切目录后重新运行，直至全部合格或达到最大重试次数上限。

- **执行命令**：`python main.py cut --input "源视频目录" --output "输出目录"`
- **核心参数配置**：
  - `--threshold` 或 `-t`：跳剪灵敏度参数，范围设定在 1.5~12 之间。数值越小越灵敏，切分出的片段越多。默认值为 3.0，请勿使用旧版脚本默认的 12。
  - `--min-scene-len`：最小镜头长度（秒）。短原片建议设为 2 以保证节奏，长原片建议设为 4 以防画面过碎，默认值为 2。
  - `--max-retry`：最大自动重切轮数，默认值为 3。

### 步骤 2：批量抽帧 (`frames`)

切片收敛完成后，为所有输出片段提取画面帧，供后续人工分类参考。系统会为每个片段提取 3 帧：首帧 (`_f0.jpg`)、中帧 (`_f1.jpg`) 和尾帧 (`_f2.jpg`)。

- **处理总目录**：`python main.py frames "输出目录"`（自动逐子目录抽取）
- **处理单目录**：`python main.py frames "输出目录/某切片目录"`

### 步骤 3：按类别映射改名 (`rename`) - **【需人工介入】**

此环节需要人工或 AI 读取抽取的截图，判断画面类别，并编制 `映射.json` 文件。JSON 格式规范为 `{"003":"上脸涂抹", "004":"展示脸部状态"}`，其中键为片段编号，值为画面类别。可以参考 `references/map_example.json` 作为编写模板。

- **预览命令（Dry-run）**：`python main.py rename -d "某个切片目录" -m "映射.json" --dry-run`
- **执行命令**：`python main.py rename -d "某个切片目录" -m "映射.json"`
- **命名规则**：重命名格式统一为 `原视频名_类别_序号.mp4`。序号按类别自增，从 `001` 开始计（如 `dedup_xxx_上脸涂抹_001.mp4`）。系统会自动跳过已命名的文件，对于没有映射的片段会发出警告但不做改动。

> **⚠️ AI 操作纪律（工作原则 · 强制）**
>
> 本环节（步骤 3）画面分类**必须由 AI 逐帧、逐个片段地实际判读**，严禁任何投机取巧的批量速成做法：
>
> 1. **逐帧判读，禁止臆测**：每个 clip 的 3 张截图（f0/f1/f2）必须逐一用 `read_file` 实际读取、肉眼判读后再决定类别。不得仅凭文件名、目录命名规律，或"看起来像之前处理过的视频"就套用分类。
> 2. **一步一个脚印**：按 clip 编号从小到大，逐个处理、逐个把判断写入映射 JSON，再执行 `rename`。不要一次性并行读取几十张图后凭整体印象批量归类。
> 3. **不确定即归"其他"**：遇到无法确定的画面，一律归入"其他"，绝不臆测。
> 4. **目前的最快就是最稳**：老老实实读帧分类是正确且最快的路径，不存在"更聪明的捷径"。任何跳过读帧、批量套用规律的做法都会引入错误分类，反而更慢、更糟。

### 步骤 4：清理抽帧目录 (`cleanup`)

重命名全部完成后，清理用于辅助分类的截图文件夹。

- **执行命令**：`python main.py cleanup "输出目录/某切片目录_frames"`
- **安全机制**：该命令仅删除以 `_frames` 结尾的目录，且要求输入 `yes` 进行二次确认，不会擅自删除文件。

## 4. 画面分类标准

在步骤 3 中，需严格对照以下特征判断画面类别。如遇分类不确定的情况，一律归入“其他”类，请勿臆测。

- **上脸涂抹**：用手把护肤品或精华涂在脸上并推开。
- **手持产品**：手里拿着产品包装或瓶子进行展示，且未上脸。
- **展示脸部状态**：正脸怼脸拍，展示皮肤状况或妆前妆后状态，无涂抹动作。
- **手持按摩棒**：手里拿着按摩仪或美容仪在脸上滚动或按压。
- **胶囊精华液**：画面主体是胶囊精华液，且正涂抹或敷在脸上。
- **片尾**：视频结尾的收尾画面，通常为口播结束语、Logo 或引导关注等。
- **其他**：无法归入以上任何一类的画面。（此类文件也照常改名，例如 `dedup_xxx_其他_001.mp4`）。

## 5. 输出报告格式

运行完毕后，系统将输出简表。报告内容包含合格与跳过数量、跳过视频的重试次数与平均时长，以及最终按类别的汇总数据。

## 6. 内部实现架构

`main.py` 利用 `Path(__file__).resolve().parent` 动态定位并使用 `subprocess` 编排 `scripts/` 目录下的 6 个独立脚本：

- `智能分割画面.py`：核心切片脚本，调用 PySceneDetect 与 FFmpeg，支持 NVENC 硬件加速。检测器模式默认使用 `adaptive`（灰度自适应），对“静止背景 + 人物动作”素材效果最优。如需更改，可编辑此脚本顶部的 `DETECTOR_MODE`。
- `analyze_clips.py`：负责质量分析，其中的 `collect_need_retry()` 函数被 `main.py` 复用作为收敛判定标准。
- `delete_undercut.py`：用于删除需重切的目录。
- `extract_frames.py`：负责批量抽 3 帧。
- `batch_rename.py`：处理映射 JSON 驱动的批量改名。
- `cleanup_frames.py`：处理带有防护校验的截图目录删除。

## 7. 环境要求与注意事项

- 必须确保 `ffmpeg` 和 `ffprobe` 已存在于系统 PATH 中。
- 需执行 `pip install scenedetect tqdm` 安装必要的 Python 库。
- 当系统具备 NVIDIA GPU 时，会自动启用 NVENC 硬件加速。需注意，多视频并发处理受限于 GPU 编码会话数，NVENC 模式最多支持 2-3 路并发。

## 8. 实战经验与避坑指南

1. **缓存跳过陷阱**：`智能分割画面.py` 遇到输出子目录已存在且有 `.mp4` 文件时会直接返回并跳过。重切前必须删除目录（`main.py cut` 内部已自动调用 `delete_undercut.py` 处理此环节），切勿手动删除或跳过该步。
2. **固定机位静止镜头**：部分口播或产品展示素材存在 10-20s 无动作且无明暗变化的固定镜头，任何检测器均无法切出跳变点，属物理极限。此类硬伤段在 `cut` 收敛时会直接标记为 `SKIPPED`。请勿强行降低阈值（如 adaptive 1.5）或更换 HSV，否则会导致连贯画面被误切。
3. **检测器选用规则**：针对“静止背景 + 人物动作”的素材，`adaptive`（灰度自适应）表现远优于 `hsv`。实测表明 `hsv` 会出现长达数十秒的明显漏切，因此默认应使用 `adaptive`，`hsv` 仅作理论备用，未调校好前不建议使用。
4. **阈值起点设置**：务必确认切片阈值起点为 3.0。若误用脚本内旧默认值 12，会导致灵敏度过迟钝，出现整批欠切。
5. **PowerShell 输出丢失**：在 PowerShell 内直接运行 Python 脚本常会造成多行输出被吞没（仅返回空壳）。建议将输出重定向到文件（`> out.txt 2>&1`）再读取，或使用 `Get-Content -Encoding utf8` 命令读取。
6. **目录删除方式**：切勿使用 `.bat` 脚本删除含中文或括号的目录，这极易引发路径解析失败。应统一使用 Python 的 `shutil.rmtree`（系统内置的清理脚本已采用）进行删除。
7. **ffprobe 超时处理**：个别损坏或极长的素材可能导致 `ffprobe` 卡死。脚本已设定 30 秒超时机制，若仍遇卡死状况，需针对该文件进行单独排查。
